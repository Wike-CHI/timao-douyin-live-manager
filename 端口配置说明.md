## 🔧 关键端口说明

### 前端端口：10050
- **作用**：Electron桌面应用的Web界面
- **访问地址**：`http://YOUR_IP:10050`
- **配置文件**：`electron/main.js`
- **必须开放**：桌面端用户需要访问此端口

### 后端端口：8181
- **作用**：FastAPI后端API服务
- **访问地址**：`http://YOUR_IP:8181/api/`
- **配置文件**：`server/main.py` 或 Nginx配置
- **必须开放**：前端需要连接后端API

## 🚀 快速配置

### 1. 开放防火墙端口
```bash
cd /www/wwwroot/wwwroot/timao-douyin-live-manager
./open-ports.sh
```

### 2. 验证端口状态
```bash
./verify-ports.sh
```

### 3. 云服务器安全组配置
如果使用腾讯云/阿里云等云服务器：

**登录云控制台** → **安全组** → **添加规则**

添加以下入站规则：
- 端口：`10050`，协议：`TCP`，来源：`0.0.0.0/0`（前端）
- 端口：`8181`，协议：`TCP`，来源：`0.0.0.0/0`（后端）
- 端口：`80`，协议：`TCP`，来源：`0.0.0.0/0`（可选）
- 端口：`443`，协议：`TCP`，来源：`0.0.0.0/0`（可选）

## 🔍 端口检查方法

### 检查端口监听
```bash
# 检查所有关键端口
netstat -tulnp | grep -E ":(10050|8181|80|443)"

# 或使用
ss -tulnp | grep -E ":(10050|8181)"
```

### 检查防火墙规则
```bash
# firewalld
firewall-cmd --list-ports

# ufw
ufw status

# iptables
iptables -L -n | grep -E "(10050|8181)"
```

### 测试端口连接
```bash
# 测试前端端口
curl http://127.0.0.1:10050

# 测试后端端口
curl http://127.0.0.1:8181/health

# 测试外网访问
curl http://YOUR_IP:10050
curl http://YOUR_IP:8181/health
```

## 📝 配置文件位置

### Electron前端配置
- 文件：`electron/main.js`
- 端口配置行：`const PORT = 10050;`

### FastAPI后端配置
- 启动命令中指定：`--port 8181`
- 或在代码中：`uvicorn.run(app, host="0.0.0.0", port=8181)`

### Nginx配置（如果使用）
- 文件：`/etc/nginx/conf.d/timao.conf` 或 `electron/renderer/nginx.conf`
- 需要配置反向代理到 `8181` 端口

## ⚠️ 常见问题

### Q1: 502 Bad Gateway
**原因**：后端服务未运行或端口未开放  
**解决**：
```bash
# 1. 检查后端服务
ps aux | grep python | grep server

# 2. 检查端口监听
netstat -tulnp | grep :8181

# 3. 重启服务
./restart-all-services.sh
```

### Q2: 无法连接
**原因**：防火墙阻止或安全组未开放  
**解决**：
```bash
# 1. 开放防火墙
./open-ports.sh

# 2. 验证端口
./verify-ports.sh

# 3. 检查云服务器安全组
```

### Q3: Connection Refused
**原因**：服务未在正确端口监听  
**解决**：
```bash
# 检查服务启动命令
ps aux | grep uvicorn

# 确认端口参数是 --port 8181
```
