# 🎯 云服务器部署方案总结

> **审查人**: 叶维哲  
> **设计原则**: 单一职责原则 (Single Responsibility Principle)

## 📊 方案架构

### 核心设计理念

本部署方案将复杂的部署流程分解为**6个独立的职责模块**，每个模块只负责一件事：

```
部署流程
├── 1️⃣ 环境准备器 (Environment Preparator)
│   └── 职责：检查和安装云服务器基础环境
│
├── 2️⃣ 代码传输器 (Code Uploader)
│   └── 职责：将项目代码上传到云服务器
│
├── 3️⃣ 配置管理器 (Config Manager)
│   └── 职责：配置环境变量和数据库连接
│
├── 4️⃣ 服务部署器 (Service Deployer)
│   └── 职责：构建Docker镜像并启动服务
│
├── 5️⃣ 部署验证器 (Deployment Validator)
│   └── 职责：验证部署是否成功，测试核心功能
│
└── 6️⃣ 运维监控器 (Operations Monitor)
    └── 职责：配置日志、监控和自动重启
```

## 🚀 使用方式

### 方式1: 一键部署（推荐）

```bash
# 1. 配置服务器信息
cp deploy/upload_config.env.template deploy/upload_config.env
vim deploy/upload_config.env

# 2. 一键部署
./deploy/deploy_all.sh
```

**优点**：
- ✅ 简单快捷
- ✅ 自动化程度高
- ✅ 适合首次部署

### 方式2: 分步部署

```bash
# 按需执行单个模块
./deploy/1_prepare_environment.sh
./deploy/2_upload_code.sh
./deploy/3_configure_environment.sh
./deploy/4_deploy_services.sh
./deploy/5_validate_deployment.sh
./deploy/6_setup_monitoring.sh
```

**优点**：
- ✅ 灵活可控
- ✅ 便于调试
- ✅ 适合问题排查

### 方式3: Git + Remote-SSH（推荐）

```bash
# 在服务器上直接操作
git clone https://github.com/your-repo/timao-douyin-live-manager.git
cd timao-douyin-live-manager
# 配置环境
cd server && cp env.production.template .env && vim .env
# 部署
cd .. && docker-compose -f docker-compose.full.yml up -d --build
```

**优点**：
- ✅ 利用Git版本控制
- ✅ 不需要传输文件
- ✅ 适合持续部署

## 🪟 Windows环境支持

### Cursor连接云服务器

Cursor可以通过以下方式连接到云服务器：

#### 1. Remote-SSH扩展（推荐）⭐

```
1. 安装 "Remote - SSH" 扩展
2. 配置 ~/.ssh/config
3. 连接到服务器
4. 直接在服务器上开发和部署
```

**优势**：
- ✅ 无缝开发体验
- ✅ 直接在服务器上操作
- ✅ 不需要文件传输

#### 2. WSL (Windows Subsystem for Linux)

```bash
# 1. 安装WSL
wsl --install

# 2. 在WSL中执行部署脚本
wsl
cd /mnt/d/gsxm/timao-douyin-live-manager
./deploy/deploy_all.sh
```

#### 3. Git Bash

```bash
# 在项目根目录右键 -> Git Bash Here
./deploy/deploy_all.sh
```

详细说明见：[WINDOWS_DEPLOY.md](WINDOWS_DEPLOY.md)

## 🔐 安全建议

### SSH安全

```bash
# 1. 使用SSH密钥而非密码
ssh-keygen -t rsa -b 4096

# 2. 禁用密码登录（服务器上）
sudo vim /etc/ssh/sshd_config
# PasswordAuthentication no

# 3. 更改默认SSH端口
# Port 22 -> Port 2222
```

### 环境变量安全

```bash
# 1. 不要提交 .env 文件到Git
echo ".env" >> .gitignore

# 2. 使用强密码
# SECRET_KEY应使用随机生成的长密钥

# 3. 限制CORS源
# CORS_ORIGINS=* -> CORS_ORIGINS=https://yourdomain.com
```

### 防火墙配置

```bash
# Ubuntu/Debian
sudo ufw enable
sudo ufw allow 22/tcp    # SSH
sudo ufw allow 80/tcp    # HTTP
sudo ufw allow 443/tcp   # HTTPS
sudo ufw allow 11111/tcp # Backend
```

## 🎓 单一职责原则体现

### 职责清晰

每个脚本模块都有明确的单一职责：

- **环境准备器** - 只检查环境，不涉及部署
- **代码传输器** - 只传输代码，不涉及配置
- **配置管理器** - 只管理配置，不涉及服务
- **服务部署器** - 只部署服务，不涉及验证
- **部署验证器** - 只验证部署，不涉及修复
- **运维监控器** - 只配置监控，不涉及部署

### 易于维护

- ✅ 每个模块独立，修改一个不影响其他
- ✅ 问题定位快速，职责明确
- ✅ 可以单独测试每个模块
- ✅ 易于扩展，添加新模块不影响旧模块

### 灵活组合

- ✅ 可以单独执行任何模块
- ✅ 可以跳过某些模块
- ✅ 可以自定义执行顺序
- ✅ 可以在不同项目中复用
