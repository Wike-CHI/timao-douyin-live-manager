# 时间戳精确化更新 - 触发点定位优化

## 📋 更新概述

基于用户需求"**精确定位触发消费的话题钩子**"，对增强版复盘系统进行了时间精确性优化。

## 🎯 核心改进

### 1. 时间窗设计：30秒精确定位
- **窗口大小**: 30秒（balance between precision & data volume）
- **分割策略**: 按中文句子（。！？）智能分割，避免截断句子
- **匹配容差**: ±30秒匹配窗口，捕捉事件与话术的因果关系

### 2. Prompt 优化重点

#### ✅ 强制时间引用规则
```markdown
### 1. 精确时间定位（最重要）
- ✅ 必须引用"触发点证据"中的具体时间（如：12:30-13:00）
- ✅ 必须引用证据中的主播话术原文
- ❌ 不要猜测或编造时间点
```

#### ✅ 置信度评估标准
```markdown
- 0.9-1.0: 多个事件在同一话术后触发，模式明确
- 0.7-0.89: 事件与话术有明显关联
- 0.5-0.69: 关联较弱，可能是巧合
- <0.5: 不要输出
```

#### ✅ JSON Schema 示例优化
将模糊的"第X分Y秒"改为精确格式"12:30 - 13:00"：

```json
{
  "trigger_analysis": {
    "consumption_triggers": [
      {
        "topic": "触发消费的话题/台词",
        "evidence": {
          "time": "12:30 - 13:00",  // 精确时间窗
          "transcript_quote": "主播说的具体话术（从触发点证据中原文引用）",
          "gifts": [{"user": "用户昵称", "gift": "小心心", "count": 5}],
          "related_comments": ["好美啊", "给姐姐刷礼物"]
        },
        "confidence": 0.85,
        "analysis": "为什么这个话术触发了消费？分析情绪、语气、时机等因素"
      }
    ]
  }
}
```

## 🎓 为什么30秒是最佳窗口？

### 对比分析
| 窗口大小 | 优点 | 缺点 |
|---------|------|------|
| 10秒 | 极精确 | 窗口过多（3600个/小时），AI难以处理 |
| **30秒** | **精确度足够，数据量可控（120个/小时）** | **推荐** ✅ |
| 60秒 | 数据量少 | 精确度下降，可能包含多个话题 |
| 句子级 | 语义完整 | 时间不均匀，匹配复杂 |

### 实践依据
1. **观众反应时间**: 观众听到话术后1-10秒内做出反应（送礼/关注/评论）
2. **±30秒容差**: 覆盖事件前15秒和后15秒，足以捕捉因果关系
3. **AI token限制**: Gemini 2.5 Flash max_tokens=6000，30秒窗口可展示100+个窗口的完整分析
