# 前端环境变量配置模板

**请复制以下内容创建 `electron/renderer/.env` 文件**

```env
# ============================================
# 提猫直播助手 - 前端开发环境变量
# ============================================
# 审查人: 叶维哲
# 遵循原则: KISS、单一职责、希克定律

# ------------------------------------------
# 🌐 开发服务器配置 (必需)
# ------------------------------------------
# Vite开发服务器端口
VITE_PORT=10050

# 开发服务器主机
VITE_HOST=127.0.0.1

# ------------------------------------------
# 🔗 后端服务地址 (必需)
# ------------------------------------------
# FastAPI主服务地址
VITE_FASTAPI_URL=http://127.0.0.1:11111

# StreamCap服务地址 (已集成到主服务)
VITE_STREAMCAP_URL=http://127.0.0.1:11111

# Douyin服务地址 (已集成到主服务)
VITE_DOUYIN_URL=http://127.0.0.1:11111

# ------------------------------------------
# 🛠️ 开发配置 (可选)
# ------------------------------------------
# 启用HMR热更新
VITE_HMR=true

# 启用源码映射
VITE_SOURCEMAP=true
```

## 🚀 快速开始

### 1. 创建配置文件

```bash
# Windows PowerShell
cd electron/renderer
Copy-Item ENV_TEMPLATE.md .env
# 然后编辑 .env 文件

# 或者手动创建
New-Item -Path . -Name ".env" -ItemType "file"
# 复制上面的内容到 .env
```

### 2. 验证配置

**必须配置的3个选项** (遵循希克定律):

1. ✅ `VITE_PORT=10050` (前端开发端口)
2. ✅ `VITE_HOST=127.0.0.1` (本地访问)
3. ✅ `VITE_FASTAPI_URL=http://127.0.0.1:11111` (后端地址)

### 3. 启动前端

```bash
npm run dev
```

应该看到:
```
✅ 开发服务器已启动 http://127.0.0.1:10050
```

## 📋 配置说明

### 前端端口 (VITE_PORT)

- **默认**: 10050
- **原因**: 与后端端口(11111)区分，避免冲突
- **修改**: 如果10050被占用，可改为10051、10052等

### 后端地址 (VITE_FASTAPI_URL)

- **默认**: http://127.0.0.1:11111
- **说明**: 必须与后端启动端口一致
- **修改**: 如果后端使用其他端口，这里也要同步修改

### 服务统一

所有后端服务已统一到11111端口:
- ✅ FastAPI主服务: 11111
- ✅ StreamCap: 11111 (已集成)
- ✅ Douyin: 11111 (已集成)

## 🔧 常见问题

### Q: 前端启动失败?

**A**: 检查以下几点:

1. ✅ `.env` 文件是否存在于 `electron/renderer/` 目录
2. ✅ 端口10050是否被占用
3. ✅ `npm install` 是否执行过

### Q: 无法连接后端?

**A**: 确保:

1. 后端服务已启动 (http://127.0.0.1:11111)
2. `VITE_FASTAPI_URL` 地址正确
3. CORS配置允许前端地址

### Q: 端口被占用?

```bash
# 临时使用其他端口
VITE_PORT=10014 npm run dev

# 或修改 .env 文件
VITE_PORT=10014
```

## ✅ 遵循的设计原则

1. **KISS原则**: 只配置3个必需项，其他有默认值
2. **单一职责原则**: 前端配置只在 `electron/renderer/.env`
3. **希克定律**: 最重要的3个配置优先，简化决策

**最重要的3个配置**:
1. VITE_PORT (前端端口)
2. VITE_HOST (主机地址)
3. VITE_FASTAPI_URL (后端地址)
