# 内存优化测试指南

审查人：叶维哲

## 测试目的

验证内存优化方案的效果，确保：
1. 服务能够长时间稳定运行（2-3小时）
2. 内存使用保持在合理范围（2.8-3.8GB）
3. 不会因内存溢出触发PM2自动重启
4. 内存清理机制有效工作

## 前置准备

### 1. 更新代码并重启服务

```bash
cd /www/wwwroot/wwwroot/timao-douyin-live-manager

# 重启PM2服务（应用新配置）

# 查看服务状态
pm2 status
```

### 2. 验证服务启动成功

检查日志中是否有以下启动信息：
- ✅ 内存监控服务已启动
- ✅ FastAPI主服务启动
- ✅ SenseVoice服务初始化成功

## 测试场景

### 场景一：长时间运行测试（推荐）

**目标**：验证服务能够稳定运行2-3小时，内存不会持续增长

**操作步骤**：

1. 启动测试监控脚本：
```bash
cd /www/wwwroot/wwwroot/timao-douyin-live-manager/tests

# 运行2小时测试，每分钟采样一次
python3 memory_test.py --duration 120 --interval 60

# 或运行3小时测试
python3 memory_test.py --duration 180 --interval 60
```

2. 在Electron应用中执行实际操作：
   - 启动直播间音频转写
   - 启动AI实时分析
   - 启动直播录制
   - 保持服务运行2-3小时

3. 观察测试结果：
   - 查看实时日志输出
   - 关注内存趋势分析
   - 检查是否有警告或错误

**预期结果**：
- ✅ 内存使用稳定在 2.8-3.8GB
- ✅ 内存波动 < 500MB
- ✅ PM2重启次数 = 0
- ✅ 内存趋势稳定或下降，无持续增长

### 场景二：高负载测试

**目标**：模拟高人气直播间，验证内存清理机制

**操作步骤**：

1. 连接高人气直播间（大量评论、弹幕）

2. 同时启动所有服务：
   - 音频转写
   - AI实时分析
   - 直播录制

3. 运行30-60分钟，观察：
   - PM2日志中的内存清理信息
   - 服务是否保持响应
   - 转写延迟是否在可接受范围（< 2秒）

**预期结果**：
- ✅ 服务保持响应，无卡顿
- ✅ 日志中有内存清理记录（如：🧹 AI分析服务内存清理）
- ✅ 内存峰值 < 4.5GB
- ✅ 转写延迟 < 2秒

### 场景三：服务重启测试

**目标**：验证服务启停过程中内存正确释放

**操作步骤**：

1. 多次启动和停止服务（至少5次）：
```bash
# 在Electron应用中执行
启动转写 -> 停止转写 -> 启动转写 -> 停止转写 ...
```

2. 每次停止后检查内存是否释放：
```bash
# 使用测试脚本查看当前内存
python3 -c "
import requests
r = requests.get('http://localhost:8081/api/model/memory-status')
print(r.json())
"
```

**预期结果**：
- ✅ 每次停止后内存显著下降
- ✅ 无内存泄漏（多次启停后内存不累积）
- ✅ 服务启停正常，无错误

## 问题排查

### 问题1：内存持续增长

**症状**：内存从3GB逐渐增长到4GB+

**排查步骤**：
1. 检查是否有未释放的数据累积
2. 查看日志中是否有内存清理记录
3. 检查是否有大量用户数据未清理

**解决方案**：
- 确认内存清理逻辑是否正确执行
- 检查数据上限设置是否合理
- 考虑降低数据保留量

### 问题2：频繁触发PM2重启

**症状**：PM2日志中频繁出现重启记录

**排查步骤**：
1. 查看重启时的内存使用情况
2. 检查是否有内存泄漏
3. 分析内存增长速度

**解决方案**：
- 如果内存确实超标，检查是否有服务异常
- 如果内存正常但仍重启，调整max_memory_restart阈值
- 检查PM2配置是否正确

### 问题3：内存清理不生效

**症状**：日志中无内存清理记录，内存持续增长

**排查步骤**：
1. 检查代码中的内存清理逻辑是否正确
2. 验证gc.collect()是否被调用
3. 查看数据量是否达到清理阈值

**解决方案**：
- 确认代码修改已生效（重启服务）
- 降低清理阈值，提高清理频率
- 添加更多日志定位问题

## 验收标准

内存优化方案通过验收的标准：

- ✅ **稳定性**：连续运行2小时无PM2重启
- ✅ **内存范围**：平均内存 2.8-3.8GB，峰值 < 4.3GB
- ✅ **内存趋势**：内存稳定或下降，无持续增长
- ✅ **清理机制**：日志中有内存清理记录，清理后内存下降
- ✅ **性能表现**：转写延迟 < 2秒，服务响应正常
- ✅ **资源释放**：服务停止后内存正确释放

满足以上所有标准，则内存优化方案验收通过。
