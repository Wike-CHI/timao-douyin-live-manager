# 提猫直播助手移动端 - 用户体验分析

## 1. 用户体验分析

### 核心用户角色
- **主播**：需要实时查看弹幕、接收AI话术建议、查看直播数据分析
- **助手**：协助主播管理直播，监控数据，生成复盘报告

### 核心使用场景
1. **登录场景**：快速登录，支持记住密码
2. **直播监控场景**：实时查看弹幕和礼物，通过悬浮窗接收AI分析和话术建议
3. **数据复盘场景**：查看历史直播数据，手动生成完整复盘报告
4. **订阅管理场景**：查看套餐，支付升级
5. **双模式切换场景**：在全功能模式和直播模式间切换以适应不同使用需求

### 关键交互需求
- **实时性**：直播监控需要实时更新，WebSocket数据推送
- **便捷性**：悬浮窗快速显示AI建议，一键复制话术
- **可视化**：数据统计使用图表展示，直观易懂
- **易用性**：界面简洁，操作流程顺畅
- **灵活性**：支持双模式切换，适应不同使用场景

## 3. UI设计规范

### 设计风格
- **设计语言**：遵循 iOS Human Interface Guidelines 和 Material Design
- **主色调**：#667eea（紫色品牌色）
- **辅助色**：#764ba2（深紫色）
- **背景**：浅色模式 #ffffff，深色模式 #121212

### 组件规范
- **卡片样式**：圆角 12px，阴影层级分明
- **按钮样式**：主要按钮使用品牌紫色，次要按钮使用灰色边框
- **悬浮窗**：半透明背景 rgba(102, 126, 234, 0.95)，圆角 12px
- **字体**：标题 20-24px，正文 14-16px，辅助文字 12px

### 双模式设计规范
- **全功能模式**：
  - 采用四宫格布局，信息展示全面
  - 支持详细配置和设置
  - 适合直播准备和数据分析场景
- **直播模式**：
  - 悬浮窗设计，最小化遮挡
  - 信息轮播显示，突出重点
  - 支持拖拽和边缘吸附
  - 适合正式直播场景

## 4. 核心页面设计

### 页面1：登录页面
- 品牌Logo和应用名称
- 用户名/邮箱输入框
- 密码输入框（支持显示/隐藏）
- "记住我"复选框
- 登录按钮
- 注册链接

### 页面2：直播监控页面
- 顶部：直播状态指示器、观众数统计
- 中部：实时弹幕列表（滚动更新）
- 底部：控制按钮（开始/暂停/停止）
- 悬浮窗：AI分析、话术建议（可拖拽）

### 页面3：双模式控制台
- **全功能模式**：
  - 顶部：直播地址输入、控制按钮、模式切换
  - 四宫格布局：
    - 左上：实时语音转写区域
    - 右上：智能话术建议区域
    - 左下：直播间互动分析区域
    - 右下：主播画像与氛围区域
  - 底部：状态栏（在线人数、礼物价值、互动率等）
- **直播模式**：
  - 悬浮窗设计：
    - 标题栏（可拖拽区域）
    - 主内容区（轮播显示AI分析、话术生成、直播间氛围）
    - 操作栏（复制、切换信息类型、更多操作）
  - 支持展开/收起状态
  - 支持拖拽到任意位置
  - 支持边缘自动吸附

### 页面4：数据统计列表页面
- 顶部：日期筛选器
- 列表：直播记录卡片（时间、时长、数据概览、复盘状态）
- 刷新和加载更多

### 页面5：复盘详情页面
- 直播基本信息
- 数据统计图表
- AI分析内容
- 深度分析和建议
- 生成复盘按钮

## 5. 后续开发建议

### 5.1 原型验证
- 邀请目标用户测试双模式切换的流畅性
- 验证悬浮窗的易用性和信息展示效果
- 收集关于四宫格布局的信息密度反馈

### 5.2 技术实现要点
- **双模式状态管理**：需要设计合理的状态管理机制
- **悬浮窗实现**：React Native中需要使用PanResponder实现拖拽
- **信息轮播**：需要实现优先级队列和轮播机制
- **WebSocket通信**：实时数据推送需要稳定可靠的连接

### 5.3 交互优化方向
- 增加模式切换的动画效果
- 优化悬浮窗的边缘吸附算法
- 添加自定义信息轮播顺序的功能
- 支持悬浮窗主题定制
