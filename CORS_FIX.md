# CORS问题修复指南

## ✅ 已修复：CORS配置已更新

### 修复内容
1. **开发环境**：允许所有来源 (`allow_origins=["*"]`)
2. **生产环境**：限制为指定域名列表

### 重要：请重启后端服务！

**CORS配置只有在重启后端服务后才会生效。**

#### 重启方法：

**如果使用 uvicorn 启动：**
```powershell
# 停止当前服务 (Ctrl+C)
# 然后重新启动
uvicorn server.app.main:app --reload --port 9019
```

**如果使用其他方式启动：**
- 停止当前服务
- 重新启动服务

### 验证CORS是否生效

1. **启动后端服务**
2. **打开浏览器开发者工具（F12）**
3. **访问管理后台**: http://localhost:3000
4. **尝试登录**
5. **查看 Network 标签**：
   - 如果看到 `OPTIONS /api/auth/login` 请求返回 200，说明CORS配置生效
   - 检查响应头中是否有 `Access-Control-Allow-Origin: *`

### 如果仍然有问题

#### 方法1：检查后端是否运行
```powershell
# 检查端口9019是否被占用
netstat -ano | findstr :9019
```

#### 方法2：检查浏览器控制台
- 打开开发者工具 (F12)
- 查看 Console 标签的错误信息
- 查看 Network 标签的请求详情

#### 方法3：临时测试（不推荐用于生产）
如果急需测试，可以暂时在浏览器中禁用CORS检查（仅用于开发）：
- Chrome: 使用 `--disable-web-security` 标志启动（仅用于测试）
