## 🚀 一键部署

### 步骤1：启动云端服务

```bash
cd /www/wwwroot/wwwroot/timao-douyin-live-manager

# 执行云端部署脚本
./scripts/deploy_cloud.sh
```

**脚本自动完成**：
- ✅ 激活Python虚拟环境
- ✅ 安装/更新依赖
- ✅ 启动PM2云端服务
- ✅ 验证服务健康状态

### 步骤2：配置Nginx反向代理

```bash
# 执行Nginx配置脚本
sudo ./scripts/setup_nginx_cloud.sh
```

**脚本自动完成**：
- ✅ 检测Nginx类型
- ✅ 备份旧配置
- ✅ 创建新配置（代理到15000端口）
- ✅ 测试并重载Nginx
- ✅ 验证连接

## 🐛 常见问题

### 问题1：服务启动失败

```bash
# 查看错误日志
pm2 logs timao-cloud --err --lines 50

# 常见原因：
# - 端口15000被占用
# - Python依赖缺失
# - 数据库连接失败
```

**解决方案**：
```bash
# 检查端口占用
netstat -tlnp | grep 15000

# 停止旧服务
pm2 stop timao-backend
pm2 delete timao-backend

# 重新安装依赖
pip install -r requirements.txt

# 重启云端服务
pm2 restart timao-cloud
```

### 问题2：Nginx配置失败

```bash
# 查看Nginx错误
sudo nginx -t

# 常见原因：
# - 配置文件语法错误
# - 端口冲突
# - 权限问题
```

**解决方案**：
```bash
# 恢复备份
sudo cp /www/server/panel/vhost/nginx/timao-cloud.conf.backup.* \
        /www/server/panel/vhost/nginx/timao-cloud.conf

# 重新配置
sudo ./scripts/setup_nginx_cloud.sh
```

### 问题3：客户端无法连接

```bash
# 检查防火墙
sudo ufw status
sudo ufw allow 80/tcp

# 检查Nginx状态
sudo systemctl status nginx

# 检查云端服务
curl http://localhost:15000/health
```

**解决方案**：
```bash
# 确保服务都在运行
pm2 status timao-cloud
sudo systemctl status nginx

# 检查客户端配置
cat admin-dashboard/.env.production
# 确保 VITE_API_BASE_URL 正确
```
