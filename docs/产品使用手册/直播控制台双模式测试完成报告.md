## 🎓 使用说明

### 开发流程

1. **开发新功能** → 先看设计文档
2. **实现功能** → 参考测试用例了解需求
3. **运行测试** → 验证实现正确性
4. **调试问题** → 使用pytest的详细输出

### 测试驱动开发（TDD）

```bash
# 1. 运行测试（应该失败，因为功能未实现）
pytest tests/test_new_feature.py -v

# 2. 实现功能
# ... 编写代码 ...

# 3. 再次运行测试（应该通过）
pytest tests/test_new_feature.py -v

# 4. 重构代码（测试保证不破坏功能）
```

### 调试技巧

```bash
# 详细输出
pytest tests/test_xxx.py -vv

# 只运行失败的测试
pytest --lf

# 进入调试模式
pytest tests/test_xxx.py --pdb

# 查看print输出
pytest tests/test_xxx.py -s
```
