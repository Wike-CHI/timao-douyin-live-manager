## 📋 概述

本指南用于部署**云端服务**到生产服务器，包含：
- ✅ **用户系统**：注册、登录、用户信息管理
- ✅ **订阅系统**：套餐管理、订阅状态
- ✅ **支付系统**：支付订单、支付回调（待域名备案）
- ✅ **积分系统**：积分消耗、充值、统计

**服务特点**：
- 轻量级：内存占用 < 600MB
- 必须联网：前端必须连接云端才能使用
- 端口：15000

### 8. 配置Nginx反向代理

⚠️ **重要**：客户端必须通过Nginx连接到云端服务，不能直接连接15000端口！

#### 方法1：自动配置（推荐）

```bash
# 赋予执行权限
chmod +x scripts/setup_nginx_cloud.sh

# 运行自动配置脚本
sudo ./scripts/setup_nginx_cloud.sh
```

脚本会自动完成：
- ✅ 检测Nginx类型（宝塔面板/标准Nginx）
- ✅ 备份旧配置
- ✅ 创建新配置（代理到15000端口）
- ✅ 测试并重载Nginx
- ✅ 验证云端服务连接

#### 方法2：宝塔面板手动配置

1. 登录宝塔面板
2. 网站 → 选择站点（或创建新站点）
3. 设置 → 配置文件
4. 复制项目根目录 `nginx-cloud.conf` 的内容
5. 替换整个 `server` 块
6. 保存并重启Nginx

#### 方法3：标准Nginx手动配置

```bash
# 1. 复制配置文件
sudo cp nginx-cloud.conf /etc/nginx/sites-available/timao-cloud.conf

# 2. 修改服务器IP/域名
sudo nano /etc/nginx/sites-available/timao-cloud.conf
# 修改: server_name 你的IP或域名;

# 3. 创建软链接
sudo ln -s /etc/nginx/sites-available/timao-cloud.conf /etc/nginx/sites-enabled/

# 4. 测试配置
sudo nginx -t

# 5. 重载Nginx
sudo nginx -s reload
```

#### 关键配置说明

```nginx
server {
    listen 80;
    server_name 129.211.218.135;  # 修改为你的IP/域名
    
    # 所有请求代理到云端服务（15000端口）
    location / {
        proxy_pass http://127.0.0.1:15000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## ⚠️ 注意事项

### 1. 必须联网运行

- ✅ 云端服务需要连接MySQL RDS
- ✅ 前端Electron应用必须连接云端API
- ❌ 不支持离线模式（与本地服务不同）

### 2. 支付配置

**当前状态**：等待域名备案

**完成备案后需配置**：
```bash
# 支付宝配置
ALIPAY_APP_ID=你的AppID
ALIPAY_PRIVATE_KEY=你的私钥
ALIPAY_PUBLIC_KEY=支付宝公钥
ALIPAY_NOTIFY_URL=https://api.timao.com/api/payment/alipay/notify
ALIPAY_RETURN_URL=https://api.timao.com/api/payment/alipay/return

# 微信支付配置
WECHAT_APP_ID=你的AppID
WECHAT_MCH_ID=商户号
WECHAT_API_KEY=API密钥
WECHAT_NOTIFY_URL=https://api.timao.com/api/payment/wechat/notify
```

### 3. 安全建议

- 修改 `SECRET_KEY` 和 `ENCRYPTION_KEY`（生产环境）
- 限制 `CORS_ORIGINS` 为具体域名（不使用 `*`）
- 使用HTTPS（配置SSL证书）
- 定期备份数据库
- 启用防火墙，只开放必要端口

### 4. 性能优化

- Redis缓存：建议启用Redis减少数据库查询
- 数据库连接池：已配置20个连接
- 内存限制：600MB，超过自动重启

## 🐛 故障排查

### 问题1：服务启动失败

```bash
# 查看错误日志
pm2 logs timao-cloud --err --lines 50

# 常见原因：
# - 端口15000被占用 → 检查：netstat -tlnp | grep 15000
# - Python依赖缺失 → 重新安装：pip install -r requirements.txt
# - 数据库连接失败 → 检查.env配置和RDS安全组
```

### 问题2：内存占用过高

```bash
# 查看内存占用
pm2 monit

# 解决方案：
# 1. 降低 DB_POOL_SIZE（当前20，可减到10）
# 2. 启用Redis缓存减少数据库查询
# 3. 升级服务器配置
```

### 问题3：前端无法连接

```bash
# 检查服务状态
curl http://localhost:15000/health

# 检查防火墙
sudo ufw status
sudo ufw allow 15000/tcp

# 检查Nginx配置
sudo nginx -t
```

### 问题4：数据库连接池耗尽

```bash
# 查看MySQL连接数
mysql -h rm-bp1sqxf05yom2hwdhko.mysql.rds.aliyuncs.com \
      -u timao -p \
      -e "SHOW PROCESSLIST"

# 解决方案：
# 1. 增加 DB_POOL_SIZE（当前20）
# 2. 减少 DB_POOL_TIMEOUT（当前30秒）
# 3. 检查慢查询并优化
```
