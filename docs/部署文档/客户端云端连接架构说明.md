## 📋 架构概述

根据《云端本地混合部署需求完成报告》，系统采用**云端+本地混合架构**：

```
┌─────────────────────────────────────────────────────────────┐
│                     客户端 (Electron应用)                      │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  前端界面 (Vue 3 + TypeScript)                         │  │
│  └──────────────────────────────────────────────────────┘  │
│           │                              │                   │
│           │ 连接云端                     │ 本地服务            │
│           ▼                              ▼                   │
│  ┌─────────────────┐          ┌─────────────────┐         │
│  │  云端API调用      │          │  本地服务调用     │         │
│  │  (必须联网)       │          │  (离线可用)       │         │
│  └─────────────────┘          └─────────────────┘         │
└─────────────────────────────────────────────────────────────┘
           │                              │
           │ HTTP/HTTPS                   │ HTTP (localhost)
           │ 端口80/443                   │ 端口16000
           ▼                              ▼
┌─────────────────────┐        ┌─────────────────────┐
│   云端服务器          │        │   本地Python服务       │
│   (Nginx反向代理)    │        │   (FastAPI)           │
│   → timao-cloud     │        │                       │
│     端口15000       │        │  • 音频转写            │
│                     │        │  • AI分析              │
│  • 用户登录          │        │  • 弹幕拉取            │
│  • 订阅管理          │        │  • 会话管理            │
│  • 支付处理          │        └─────────────────────┘
│  • 积分统计          │
└─────────────────────┘
```

## 🔌 连接说明

### 云端连接（必须联网）

**目的**：检查用户状态、积分、支付

**连接地址**：
- 开发环境：`http://localhost:15000`
- 生产环境：`http://服务器IP:80` 或 `https://api.timao.com`（待域名备案）

**API端点**：
```
✅ 用户认证
   POST   /api/auth/register           注册
   POST   /api/auth/login              登录
   POST   /api/auth/refresh            刷新Token
   GET    /api/auth/me                 当前用户信息

✅ 用户资料
   GET    /profile/basic               获取基本信息
   PUT    /profile/basic               更新基本信息
   POST   /profile/avatar              上传头像

✅ 订阅系统
   GET    /api/subscription/plans      套餐列表
   GET    /api/subscription/current    当前订阅
   POST   /api/subscription/create-payment    创建支付
   GET    /api/subscription/usage-stats       使用统计

✅ 支付系统
   POST   /api/payment/alipay/notify   支付宝回调
   POST   /api/payment/wechat/notify   微信回调
```

### 本地连接（离线可用）

**目的**：执行核心直播功能

**连接地址**：`http://localhost:16000`

**功能**：
```
✅ 音频转写
   POST   /api/transcribe/stream/start
   POST   /api/transcribe/stream/stop
   GET    /api/transcribe/stream/status

✅ AI分析
   POST   /api/ai/live/analyze
   GET    /api/ai/live/status

✅ 弹幕拉取

   GET    /api/douyin/web/status
```

## 🔧 Nginx反向代理配置

### 1. 架构变化

**原架构**（已废弃）：
```
客户端 → Nginx:80 → timao-backend:11111（所有服务）
```

**新架构**（当前）：
```
客户端 
  ├─ 云端API → Nginx:80 → timao-cloud:15000（用户/订阅/支付）
  └─ 本地服务 → localhost:16000（音频/AI/弹幕）
```

### 2. 配置文件

**文件位置**：
- 宝塔面板：`/www/server/panel/vhost/nginx/timao-cloud.conf`
- 标准Nginx：`/etc/nginx/sites-available/timao-cloud.conf`
- 项目根目录：`nginx-cloud.conf`（模板）

**关键配置**：
```nginx
server {
    listen 80;
    server_name 服务器IP;
    
    # 云端服务代理（端口15000）
    location / {
        proxy_pass http://127.0.0.1:15000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

### 3. 自动配置脚本

```bash
# 一键配置Nginx
sudo ./scripts/setup_nginx_cloud.sh
```

**脚本自动完成**：
1. ✅ 检测Nginx类型（宝塔/标准）
2. ✅ 备份旧配置
3. ✅ 创建新配置（代理到15000端口）
4. ✅ 测试配置
5. ✅ 重载Nginx
6. ✅ 验证部署

## 🔐 客户端配置

### 前端环境变量

**文件**：`admin-dashboard/.env.production`

```env
# 云端API地址（修改为实际服务器IP）
VITE_API_BASE_URL=http://服务器IP

# 必须联网
VITE_REQUIRE_ONLINE=true

# 离线模式（禁用）
VITE_OFFLINE_MODE=false
```

### Electron配置

**文件**：`electron/main/config.ts`

```typescript
// 云端API配置
const CLOUD_API_BASE = process.env.VITE_API_BASE_URL || 'http://localhost:15000';

// 本地服务配置
const LOCAL_SERVICE_PORT = 16000;
const LOCAL_API_BASE = `http://localhost:${LOCAL_SERVICE_PORT}`;

// API调用分发
export function getApiUrl(endpoint: string): string {
  // 云端端点
  if (endpoint.startsWith('/api/auth/') ||
      endpoint.startsWith('/profile/') ||
      endpoint.startsWith('/api/subscription/') ||
      endpoint.startsWith('/api/payment/')) {
    return CLOUD_API_BASE + endpoint;
  }
  
  // 本地端点
  return LOCAL_API_BASE + endpoint;
}
```

## ⚠️ 注意事项

### 1. 端口说明

| 服务 | 端口 | 用途 | 公开 |
|------|-----|------|------|
| Nginx | 80/443 | 反向代理 | ✅ 公网访问 |
| timao-cloud | 15000 | 云端服务 | ❌ 仅内网 |
| 本地服务 | 16000 | 本地功能 | ❌ 仅本机 |

### 2. 防火墙配置

```bash
# 开放80端口（HTTP）
sudo ufw allow 80/tcp

# 开放443端口（HTTPS，待启用）
sudo ufw allow 443/tcp

# 不要开放15000端口（只允许Nginx访问）
```

### 3. 必须联网运行

- ✅ 用户**必须登录**才能使用应用
- ✅ 启动时检查云端连接
- ✅ 登录失败无法使用核心功能
- ❌ 不支持离线模式（与本地服务不同）

### 4. 与旧架构的区别

**旧架构**（timao-backend，端口11111）：
- ❌ 包含所有功能（音频、AI、弹幕、用户、支付）
- ❌ 内存占用高（4.5GB+）
- ❌ 客户端必须连接服务器才能使用所有功能

**新架构**（timao-cloud + 本地服务）：
- ✅ 云端仅用户/订阅/支付（轻量级，<600MB）
- ✅ 本地执行音频/AI/弹幕（重度功能）
- ✅ 降低服务器压力
- ✅ 提升响应速度

## 🐛 故障排查

### 问题1：客户端无法连接云端

**现象**：登录时报错"网络连接失败"

**排查步骤**：
```bash
# 1. 检查云端服务
pm2 status timao-cloud

# 2. 检查健康端点
curl http://localhost:15000/health

# 3. 检查Nginx
nginx -t
curl http://localhost/health

# 4. 检查防火墙
sudo ufw status
```

### 问题2：Nginx配置错误

**现象**：`nginx -t` 失败

**解决方案**：
```bash
# 1. 查看错误日志
tail -f /www/wwwlogs/timao-cloud-error.log

# 2. 检查配置文件
cat /www/server/panel/vhost/nginx/timao-cloud.conf

# 3. 恢复备份
cp /www/server/panel/vhost/nginx/timao-cloud.conf.backup.* \
   /www/server/panel/vhost/nginx/timao-cloud.conf

# 4. 重新配置
sudo ./scripts/setup_nginx_cloud.sh
```

### 问题3：端口冲突

**现象**：15000端口被占用

**解决方案**：
```bash
# 查看占用进程
netstat -tlnp | grep 15000

# 停止旧服务
pm2 stop timao-backend
pm2 delete timao-backend

# 启动新服务
pm2 start ecosystem.cloud.config.js
```
