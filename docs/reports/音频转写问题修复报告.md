## 🔍 问题分析

### 问题1：WebSocket CSP 阻止

**原因：**
- CSP 配置中缺少 `ws://129.211.218.135` 规则
- 只有 `http://129.211.218.135`，没有 WebSocket 协议

**影响：**
- ❌ 前端无法连接音频 WebSocket
- ❌ 转写结果无法实时推送到前端
- ❌ 语音转写卡片无法显示内容

### 问题2：Window Size 错误

**原因：**
- librosa STFT 处理时，某些极短音频片段导致窗口大小计算错误
- FunASR 内部的音频预处理对极短音频支持不完善

**影响：**
- ⚠️ 部分音频片段转写失败
- ⚠️ 错误日志频繁输出
- ✅ 不影响整体功能（大部分音频正常）

---

## ✅ 修复方案

### 修复1：CSP 配置

#### 修改文件：`electron/renderer/index.html`

```diff
- connect-src 'self' ws://127.0.0.1:* ws://localhost:* http://129.211.218.135 http://129.211.218.135:* https:
+ connect-src 'self' ws://127.0.0.1:* ws://localhost:* ws://129.211.218.135 ws://129.211.218.135:* wss://129.211.218.135:* http://127.0.0.1:* http://localhost:* http://129.211.218.135 http://129.211.218.135:* https:
```

**添加的规则：**
- ✅ `ws://129.211.218.135` - HTTP WebSocket
- ✅ `ws://129.211.218.135:*` - HTTP WebSocket 任意端口
- ✅ `wss://129.211.218.135:*` - HTTPS WebSocket 任意端口

#### 修改文件：`electron/renderer/vite.config.ts`

```diff
- "connect-src 'self' ws://127.0.0.1:* ws://localhost:* http://127.0.0.1:* http://localhost:* http://129.211.218.135 http://129.211.218.135:* https:"
+ "connect-src 'self' ws://127.0.0.1:* ws://localhost:* ws://129.211.218.135 ws://129.211.218.135:* wss://129.211.218.135:* http://127.0.0.1:* http://localhost:* http://129.211.218.135 http://129.211.218.135:* https:"
```

### 修复2：Window Size 错误处理

#### 修改文件：`server/modules/ast/sensevoice_service.py`

**策略：** 优雅处理，而不是跳过

```python
# 在 model.generate() 的异常处理中添加
except ValueError as exc:
    error_msg = str(exc)
    if "window size" in error_msg:
        # 音频处理异常，返回空结果（静默处理）
        return {
            "success": True,
            "type": "silence",
            "text": "",
            "confidence": 0.0,
            "timestamp": time.time(),
            "words": [],
        }
    raise  # 其他 ValueError 继续抛出
```

**好处：**
- ✅ 不中断转写流程
- ✅ 不丢失其他正常音频的转写结果
- ✅ 静默处理异常情况
- ✅ 继续收集主播语音

## ⚠️ 已知限制

### Window Size 错误的根本原因

此错误来自 librosa 对极短音频的 STFT 处理：
- 当音频 < 某个阈值时，窗口大小计算为 0
- FunASR 内部使用 librosa 做音频预处理

**当前方案：**
- 静默处理（返回空结果）
- 不中断转写流程
- 继续处理后续音频

**未来优化：**
- 研究 FunASR 音频预处理参数
- 调整音频切片策略
- 优化实时流处理
