## 🚀 一键启动

### 前置条件

1. **Python 虚拟环境已激活**
```bash
.venv\Scripts\activate
```

2. **依赖已安装**
```bash
pip install -r requirements.txt
npm install
```

### 启动命令

```bash
npm run dev
```

这条命令会自动：
1. ✅ 清理端口 11111 和 10065
2. ✅ 启动后端服务（端口 11111）
3. ✅ 启动前端服务（端口 10065）
4. ✅ 启动 Electron 桌面应用

### 等待时间

- 后端启动：约 **10-15 秒**（需要连接数据库）
- 前端启动：约 **5 秒**
- Electron 启动：约 **3 秒**

**总计约 20-25 秒即可看到桌面应用窗口**

## ⚠️ 常见问题

### 问题 1: 端口被占用

**症状**：启动失败，提示端口被占用

**解决**：
```bash
# 自动清理端口
npm run kill:ports

# 或手动清理
npm run kill:backend  # 清理后端端口
npm run kill:frontend # 清理前端端口
```

### 问题 2: 数据库连接失败

**症状**：后端启动失败，日志显示数据库连接错误

**解决**：

1. 检查 `server/.env` 中的数据库配置：
```ini
DB_TYPE=mysql
MYSQL_HOST=rm-bp1sqxf05yom2hwdhko.mysql.rds.aliyuncs.com
MYSQL_PORT=3306
MYSQL_USER=timao
MYSQL_PASSWORD=timao20251102Xjys
MYSQL_DATABASE=timao
```

2. 测试数据库连接：
```bash
python -c "from server.app.database import init_database; from server.config import config_manager; init_database(config_manager.config.database); print('数据库连接成功')"
```

### 问题 3: 后端启动慢

**症状**：等待超过 30 秒仍未启动

**原因**：
- 数据库连接慢
- 首次加载模型
- 网络问题

**解决**：
1. 等待更长时间（首次启动可能需要 1-2 分钟）
2. 查看日志：`logs/service_manager.log`
3. 手动启动查看详细错误：
```bash
python -m uvicorn server.app.main:app --host 127.0.0.1 --port 11111 --log-level debug
```

### 问题 4: Electron 窗口空白

**症状**：Electron 窗口打开但显示空白

**原因**：前端未完成启动

**解决**：
1. 确认前端服务已启动（访问 http://127.0.0.1:10065）
2. 查看 Electron 控制台（开发者工具）查看错误
3. 重启 Electron：关闭窗口，重新运行 `npm run dev:app`

## 🛠️ 手动分步启动

如果一键启动失败，可以分步启动调试：

### 步骤 1: 启动后端

```bash
npm run dev:backend
```

等待看到：
```
✅ FastAPI服务已启动
[OK] 所有服务启动完成
```

### 步骤 2: 启动前端

**新开一个终端**，运行：
```bash
npm run dev:frontend
```

等待看到：
```
VITE v5.4.21  ready in XXX ms
➜  Local:   http://127.0.0.1:10065/
```

### 步骤 3: 启动 Electron

**新开一个终端**，运行：
```bash
npm run dev:app
```

应该看到 Electron 窗口打开。
