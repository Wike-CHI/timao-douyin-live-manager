# 登录跳转问题分析与解决方案

## 🔍 问题描述

**症状**: 用户在使用过程中，系统会突然跳转到登录界面，需要重新登录。

**影响**: 
- 用户体验差
- 可能丢失当前工作进度
- 频繁需要重新登录

---

## 🎯 根本原因分析

经过代码审查，发现有 **4 个主要触发点** 会导致自动跳转到登录页面：

### 1. Token 过期时间设置过短 ⚠️

**位置**: `server/app/core/security.py:43`

```python
ACCESS_TOKEN_EXPIRE_MINUTES = 30  # ⚠️ 仅 30 分钟
REFRESH_TOKEN_EXPIRE_DAYS = 7
```

**问题**:
- Access Token 只有 **30 分钟**有效期
- 用户使用超过 30 分钟就会遇到 token 过期
- 虽然有 refresh token，但刷新机制可能失败

**影响时间线**:
```
0 分钟 - 登录成功
30 分钟 - Access Token 过期
   ↓
自动尝试刷新 Token
   ↓
如果刷新失败 → 跳转登录页面
```

---

### 2. 前端定期检查过于频繁 ⚡

**位置**: `electron/renderer/src/hooks/useAuthInterceptor.tsx:28-38`

```typescript
// 设置定期检查token状态（每5分钟）
const interval = setInterval(async () => {
  if (isAuthenticated && token) {
    const isValid = await authService.checkAuthStatus();
    if (!isValid) {
      console.log('Periodic auth check failed, redirecting to login');
      clearAuth();
      navigate('/auth/login');  // ⚠️ 直接跳转
    }
  }
}, 5 * 60 * 1000); // 每 5 分钟检查一次
```

**问题**:
- 每 5 分钟检查一次认证状态
- 如果在 token 即将过期时检查，可能来不及刷新
- 没有给 token 刷新机制足够的时间

**时间窗口问题**:
```
第 28 分钟 - 定期检查触发
第 29 分钟 - Token 检测为"即将过期"
第 30 分钟 - Token 过期
第 30.5 分钟 - 刷新请求发出
第 31 分钟 - 后端返回 401（token 已过期）
   ↓
clearAuth() 被调用
   ↓
跳转到登录页面
```

---

### 3. 页面重新获得焦点时的检查 👁️

**位置**: `electron/renderer/src/hooks/useAuthInterceptor.tsx:42-56`

```typescript
// 监听页面可见性变化，页面重新获得焦点时检查认证状态
useEffect(() => {
  const handleVisibilityChange = async () => {
    if (document.visibilityState === 'visible' && isAuthenticated && token) {
      const isValid = await authService.checkAuthStatus();
      if (!isValid) {
        console.log('Visibility change auth check failed, redirecting to login');
        clearAuth();
        navigate('/auth/login');  // ⚠️ 直接跳转
      }
    }
  };

  document.addEventListener('visibilitychange', handleVisibilityChange);
  return () => document.removeEventListener('visibilitychange', handleVisibilityChange);
}, [isAuthenticated, token, clearAuth, navigate]);
```

**问题**:
- 用户切换标签页或最小化窗口后，再返回时会检查
- 如果离开时间较长（> 30 分钟），token 已过期
- 直接跳转到登录页面，没有尝试刷新

**典型场景**:
```
用户操作:
1. 打开应用，开始录制直播
2. 切换到其他应用工作 40 分钟
3. 切回直播录制应用

系统行为:
1. visibilitychange 事件触发
2. 检测到 token 已过期（超过 30 分钟）
3. clearAuth() → navigate('/auth/login')
4. 用户看到登录页面，录制中断 😢
```

---

### 4. Token 刷新失败的情况 🔄

**位置**: `electron/renderer/src/services/authService.ts:41-82`

```typescript
private async refreshAccessToken(): Promise<boolean> {
  const { refreshToken, setAuth, clearAuth } = useAuthStore.getState();
  
  if (!refreshToken) {
    console.log('No refresh token available');
    return false;
  }

  try {
    const response = await fetch(`${API_BASE_URL}/api/auth/refresh`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({ refresh_token: refreshToken }),
    });

    if (!response.ok) {
      console.log('Refresh token request failed:', response.status);
      clearAuth();  // ⚠️ 失败后清除认证
      return false;
    }
    // ...
  } catch (error) {
    console.error('Error refreshing token:', error);
    clearAuth();  // ⚠️ 异常时清除认证
    return false;
  }
}
```

**可能失败的原因**:
1. **网络问题** - 刷新请求超时或失败
2. **Refresh Token 也过期** - 7 天未使用
3. **后端服务重启** - Session 丢失
4. **并发刷新冲突** - 多个请求同时刷新

## 🔧 解决方案

### 方案一：延长 Token 有效期（推荐 ⭐⭐⭐⭐⭐）

**修改文件**: `server/app/core/security.py`

```python
# 原来
ACCESS_TOKEN_EXPIRE_MINUTES = 30

# 改为
ACCESS_TOKEN_EXPIRE_MINUTES = 480  # 8 小时
# 或者
ACCESS_TOKEN_EXPIRE_MINUTES = 1440  # 24 小时
```

**优点**:
- ✅ 最简单直接
- ✅ 立即生效
- ✅ 覆盖大部分使用场景
- ✅ 减少刷新频率

**缺点**:
- ⚠️ 安全性略微降低（但对内部工具可接受）
- ⚠️ Token 被盗用的窗口期变长

**适用场景**:
- 内部管理工具
- 本地 Electron 应用
- 可信任的环境

---

### 方案二：优化 Token 刷新机制（推荐 ⭐⭐⭐⭐）

#### 2.1 提前刷新窗口

**修改文件**: `electron/renderer/src/services/authService.ts`

```typescript
// 原来
private isTokenExpiringSoon(token: string): boolean {
  // ...
  const fiveMinutes = 5 * 60 * 1000;  // 5 分钟
  return exp - now < fiveMinutes;
}

// 改为
private isTokenExpiringSoon(token: string): boolean {
  // ...
  const tenMinutes = 10 * 60 * 1000;  // ✅ 改为 10 分钟
  return exp - now < tenMinutes;
}
```

#### 2.2 重试刷新机制

```typescript
private async refreshAccessToken(retries = 3): Promise<boolean> {
  const { refreshToken, setAuth, clearAuth } = useAuthStore.getState();
  
  if (!refreshToken) {
    console.log('No refresh token available');
    return false;
  }

  for (let i = 0; i < retries; i++) {
    try {
      const response = await fetch(`${API_BASE_URL}/api/auth/refresh`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ refresh_token: refreshToken }),
      });

      if (!response.ok) {
        if (i < retries - 1) {
          console.log(`Refresh attempt ${i + 1} failed, retrying...`);
          await new Promise(resolve => setTimeout(resolve, 1000 * (i + 1))); // 指数退避
          continue;
        }
        console.log('All refresh attempts failed');
        clearAuth();
        return false;
      }

      const data = await response.json();
      const currentState = useAuthStore.getState();
      setAuth({
        user: currentState.user!,
        token: data.access_token,
        refreshToken: data.refresh_token,
        isPaid: currentState.isPaid
      });

      console.log('Token refreshed successfully');
      return true;
      
    } catch (error) {
      if (i < retries - 1) {
        console.log(`Refresh attempt ${i + 1} error, retrying...`, error);
        await new Promise(resolve => setTimeout(resolve, 1000 * (i + 1)));
        continue;
      }
      console.error('All refresh attempts failed with error:', error);
      clearAuth();
      return false;
    }
  }
  
  return false;
}
```

### 方案三：改进认证检查逻辑（推荐 ⭐⭐⭐⭐）

**修改文件**: `electron/renderer/src/hooks/useAuthInterceptor.tsx`

```typescript
const useAuthInterceptor = () => {
  const navigate = useNavigate();
  const { isAuthenticated, token, clearAuth } = useAuthStore();

  useEffect(() => {
    // 应用启动时检查认证状态
    const checkInitialAuth = async () => {
      if (isAuthenticated && token) {
        // ✅ 尝试刷新而不是直接清除
        const isValid = await authService.ensureValidToken();
        if (!isValid) {
          console.log('Initial auth check failed after refresh attempt, redirecting to login');
          clearAuth();
          navigate('/auth/login');
        }
      }
    };

    checkInitialAuth();

    // ✅ 改为每 10 分钟检查一次（给刷新更多时间）
    const interval = setInterval(async () => {
      if (isAuthenticated && token) {
        // ✅ 使用 ensureValidToken 自动刷新
        const newToken = await authService.ensureValidToken();
        if (!newToken) {
          console.log('Periodic auth check failed, redirecting to login');
          clearAuth();
          navigate('/auth/login');
        }
      }
    }, 10 * 60 * 1000); // ✅ 10 分钟

    return () => clearInterval(interval);
  }, [isAuthenticated, token, clearAuth, navigate]);

  // 监听页面可见性变化
  useEffect(() => {
    const handleVisibilityChange = async () => {
      if (document.visibilityState === 'visible' && isAuthenticated && token) {
        // ✅ 先尝试刷新，而不是直接检查是否有效
        const newToken = await authService.ensureValidToken();
        if (!newToken) {
          console.log('Visibility change auth check failed after refresh attempt, redirecting to login');
          clearAuth();
          navigate('/auth/login');
        } else {
          console.log('Token refreshed after visibility change');
        }
      }
    };

    document.addEventListener('visibilitychange', handleVisibilityChange);
    return () => document.removeEventListener('visibilitychange', handleVisibilityChange);
  }, [isAuthenticated, token, clearAuth, navigate]);
};
```
