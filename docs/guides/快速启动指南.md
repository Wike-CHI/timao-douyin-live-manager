- **后端端口**: `11111`

## 📋 启动命令

### 1️⃣ 后端启动（端口 11111）

```bash
# 进入 server 目录
cd /www/wwwroot/wwwroot/timao-douyin-live-manager/server

# 启动后端（带热重载）
python -m uvicorn app.main:app --host 127.0.0.1 --port 11111 --reload
```

**预期输出**:
```
INFO:     Uvicorn running on http://127.0.0.1:11111 (Press CTRL+C to quit)
✅ FastAPI服务已启动
```

**健康检查**:
```bash
curl http://127.0.0.1:11111/health
```

---

### 2️⃣ 前端启动（端口 10050）

**⚠️ 重要**: 修改了 CSP 配置后需要重启前端！

```bash

cd /www/wwwroot/wwwroot/timao-douyin-live-manager/electron/renderer
npm run dev

# 方式2：从项目根目录启动
```

**预期输出**:

## 🔧 CSP 问题已修复

### 问题原因
前端 Vite 的 Content Security Policy 不允许连接到 `129.211.218.135`

### 修复内容
已在 `electron/renderer/vite.config.ts` 中添加：
```typescript
"connect-src 'self' ... http://129.211.218.135 http://129.211.218.135:* ..."
```

### ⚠️ 重要：必须重启前端
修改 CSP 后，**必须重启前端开发服务器**才能生效！

```bash
# 1. 停止当前前端服务（Ctrl+C）
# 2. 重新启动
cd electron/renderer
npm run dev
```

## 🐛 常见问题

### 1. CSP 错误仍然存在
**症状**: 浏览器控制台显示 "violates the document's Content Security Policy"

**解决**:
```bash
# 1. 确认已修改 vite.config.ts
# 2. 停止前端服务（Ctrl+C）
# 3. 清除浏览器缓存（Ctrl+Shift+Delete）
# 4. 重新启动前端
cd electron/renderer
npm run dev
```

### 2. 端口被占用
**症状**: `EADDRINUSE: address already in use`

**解决**:
```bash
# 清理所有端口
npm run kill:ports

# 或单独清理
npm run kill:backend   # 清理 11111
npm run kill:frontend  # 清理 10050
```

### 3. 前端无法连接后端
**症状**: 前端页面显示 "连接失败" 或 "网络错误"

**检查清单**:
- [ ] 后端是否启动？`curl http://127.0.0.1:11111/health`
- [ ] 前端是否启动？`curl http://127.0.0.1:10050`
- [ ] CSP 是否配置正确？检查浏览器控制台
- [ ] 是否重启了前端服务？
