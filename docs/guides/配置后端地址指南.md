# 📝 配置后端地址指南

**问题**：打包的应用默认连接本地 `127.0.0.1`，但后端已部署到云服务器  
**解决**：修改配置，让应用连接到远程服务器 `129.211.218.135:8181`

---

## 🔍 当前问题

### 当前配置（错误）
```javascript
// electron/main.js 第 41-57 行
const serviceConfig = {
    main: {
        port: '8181',
        // 实际使用: http://127.0.0.1:8181 ❌ 本地
    }
};
```

### 需要修改为（正确）
```javascript
const serviceConfig = {
    main: {
        host: '129.211.218.135',  // 添加 host 配置
        port: '8181',
        // 实际使用: http://129.211.218.135:8181 ✅ 远程
    }
};
```

## ✅ 方案一：修改 serviceConfig（推荐）

### 1. 备份原文件
```bash
cd /www/wwwroot/wwwroot/timao-douyin-live-manager
cp electron/main.js electron/main.js.backup
```

### 2. 修改 electron/main.js

找到第 41-57 行的 `serviceConfig`，修改为：

```javascript
// 服务配置
const serviceConfig = {
    main: {
        name: 'FastAPI',
        host: process.env.BACKEND_HOST || '129.211.218.135',  // ✅ 添加这行
        port: process.env.BACKEND_PORT || '8181',
        healthPath: '/health'
    },
    streamcap: {
        name: 'StreamCap',
        host: process.env.BACKEND_HOST || '129.211.218.135',  // ✅ 添加这行
        port: process.env.STREAMCAP_PORT || '8181',
        healthPath: '/health'
    },
    douyin: {
        name: 'DouyinLive',
        host: process.env.BACKEND_HOST || '129.211.218.135',  // ✅ 添加这行
        port: process.env.DOUYIN_PORT || '8181',
        healthPath: '/health'
    }
};
```

### 3. 修改使用 serviceConfig 的地方

在 `electron/main.js` 中，查找所有使用 `127.0.0.1` 的地方，替换为 `${config.host}`：

```javascript
// 修改前：
`http://127.0.0.1:${serviceConfig.main.port}`

// 修改后：
`http://${serviceConfig.main.host}:${serviceConfig.main.port}`
```

### 4. 完整修改清单

在 `electron/main.js` 中查找并替换以下内容：

```bash
# 查找需要修改的位置
cd /www/wwwroot/wwwroot/timao-douyin-live-manager
grep -n "127.0.0.1.*serviceConfig" electron/main.js
```

修改示例：

```javascript
// 位置 1: 环境变量设置（约第 185 行）

// 位置 3: 其他使用位置
```

## ✅ 方案二：使用配置文件（灵活）

### 1. 创建配置文件

创建 `electron/config.json`:

```json
{
  "backend": {
    "host": "129.211.218.135",
    "port": "8181"
  },
  "frontend": {
    "host": "129.211.218.135",
    "port": "10050"
  },
  "environment": "production"
}
```

### 2. 在 main.js 中读取配置

在 `electron/main.js` 开头添加：

```javascript
const { app, BrowserWindow, ipcMain, shell } = require('electron');
const { spawn, spawnSync } = require('child_process');
const path = require('path');
const fs = require('fs');

// ✅ 读取配置文件
let backendConfig = {
    host: '129.211.218.135',
    port: '8181'
};

try {
    const configPath = path.join(__dirname, 'config.json');
    if (fs.existsSync(configPath)) {
        const config = JSON.parse(fs.readFileSync(configPath, 'utf8'));
        if (config.backend) {
            backendConfig = config.backend;
        }
    }
} catch (error) {
    console.error('读取配置文件失败，使用默认配置:', error);
}

// 修改 serviceConfig
const serviceConfig = {
    main: {
        name: 'FastAPI',
        host: backendConfig.host,  // ✅ 从配置文件读取
        port: backendConfig.port,
        healthPath: '/health'
    },
    // ...
};
```

## ⚠️ 注意事项

### 1. 生产环境 vs 开发环境

建议区分开发和生产环境：

```javascript
const isDev = !app.isPackaged;

const serviceConfig = {
    main: {
        host: isDev ? '127.0.0.1' : '129.211.218.135',  // 开发用本地，生产用远程
        port: '8181',
        healthPath: '/health'
    }
};
```

### 2. 前端配置

还需要检查前端配置：

```bash
# 检查前端配置
cd electron/renderer
grep -r "127.0.0.1" src/
```

如果前端也有硬编码的地址，需要一并修改。

### 3. CSP 策略

检查 `electron/renderer/vite.config.ts` 中的 CSP 策略是否包含服务器地址：

```typescript
"connect-src 'self' http://129.211.218.135:8181 ws://129.211.218.135:8181"
```

## 🆘 遇到问题？

### Q: 修改后无法连接后端

**A**: 检查：
1. 服务器防火墙是否开放 8181 端口
2. 后端服务是否正常运行：`curl http://129.211.218.135:8181/health`
3. 应用是否真的使用了新配置（检查应用日志）

### Q: 如何查看应用使用的地址？

**A**: 在应用中打开开发者工具（F12），查看 Network 请求

### Q: 能否让用户自己配置服务器地址？

**A**: 可以，在应用中添加设置页面，让用户输入服务器地址并保存

**推荐使用方案一（修改 serviceConfig）+ 环境变量**：

3. 生产环境默认使用服务器地址
4. 保留环境变量覆盖选项
