## 🏗️ 架构对比

### 之前的架构 ❌

```
打包流程:
1. 构建 Python 后端 → backend_dist/
2. 打包 Python 环境 (.venv)
3. 构建前端 → electron/renderer/dist/
4. 打包 Electron + 后端 + Python
5. 生成 500MB 安装包

运行流程:
1. 用户安装应用
2. Electron 启动
3. 启动本地 Python 后端
4. 前端连接本地后端 (127.0.0.1:11111)
```

### 现在的架构 ✅

```
打包流程:
1. 构建前端 → electron/renderer/dist/
2. 打包 Electron + 前端
3. 生成 100MB 安装包

运行流程:
1. 用户安装应用
2. Electron 启动
3. 前端直接连接公网后端 (129.211.218.135)
```

## 🔧 配置文件

### 核心配置文件

1. **build-config.json** - Electron Builder 配置
   - 定义打包规则
   - 指定包含/排除文件
   - 配置目标平台

2. **package.json** - 项目配置
   - 定义构建脚本
   - 管理依赖
   - 配置应用元数据

3. **.env.production** (自动生成) - 前端环境变量
   - 配置后端 API 地址
   - 设置环境标识
   - 记录构建信息

### 环境变量

**前端 (自动配置)**:
```bash
VITE_FASTAPI_URL=http://129.211.218.135
VITE_STREAMCAP_URL=http://129.211.218.135
VITE_DOUYIN_URL=http://129.211.218.135
VITE_APP_ENV=production
```

**打包脚本 (可自定义)**:
```bash
# 修改后端地址
PRODUCTION_API_URL="http://你的IP"

# 修改构建目标
BUILD_TARGET=win  # 或 mac, linux, all
```
