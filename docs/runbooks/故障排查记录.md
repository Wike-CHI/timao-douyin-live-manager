# 提猫直播助手 - 故障排查记录

## 2025-11-09 部署故障记录

### 问题1: Nginx 502错误 - Nginx配置问题
**时间**: 08:00-08:15

**症状**:
- 客户端访问所有API返回502
- Nginx日志显示 404 Not Found
- 后端服务运行正常

**原因**:
```nginx
# 错误配置
location /api/ {
    proxy_pass http://127.0.0.1:8181;  # ❌ 缺少/api/路径
}
```

**解决**:
```nginx
# 正确配置
location /api/ {
    proxy_pass http://127.0.0.1:8181/api/;  # ✅ 包含完整路径
}
```

**命令**:
```bash
vim /www/server/panel/vhost/nginx/129.211.218.135.conf
nginx -t
systemctl reload nginx
```

### 问题2: 后端频繁重启 - 内存限制过低
**时间**: 08:15-08:30

**症状**:
- 服务每隔几分钟自动重启
- 客户端出现间歇性502错误
- systemd日志显示 "Scheduled restart"

**原因**:
```bash
Memory: 794.0M (peak: 806.1M)  # 超过限制
MemoryHigh=800M  # 触发限制导致重启
```

**解决**:
```systemd
# 修改 /etc/systemd/system/talkingcat-backend.service
[Service]
MemoryMax=2G      # 从1G增加到2G
MemoryHigh=1.5G   # 从800M增加到1.5G
```

**命令**:
```bash
vim /etc/systemd/system/talkingcat-backend.service
systemctl daemon-reload
systemctl restart talkingcat-backend
systemctl status talkingcat-backend
```

**结果**:
- 内存使用：353M（有1.1G可用空间）
- 服务稳定运行
- 无重启记录

## 常见问题处理

### 502 Bad Gateway
1. 检查后端服务是否运行：`systemctl status talkingcat-backend`
2. 检查端口监听：`netstat -tlnp | grep 8181`
3. 检查Nginx配置：`nginx -t`
4. 查看Nginx错误日志：`tail -f /www/wwwlogs/129.211.218.135.error.log`

### 服务频繁重启
1. 查看systemd日志：`journalctl -u talkingcat-backend -n 50`
2. 检查内存使用：`systemctl status talkingcat-backend | grep Memory`
3. 调整内存限制：编辑`/etc/systemd/system/talkingcat-backend.service`

### API响应慢
1. 查看后端日志：`tail -f logs/backend.log`
2. 检查Redis连接：`redis-cli ping`
3. 检查数据库连接：查看日志中的数据库响应时间
