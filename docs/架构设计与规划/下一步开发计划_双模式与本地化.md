# 下一步开发计划（双模式 + 本地化）

---

## 1. 目标与范围

- 双模式体验落地
  - 全功能/直播模式切换的可控性（自动切换开关、确认弹窗、失败回滚）
  - 悬浮窗 MVP：收起/展开、拖拽/吸附、位置记忆、三类信息展示（AI分析/话术/氛围）
  - 信息优先级：高优先级即时提醒 + 基础轮播调度
- 本地化改造（云-本混合）
  - 在 Electron 内启动本地 FastAPI（默认 16000），承载弹幕、语音转写、AI相关重服务
  - 前端 API 动态路由：优先命中本地服务，不可用时回退云端（15000/80 通过 Nginx）
  - 健康检查、断线重试、日志与资源回收

超出范围（本次不做）
- 多悬浮窗、主题皮肤、A/B 实验平台、效果归因与统计

---

## 2. 里程碑与验收标准

M1 本地服务骨架与服务发现（2-3 天）
- 交付物
  - Electron 主进程能随应用启动/退出本地 FastAPI，端口 16000 可配置
  - `apiConfig.ts` 支持本地优先、云端回退；新增本地健康检查项
  - 健康检查面板可见三项状态：main/streamcap/douyin（本地或云）
- 验收
  - 断开本地服务 → 前端自动退回云端；恢复后自动切回（无需重启应用）
  - Windows/macOS 均能启动与回收 Python 进程

M2 直播模式 MVP（3-4 天）
- 交付物
  - 悬浮窗组件：收起/展开、拖拽、吸附、位置记忆、复制按钮、①②③指示器
  - 模式切换：设置中“自动切换到直播模式”开关、生效的确认弹窗
  - WebSocket/SSE 数据接入（本地优先），三类信息基本展示
- 验收
  - 开始转写后，服务就绪且用户同意 → 切换直播模式并显示悬浮窗
  - 关闭自动切换时，出现确认对话框且可留在全功能模式继续运行

M3 优先级调度与高优提醒（3 天）
- 交付物
  - 高优先级事件：大额礼物、关键问题、冷场、异常（支持本地服务推送）
  - 轮播调度：高优插队、普通信息轮播、手动切换暂停/恢复
  - `generate_smart`、`silence_check` 等接口初版（本地 FastAPI）
- 验收
  - 高优触发时悬浮窗自动展开并突出显示，10s 后恢复轮播或等待确认
  - 冷场检测能在无弹幕 30s 后给出破冰建议

M4 打包与运维（1-2 天）
- 交付物
  - Windows 安装包内置本地 FastAPI 与模型缓存目录初始化
  - Nginx 配置更新指引；`.env` 模板与运行参数说明
- 验收
  - 全新环境安装后一键运行；首次启动本地模型预热可见、可取消

## 3. 前端开发清单

3.1 设置页与状态管理
- 新增设置项：`autoSwitchLiveMode`（默认开启），落盘至 LocalStorage
- `useLiveModeStore` 扩展：`currentMode`、`floatingWindow`、`switchMode/rollback` 等
- 加入 `confirmSwitchDialog()`；`rollbackServices()` 的 UI 与提示

3.2 API 动态路由与健康检查（修改 electron/renderer/src/services/apiConfig.ts）
- 新增环境变量
  - `VITE_LOCAL_API_URL`（默认 `http://127.0.0.1:16000`）
  - `VITE_CLOUD_API_URL`（默认 Nginx 域名；保留现有变量）
- 本地优先策略
  - 健康检查时先探测本地 `GET /health`，失败再改探云端
  - `buildServiceUrl()` 支持显式覆盖 baseUrl（用于特定接口走本地/云端）
- 故障回退
  - 本地不可用 → 标记状态并回退；周期性重试以恢复

3.3 直播模式 UI（渲染进程）
- 悬浮窗组件
  - 收起 80x80；展开 320x240（小屏 280x200）；拖拽、边缘吸附、位置记忆
  - 指示器①②③切换信息类别；复制按钮；“更多”跳全功能页面
- 动画与交互
  - `fade-in/out`、`zoom-in`；键盘快捷键 Ctrl+1/2/3、Ctrl+Shift+F
- 轮播/高优管理
  - `CarouselManager` 与 `AlertManager`；高优插队显示、恢复轮播

3.4 模式切换逻辑
- 启动流程加入：
  - 用户确认逻辑（当自动切换关闭时）
  - 失败回滚：关闭已启动服务、断开 WS、恢复 UI
  - 自动重试：2s 指数回退，最多 3 次

## 4. 后端开发清单（本地 FastAPI）

4.1 服务骨架与 CORS
- 新建/复用本地 FastAPI 入口，暴露 `GET /health`
- CORS 允许 Electron 渲染进程域名与端口

4.2 直播相关接口（与设计文档一致）
- 语音转写
  - `POST /api/live_audio/start`、`POST /api/live_audio/stop`、`GET /api/live_audio/status`
  - `WS /api/live_audio/ws` 推送转写结果
- 抖音弹幕/礼物
  - `POST /api/douyin/web/start`、`SSE /api/douyin/web/stream`
- AI 能力
  - `POST /api/ai/scripts/generate_smart`（含高价值用户检测）
  - `GET /api/ai/live/silence_check`
  - `WS /ws/live_mode` 推送高优事件与队列消息

4.3 优先级与调度
- `PriorityQueue` 与 `InfoScheduler`（高/中/低→3/2/1）
- `MessageDispatcher`：高优立即推送；中优队列；低优存储

4.4 资源管理与回滚
- 接口失败/中断时释放音频拉流、关闭线程/子进程
- 统一 `rollback_services()`（供前端失败后调用）

## 8. 风险与预案

- Windows 防火墙阻断本地端口：首次启动提示放行；端口可切换
- 模型加载耗时与内存占用：懒加载与低精度模型选项；内存告警提示
- 本地进程崩溃：主进程守护 + 退回云端 + 用户提示
- 账号/权限耦合：云端鉴权独立于本地服务，令牌仅在云端校验
