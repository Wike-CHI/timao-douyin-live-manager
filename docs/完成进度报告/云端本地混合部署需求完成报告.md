#### 2.1.1 模型下载优化（新增 2025-11-16）

✅ **问题修复**: 模型重复下载问题

**发现问题**：
- 用户反馈：模型已下载到 `~/.cache/modelscope/hub/models/iic/SenseVoiceSmall`，但运行时仍重新下载
- 根本原因：`check_sensevoice_model()` 只检查项目本地路径 `server/models/models/iic/SenseVoiceSmall`，未检查 FunASR 的 ModelScope 缓存目录

**解决方案**：

1. **新增跨平台缓存路径获取函数**
   ```python
   def _get_modelscope_cache_path(model_id: str) -> Optional[Path]:
       """获取 ModelScope 缓存路径（跨平台兼容）"""
       home = Path.home()
       if platform.system() == "Windows":
           cache_root = home / ".cache" / "modelscope" / "hub" / "models"
       else:  # Linux/macOS
           cache_root = home / ".cache" / "modelscope" / "hub" / "models"
       
       model_path = cache_root / model_id.replace("/", os.sep)
       return model_path if model_path.exists() else None
   ```

2. **完善模型检查逻辑**
   ```python
   def check_sensevoice_model(models_dir: Path) -> Optional[Path]:
       # 1. 优先检查项目本地模型
       model_dir = models_dir / "models" / "iic" / "SenseVoiceSmall"
       if model_dir.exists() and (model_dir / "model.pt").exists():
           logger.info(f"✅ 发现本地模型: {model_dir}")
           return model_dir
       
       # 2. 检查 ModelScope 缓存（新增）
       modelscope_cache = _get_modelscope_cache_path("iic/SenseVoiceSmall")
       if modelscope_cache and (modelscope_cache / "model.pt").exists():
           logger.info(f"✅ 发现 ModelScope 缓存模型: {modelscope_cache}")
           return modelscope_cache
       
       return None
   ```

3. **优化下载流程**
   ```python
   async def ensure_sensevoice_model(...) -> str:
       # 1. 检查本地模型（包括项目本地和 ModelScope 缓存）
       local_model = check_sensevoice_model(models_dir)
       if local_model:
           logger.info(f"✅ 使用已下载的模型: {local_model}")
           return str(local_model)
       
       # 2. 如果启用 FunASR 自动下载，直接返回 ModelScope ID
       if use_funasr_auto:
           logger.info("使用 FunASR 自动下载机制（ModelScope）")
           return "iic/SenseVoiceSmall"
       ...
   ```

       this.process = null;

     }
     
     start() {
       const pythonPath = this.getPythonPath();
       const scriptPath = path.join(__dirname, '../../server/local/main.py');
       
       this.process = spawn(pythonPath, ['-m', 'server.local.main'], {

### 任务3：Python运行时打包方案选择（预计1天）

## 🎓 经验总结

### 成功经验

1. **优先使用现有框架**
   - 使用FunASR自动下载替代手动实现HTTP下载
   - 节省开发时间，提高稳定性

2. **测试先行策略**
   - 在实现功能前编写测试用例框架
   - 使用真实直播间URL测试，提前发现问题

3. **模块化设计**
   - `model_downloader.py`独立可测试
   - `live_audio_stream_service.py`职责单一
   - 降低耦合，提高可维护性

4. **自动化工具优先**（新增 2025-11-16）
   - 创建 `check_local_imports.py` 自动检测外部依赖
   - 使用正则表达式精确匹配导入模式
   - 避免手动遗漏，提高检查效率

### 遇到的挑战

1. **VAD模型路径检查不完整**
   - 问题：只检查目录存在，未验证`model.pb`文件
   - 解决：添加文件级别的存在性检查
   - 教训：路径检查需验证关键文件，而非仅检查目录

2. **API方法名不匹配**
   - 问题：测试脚本使用了错误的方法名（`get_status()` vs `status()`）
   - 解决：使用`grep_search`查找正确API，批量修正
   - 教训：在编写测试前，先验证API签名

3. **真实直播间测试不稳定**
   - 问题：直播间可能下播导致测试失败
   - 解决：添加fallback机制，使用测试音频文件
   - 教训：集成测试需要准备备用方案

4. **模型重复下载问题（新增 2025-11-16）**
   - 问题：用户已有 ModelScope 缓存模型，但仍重新下载浪费时间和流量
   - 根因：`check_sensevoice_model()` 只检查项目本地路径，未检查 FunASR 的 `~/.cache/modelscope/` 缓存
   - 解决：
     * 新增 `_get_modelscope_cache_path()` 跨平台获取缓存路径
     * 完善检查逻辑：项目本地 → ModelScope 缓存 → 返回 None
     * 添加详细日志标注模型来源
   - 教训：
     * 集成第三方库时，需充分理解其缓存机制和存储路径
     * 模型文件体积大，必须优先复用已有文件避免重复下载
     * 跨平台路径处理需使用 `platform.system()` 和 `os.sep`
