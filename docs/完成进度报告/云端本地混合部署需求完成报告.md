#### 2.1.1 模型下载优化（新增 2025-11-16）

✅ **问题修复**: 模型重复下载问题

**发现问题**：
- 用户反馈：模型已下载到 `~/.cache/modelscope/hub/models/iic/SenseVoiceSmall`，但运行时仍重新下载
- 根本原因：`check_sensevoice_model()` 只检查项目本地路径 `server/models/models/iic/SenseVoiceSmall`，未检查 FunASR 的 ModelScope 缓存目录

**解决方案**：

1. **新增跨平台缓存路径获取函数**
   ```python
   def _get_modelscope_cache_path(model_id: str) -> Optional[Path]:
       """获取 ModelScope 缓存路径（跨平台兼容）"""
       home = Path.home()
       if platform.system() == "Windows":
           cache_root = home / ".cache" / "modelscope" / "hub" / "models"
       else:  # Linux/macOS
           cache_root = home / ".cache" / "modelscope" / "hub" / "models"
       
       model_path = cache_root / model_id.replace("/", os.sep)
       return model_path if model_path.exists() else None
   ```

2. **完善模型检查逻辑**
   ```python
   def check_sensevoice_model(models_dir: Path) -> Optional[Path]:
       # 1. 优先检查项目本地模型
       model_dir = models_dir / "models" / "iic" / "SenseVoiceSmall"
       if model_dir.exists() and (model_dir / "model.pt").exists():
           logger.info(f"✅ 发现本地模型: {model_dir}")
           return model_dir
       
       # 2. 检查 ModelScope 缓存（新增）
       modelscope_cache = _get_modelscope_cache_path("iic/SenseVoiceSmall")
       if modelscope_cache and (modelscope_cache / "model.pt").exists():
           logger.info(f"✅ 发现 ModelScope 缓存模型: {modelscope_cache}")
           return modelscope_cache
       
       return None
   ```

3. **优化下载流程**
   ```python
   async def ensure_sensevoice_model(...) -> str:
       # 1. 检查本地模型（包括项目本地和 ModelScope 缓存）
       local_model = check_sensevoice_model(models_dir)
       if local_model:
           logger.info(f"✅ 使用已下载的模型: {local_model}")
           return str(local_model)
       
       # 2. 如果启用 FunASR 自动下载，直接返回 ModelScope ID
       if use_funasr_auto:
           logger.info("使用 FunASR 自动下载机制（ModelScope）")
           return "iic/SenseVoiceSmall"
       ...
   ```

#### 2.1.2 模型存储路径方案（新增 2025-11-16）

✅ **最终决策**: 使用**用户数据目录**存储模型

**路径规划**：

| 平台 | 存储路径 | 示例 |
|------|---------|------|
| Windows | `%APPDATA%\TimaoLiveManager\models\` | `C:\Users\dell\AppData\Roaming\TimaoLiveManager\models\` |
| macOS | `~/Library/Application Support/TimaoLiveManager/models/` | `/Users/username/Library/Application Support/TimaoLiveManager/models/` |
| Linux | `~/.config/TimaoLiveManager/models/` | `/home/username/.config/TimaoLiveManager/models/` |

**方案对比**：

| 方案 | 优点 | 缺点 | 选择 |
|------|-----|------|------|
| **安装路径** | 集中管理，易于卸载 | 需管理员权限，可能误删 | ❌ |
| **用户数据目录** | 无需权限，独立保存 | 用户不易发现位置 | ✅ **推荐** |

**Electron 实现代码**：
```javascript
const { app } = require('electron');
const path = require('path');

// 获取用户数据目录
const userDataPath = app.getPath('userData');
// Windows: C:\Users\<用户名>\AppData\Roaming\timao-live
// macOS: ~/Library/Application Support/timao-live
// Linux: ~/.config/timao-live

const MODEL_PATHS = {
  sensevoice: path.join(userDataPath, 'models', 'sensevoice'),
  vad: path.join(userDataPath, 'models', 'vad'),
  temp: path.join(userDataPath, 'models', '.download'),
};
```

**注意事项**：
1. ✅ 启动前检查磁盘空间（模型大小 + 10% 安全边际）

   ```bash
   ffmpeg -i {stream_url} \
          -f s16le \           # PCM16格式
          -acodec pcm_s16le \
          -ar 16000 \          # 16kHz采样率

          pipe:1
   ```

3. **VAD语音活动检测**
   - 模型：`speech_fsmn_vad_zh-cn-16k-common-pytorch`

   + if local_vad.exists() and (local_vad / "model.pb").exists():

   - 内存占用：~3.3GB

## 🔍 技术债务与风险

### 已识别技术债务

1. **模型文件体积过大**
   - 问题：SenseVoice模型1.7GB，首次下载耗时较长
   - 影响：用户首次启动体验差
   - 缓解措施：
     - ✅ 实现断点续传（已完成）
     - 🟡 提供离线模型包（计划中）
     - 🟡 显示下载进度UI（待实施）

2. **内存占用较高**
   - 问题：模型加载后占用3.3GB内存
   - 影响：低配设备可能无法运行
   - 缓解措施：
     - 🟡 模型量化（减少到1.5GB，计划中）
     - 🟡 GPU加速（降低CPU占用，待研究）
     - ⏳ 内存占用监控（未开始）

3. **Python运行时打包方案未定**
   - 问题：Electron集成Python的最佳方案未确定

## 🎓 经验总结

### 成功经验

1. **优先使用现有框架**
   - 使用FunASR自动下载替代手动实现HTTP下载
   - 节省开发时间，提高稳定性

2. **测试先行策略**
   - 在实现功能前编写测试用例框架
   - 使用真实直播间URL测试，提前发现问题

3. **模块化设计**
   - `model_downloader.py`独立可测试
   - `live_audio_stream_service.py`职责单一
   - 降低耦合，提高可维护性

### 遇到的挑战

1. **VAD模型路径检查不完整**
   - 问题：只检查目录存在，未验证`model.pb`文件
   - 解决：添加文件级别的存在性检查
   - 教训：路径检查需验证关键文件，而非仅检查目录

2. **API方法名不匹配**
   - 问题：测试脚本使用了错误的方法名（`get_status()` vs `status()`）
   - 解决：使用`grep_search`查找正确API，批量修正
   - 教训：在编写测试前，先验证API签名

3. **真实直播间测试不稳定**
   - 问题：直播间可能下播导致测试失败
   - 解决：添加fallback机制，使用测试音频文件
   - 教训：集成测试需要准备备用方案

4. **模型重复下载问题（新增 2025-11-16）**
   - 问题：用户已有 ModelScope 缓存模型，但仍重新下载浪费时间和流量
   - 根因：`check_sensevoice_model()` 只检查项目本地路径，未检查 FunASR 的 `~/.cache/modelscope/` 缓存
   - 解决：
     * 新增 `_get_modelscope_cache_path()` 跨平台获取缓存路径
     * 完善检查逻辑：项目本地 → ModelScope 缓存 → 返回 None
     * 添加详细日志标注模型来源
   - 教训：
     * 集成第三方库时，需充分理解其缓存机制和存储路径
     * 模型文件体积大，必须优先复用已有文件避免重复下载
     * 跨平台路径处理需使用 `platform.system()` 和 `os.sep`
