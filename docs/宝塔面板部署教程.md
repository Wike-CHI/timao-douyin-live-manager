## 1. 创建站点

### 1.1 登录宝塔面板
1. 打开浏览器，访问：`http://你的服务器IP:8888`
2. 输入宝塔面板的账号和密码登录

### 1.2 添加站点
1. 点击左侧菜单 **"网站"**
2. 点击 **"添加站点"** 按钮
3. 填写站点信息：
   - **域名**：填写你的域名，例如：`live.example.com`
   - **备注**：提猫直播助手
   - **根目录**：保持默认或选择：`/www/wwwroot/live.example.com`
   - **FTP**：不创建
   - **数据库**：如果需要，选择MySQL
   - **PHP版本**：选择"纯静态"
4. 点击 **"提交"**

### 1.3 站点配置说明
- 如果您只需要反向代理到后端服务，根目录可以是空的或放置一个简单的静态页面
- 如果您要部署前端文件，将前端打包后的文件放到根目录

## 2. 配置反向代理

### 2.1 打开站点设置
1. 在网站列表中，找到刚创建的站点
2. 点击 **"设置"** 按钮

### 2.2 配置反向代理
1. 点击 **"反向代理"** 选项卡
2. 点击 **"添加反向代理"**
3. 填写反向代理信息：

#### 配置示例（后端API）
```
代理名称：后端API
目标URL：http://127.0.0.1:11111
发送域名：$host
内容替换：留空
```

#### 高级配置
点击"高级"按钮，添加以下配置：

```nginx
# WebSocket支持
proxy_http_version 1.1;
proxy_set_header Upgrade $http_upgrade;
proxy_set_header Connection "upgrade";

# 代理头设置
proxy_set_header Host $host;
proxy_set_header X-Real-IP $remote_addr;
proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
proxy_set_header X-Forwarded-Proto $scheme;

# 超时设置
proxy_connect_timeout 60s;
proxy_send_timeout 300s;
proxy_read_timeout 300s;

# 缓冲设置
proxy_buffering off;
```

4. 点击 **"提交"**

### 2.3 完整的Nginx配置（手动配置）

如果需要更精细的控制，可以点击"配置文件"直接编辑：

```nginx
server {
    listen 80;
    server_name live.example.com;
    
    # 日志
    access_log /www/wwwlogs/live.example.com.log;
    error_log /www/wwwlogs/live.example.com.error.log;
    
    # 前端静态文件（如果有）
    location / {
        root /www/wwwroot/live.example.com;
        index index.html;
        try_files $uri $uri/ /index.html;
    }
    
    # 后端API反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:11111;
        proxy_http_version 1.1;
        
        # WebSocket支持
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        
        # 代理头
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # 超时
        proxy_connect_timeout 60s;
        proxy_send_timeout 300s;
        proxy_read_timeout 300s;
        
        # 关闭缓冲（适合流式响应）
        proxy_buffering off;
    }
    
    # WebSocket专用路径（如果有）
    location /ws/ {
        proxy_pass http://127.0.0.1:11111;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        
        # WebSocket超时（更长）
        proxy_read_timeout 3600s;
        proxy_send_timeout 3600s;
    }
    
    # 健康检查
    location /health {
        proxy_pass http://127.0.0.1:11111;
        access_log off;
    }
}
```

## 3. 配置SSL证书（可选但推荐）

### 3.1 Let's Encrypt免费证书
1. 在站点设置中，点击 **"SSL"** 选项卡
2. 选择 **"Let's Encrypt"**
3. 勾选你的域名
4. 填写邮箱
5. 点击 **"申请"**

### 3.2 强制HTTPS
1. SSL证书申请成功后
2. 开启 **"强制HTTPS"** 开关

### 3.3 HTTPS配置示例
```nginx
server {
    listen 443 ssl http2;
    server_name live.example.com;
    
    # SSL证书
    ssl_certificate /www/server/panel/vhost/cert/live.example.com/fullchain.pem;
    ssl_certificate_key /www/server/panel/vhost/cert/live.example.com/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    
    # ... 其他配置同上
}

# HTTP自动跳转HTTPS
server {
    listen 80;
    server_name live.example.com;
    return 301 https://$host$request_uri;
}
```

## 4. 配置防火墙

### 4.1 开放端口
1. 点击宝塔面板左侧 **"安全"**
2. 添加以下端口规则：

| 端口 | 协议 | 说明 |
|------|------|------|
| 80 | TCP | HTTP |
| 443 | TCP | HTTPS |
| 11111 | TCP | 后端API（内部访问，可选） |

**注意**：如果使用反向代理，后端端口11111不需要对外开放，只在内部访问即可。

### 4.2 云服务器安全组
如果使用阿里云/腾讯云等云服务器，还需要在云控制台配置安全组：
1. 登录云服务器控制台
2. 找到"安全组"设置
3. 添加入站规则：允许80、443端口

## 5. 测试访问

### 5.1 测试反向代理
1. 在浏览器访问：`http://你的域名/health`
2. 应该返回类似：
```json
{
  "status": "healthy",
  "service": "提猫直播助手",
  "version": "1.0.0"
}
```

### 5.2 测试API
访问：`http://你的域名/api/cors-test`
应该返回CORS配置信息。

### 5.3 测试WebSocket（如果有）
使用浏览器控制台：
```javascript
const ws = new WebSocket('ws://你的域名/ws/test');
ws.onopen = () => console.log('WebSocket连接成功');
ws.onerror = (e) => console.error('WebSocket错误', e);
```

## 🔧 常见问题排查

### 问题1：502 Bad Gateway
**原因**：后端服务未启动或端口不对
**解决**：
```bash
# 检查后端服务是否运行
ps aux | grep python
# 检查端口是否监听
netstat -tlnp | grep 11111
# 检查后端日志
tail -f /www/wwwroot/wwwroot/timao-douyin-live-manager/logs/backend.log
```

### 问题2：504 Gateway Timeout
**原因**：后端响应超时
**解决**：增加Nginx超时时间（见上面配置）

### 问题3：CORS错误
**原因**：跨域配置问题
**解决**：检查后端CORS配置，确保允许你的域名

### 问题4：SSL证书申请失败
**原因**：域名未解析或80端口未开放
**解决**：
1. 检查域名DNS解析是否正确
2. 确保80端口已开放
3. 暂时关闭CDN（如果使用）
