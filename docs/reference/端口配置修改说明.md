## 🚀 启动方式

### 方式1: 集成启动 (推荐)
```bash
npm run start:integrated
```
此命令会：
1. 启动后端服务 (端口 11111)
2. 启动前端开发服务器 (端口 10050)
3. 启动 Electron 应用

### 方式2: 分步启动
```bash
# 1. 启动后端
cd server
python -m uvicorn app.main:app --host 127.0.0.1 --port 11111

# 2. 启动前端 (新终端)
cd electron/renderer
npm run dev

# 3. 启动 Electron (新终端)
npm run dev:electron
```

## ⚠️ 重要提醒

### 1. 端口占用检查
如果启动失败，检查端口是否被占用:
```bash
# Linux/Mac
lsof -i :10050
lsof -i :11111

# Windows
netstat -ano | findstr :10050
netstat -ano | findstr :11111
```

### 2. 清理端口
```bash
# 使用项目提供的脚本清理端口
npm run kill:ports
```

### 3. 防火墙配置
如果部署到服务器，确保防火墙允许这些端口:
```bash
# Ubuntu/Debian
sudo ufw allow 10050/tcp
sudo ufw allow 11111/tcp

# CentOS/RHEL
sudo firewall-cmd --permanent --add-port=10050/tcp
sudo firewall-cmd --permanent --add-port=11111/tcp
sudo firewall-cmd --reload
```

## 🔧 故障排除

### 问题1: 前端端口被占用
**现象**: `Error: listen EADDRINUSE: address already in use 127.0.0.1:10050`

**解决**:
```bash
# 杀死占用进程
npm run kill:frontend

# 或手动查找并杀死
lsof -ti:10050 | xargs kill -9  # Linux/Mac
```

### 问题2: 后端端口被占用
**现象**: `ERROR:    [Errno 98] Address already in use`

**解决**:
```bash
# 杀死占用进程
npm run kill:backend

# 或手动查找并杀死
lsof -ti:11111 | xargs kill -9  # Linux/Mac
```

### 问题3: Electron无法连接到前端
**现象**: Electron窗口显示"无法加载页面"

**原因**: 前端开发服务器未启动或端口不匹配

**解决**:
1. 确认前端服务器已启动: `curl http://127.0.0.1:10050`
2. 检查配置是否正确: `node scripts/检查与校验/verify-port-config.js`
3. 重启服务
