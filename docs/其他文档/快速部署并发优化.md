## 📊 关键指标

### 正常状态
- **活跃请求**: 0-2（不超过2）
- **超时次数**: 0或个位数
- **错误次数**: 0
- **内存占用**: 2.0-2.8GB

### 异常状态（需要处理）
- **活跃请求**: 持续>2 → 检查信号量是否生效
- **超时次数**: 持续增长 → 系统负载过高或超时设置过短
- **错误次数**: 持续增长 → 模型加载失败或内存不足
- **内存占用**: >3.0GB → 内存泄漏，建议重启

## 🚨 故障快速恢复

### 问题1: 服务启动失败

```bash
# 查看错误日志
pm2 logs timao-backend --err --lines 50

# 常见原因：依赖包缺失
cd /www/wwwroot/wwwroot/timao-douyin-live-manager
source .venv/bin/activate
pip install -r requirements.txt

# 重启
pm2 restart timao-backend
```

### 问题2: 转写超时频繁

```bash
# 临时方案：减少并发数或增加超时时间
# 编辑 server/modules/ast/sensevoice_service.py
# 修改：
#   self._max_concurrent = 1  # 降至1并发
#   self._timeout_seconds = 15.0  # 延长至15秒

pm2 restart timao-backend
```

### 问题3: 内存仍然溢出

```bash
# 定时重启服务（每6小时）
pm2 restart timao-backend --cron "0 */6 * * *"

# 或手动重启
pm2 restart timao-backend
```

## 📈 性能调优

### 如果升级到16GB内存

编辑 `server/modules/ast/sensevoice_service.py`:

```python
self._max_concurrent = 4  # 可支持4并发
self._timeout_seconds = 15.0  # 延长超时
```

### 如果只有4GB内存

```python
self._max_concurrent = 1  # 只支持1并发
self._timeout_seconds = 8.0  # 缩短超时
```

配合内存优化：
```python
chunk_size: int = 800  # 进一步减小chunk（牺牲准确性）
```
