# 版本管理指南

**审查人**: 叶维哲  
**最后更新**: 2025-11-14

## 📋 概述

本指南说明如何管理提猫直播助手的版本号，以及版本发布流程。

## 🔢 版本号格式

我们使用[语义化版本号](https://semver.org/lang/zh-CN/)：`主版本号.次版本号.修订号`

```
1.2.3
│ │ │
│ │ └─ 修订号 (PATCH)：修复bug，向下兼容
│ └─── 次版本号 (MINOR)：新功能，向下兼容  
└───── 主版本号 (MAJOR)：不兼容的API修改
```

### 版本号规则

- **1.0.0 → 1.0.1**: 修复bug（如：修复崩溃、修复UI错误）
- **1.0.0 → 1.1.0**: 新增功能（如：新增AI模型、新增UI功能）
- **1.0.0 → 2.0.0**: 重大更新（如：重构架构、API不兼容）

## 📝 修改版本号

### 需要修改的文件

版本号存储在以下两个文件中，**必须同时修改**：

#### 1. 根目录 package.json

**文件路径**: `/package.json`  
**位置**: 第4行

```json
{
  "$schema": "./schemas/package.schema.json",
  "name": "timao-douyin-live-manager",
  "version": "1.1.0",  ← 修改这里
  "description": "抖音直播评论实时分析与AI话术生成工具"
}
```

#### 2. Electron应用 package.json

**文件路径**: `/electron/package.json`  
**位置**: 第3行

```json
{
  "name": "ast-voice-transcription-app",
  "version": "1.1.0",  ← 修改这里
  "description": "语音转录服务桌面应用"
}
```

### 快速修改方法

#### 方法1：手动修改（推荐）

1. 打开 `package.json` 文件
2. 找到 `"version": "1.0.0"` 行
3. 修改为目标版本号，如 `"version": "1.1.0"`
4. 重复步骤1-3修改 `electron/package.json`

#### 方法2：使用 npm version 命令

```bash
# 自动递增修订号 (1.0.0 → 1.0.1)
npm version patch

# 自动递增次版本号 (1.0.0 → 1.1.0)
npm version minor

# 自动递增主版本号 (1.0.0 → 2.0.0)
npm version major

# 指定版本号
npm version 1.2.3
```

⚠️ **注意**: npm version 命令只会修改根目录的 `package.json`，还需要手动修改 `electron/package.json`

#### 方法3：使用查找替换

在编辑器中：
1. 全局搜索: `"version": "1.0.0"`
2. 替换为: `"version": "1.1.0"`
3. 仅替换 `package.json` 文件中的（忽略 `package-lock.json` 和 `node_modules`）

## 📦 版本发布流程

### 完整发布流程

```bash
# 1. 更新版本号
# 手动修改 package.json 和 electron/package.json

# 2. 提交版本更新
git add package.json electron/package.json
git commit -m "chore: 发布版本 v1.1.0"

# 3. 创建版本标签
git tag -a v1.1.0 -m "版本 1.1.0 - 新增XX功能"

# 4. 推送到远程仓库
git push origin main
git push origin v1.1.0

# 5. 打包应用
npm run package:windows

# 6. 发布
# 将 dist/ 目录下的文件上传至发布平台
```

## 📂 打包文件命名

版本号会自动包含在打包后的文件名中：

```
TalkingCat-Portable-{version}-{arch}.exe
TalkingCat-Setup-{version}-{arch}.exe
```

**示例**:
- 版本 1.0.0: `TalkingCat-Portable-1.0.0-x64.exe`
- 版本 1.1.0: `TalkingCat-Portable-1.1.0-x64.exe`
- 版本 2.0.0: `TalkingCat-Portable-2.0.0-x64.exe`

## 📋 版本发布清单

每次发布新版本时，请确保完成以下任务：

- [ ] 更新 `package.json` 版本号
- [ ] 更新 `electron/package.json` 版本号
- [ ] 编写 CHANGELOG（更新日志）
- [ ] 运行测试确保功能正常
- [ ] 本地打包测试
- [ ] Git 提交并打标签
- [ ] 打包生产版本
- [ ] 上传至发布平台
- [ ] 发布公告

## 🔍 版本号验证

### 检查当前版本

```bash
# 查看根目录版本
npm version

# 查看 package.json 版本
grep '"version"' package.json

# 查看 electron/package.json 版本
grep '"version"' electron/package.json
```

### 验证版本一致性

确保两个文件的版本号一致：

```bash
# Linux/Mac
diff <(grep '"version"' package.json) <(grep '"version"' electron/package.json)

# Windows PowerShell
(Select-String -Path package.json -Pattern '"version"').Line
(Select-String -Path electron/package.json -Pattern '"version"').Line
```

## 🎯 版本策略建议

### 开发阶段

- 使用 `0.x.x` 版本号
- 快速迭代，频繁发布

### 稳定版本

- 使用 `1.x.x` 及以上
- 遵循语义化版本规范
- 维护 CHANGELOG

### 测试版本

- 使用预发布标识：`1.0.0-beta.1`, `1.0.0-rc.1`
- 用于内部测试或公开测试

## ⚠️ 注意事项

1. **版本号必须一致**: 确保两个 `package.json` 文件中的版本号相同
2. **不要手动修改 package-lock.json**: 版本号会自动更新
3. **打包前检查**: 确认版本号已正确修改
4. **Git 标签**: 使用 `v` 前缀创建版本标签（如 `v1.1.0`）
5. **向后兼容**: 次版本号和修订号更新应保持向后兼容

## 🛠️ 工具推荐

### 自动化版本管理工具

- **standard-version**: 自动生成 CHANGELOG 和版本号
- **semantic-release**: 完全自动化的版本发布
- **release-it**: 交互式版本发布工具

### 安装 standard-version（可选）

```bash
npm install --save-dev standard-version

# package.json 中添加脚本
"scripts": {
  "release": "standard-version"
}

# 使用
npm run release
```
