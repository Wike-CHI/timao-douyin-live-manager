# 内存优化部署指南

审查人：叶维哲

## 概述

本次内存优化修复了前端Electron应用接收转写后自动停止的问题，该问题由服务器内存溢出导致PM2自动重启引起。

### 核心改进

1. **PM2内存限制提升**：从3GB提升到4.5GB
2. **AI分析服务优化**：限制user_scores数量，添加主动GC
3. **转写服务优化**：定期垃圾回收机制
4. **录制服务优化**：限制comments和analysis数据量
5. **前端优化**：限制log和aiEvents条数
6. **内存监控服务**：实时监控，主动预警和清理
7. **PM2重启策略优化**：减少频繁重启

## 故障排查

### 问题1：服务无法启动

**症状**：PM2显示服务状态为 `errored` 或 `stopped`

**排查步骤**：

1. 查看错误日志：
```bash
pm2 logs timao-server --err --lines 100
```

2. 检查是否有Python导入错误
3. 验证所有依赖是否已安装：
```bash
cd /www/wwwroot/wwwroot/timao-douyin-live-manager
pip list | grep psutil  # 确认psutil已安装
```

**解决方案**：
```bash
# 如果缺少psutil
pip install psutil

# 重启服务
pm2 restart timao-server
```

### 问题2：内存监控不生效

**症状**：日志中没有内存监控相关信息

**排查步骤**：

1. 检查启动日志：
```bash
pm2 logs timao-server | grep "内存监控"
```

2. 手动调用API：
```bash
curl http://localhost:8081/api/model/memory-status
```

**解决方案**：
- 确认 `psutil` 已安装
- 检查 `main.py` 中的启动代码
- 查看是否有异常被捕获

### 问题3：转写仍然中断

**症状**：运行一段时间后转写停止

**排查步骤**：

1. 检查PM2重启次数：
```bash
pm2 show timao-server | grep restart
```

2. 查看内存使用：
```bash
pm2 show timao-server | grep memory
```

3. 检查日志中的内存警告

**解决方案**：
- 如果内存仍超4.5GB，考虑进一步提升限制或减少同时运行的服务
- 检查是否有内存泄漏
- 联系开发者进一步排查

## 性能指标

### 正常运行指标

| 指标 | 优化前 | 优化后 | 改善 |
|------|--------|--------|------|
| 内存限制 | 3GB | 4.5GB | +50% |
| 平均内存 | 2.8GB | 2.8-3.8GB | 稳定 |
| PM2重启频率 | 频繁 | 0次 | ✅ |
| 转写持续时间 | <10分钟 | >2小时 | ✅ |
| GC频率 | 被动 | 主动 | ✅ |

### 内存使用阶段

- **启动阶段**（0-2分钟）：2.4-2.8GB
  - SenseVoice模型加载：~2.3GB
  - VAD模型加载：~140MB
  - 基础运行内存：~200MB

- **稳定运行**（2分钟-2小时）：2.8-3.5GB
  - 模型内存：2.44GB（固定）
  - 运行数据：400-1000MB（波动）

- **高负载**（高人气直播）：3.5-4.2GB
  - 模型内存：2.44GB
  - 大量评论和转写数据：1-1.5GB

## 后续优化建议

如果长期运行后仍发现问题，可以考虑：

1. **服务拆分**：将AI分析服务独立部署
2. **数据库优化**：使用Redis缓存减少内存占用
3. **模型优化**：使用更小的模型或量化版本
4. **限流策略**：限制高频请求的数据累积
5. **定期重启**：使用cron定期重启服务（如每天凌晨）
