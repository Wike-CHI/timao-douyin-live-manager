# 请求清理机制验证指南

## 快速验证步骤

### 1. 启动开发环境

```bash
cd /www/wwwroot/wwwroot/timao-douyin-live-manager/electron/renderer
npm run dev
```

同时在另一个终端启动 Electron:

```bash
cd /www/wwwroot/wwwroot/timao-douyin-live-manager/electron
npm run dev
```

### 2. 打开开发者工具

在 Electron 应用中按 `F12` 或 `Ctrl+Shift+I` 打开开发者工具。

### 3. 观察初始化日志

在控制台中你应该看到:

```
🔧 初始化应用清理服务...
✅ Electron IPC 清理监听器已设置
✅ 浏览器 beforeunload 清理监听器已设置
✅ 应用清理服务初始化完成
🔍 启动服务健康检查...
```

### 4. 进行一些操作

- 导航到不同页面 (仪表盘、报告、设置等)
- 触发一些 API 请求
- 等待几秒让健康检查定时器运行

### 5. 关闭应用

点击窗口关闭按钮或按 `Alt+F4`。

### 6. 检查主进程日志

在启动 Electron 的终端中,你应该看到:

```
[electron] App is about to quit. Cleaning up resources...
[electron] Notifying 1 window(s) to cleanup...
[electron] Cleanup signal sent to window
[electron] Waiting for renderer cleanup (500ms)...
[electron] Renderer cleanup wait completed
[electron] Backend services stopped
```

### 7. 检查渲染进程日志

在关闭前的最后几秒,开发者工具控制台应该显示:

```
📡 收到主进程清理信号,开始清理资源...
🧹 开始清理所有资源...
📡 取消 X 个正在进行的请求...
⏱️  清除 X 个定时器...
🔄 清除 X 个轮询定时器...
🔧 执行 X 个清理回调...
🛑 服务健康检查已停止
✅ 资源清理完成
✅ 应用资源清理完成
```

## 高级验证

### 验证请求取消

1. **在 Network 面板中观察请求**
   - 打开开发者工具的 Network 标签
   - 筛选 XHR/Fetch 请求
   - 关闭应用
   - 确认所有 pending 的请求状态变为 "canceled"

2. **使用代码验证**
   ```typescript
   // 在控制台中执行
   import { requestManager } from './services/requestManager';
   
   // 创建一个长时间运行的请求
   const controller = requestManager.createAbortController();
   fetch('http://129.211.218.135/health', { signal: controller.signal })
     .then(() => console.log('请求完成'))
     .catch(err => console.log('请求被取消:', err.name));
   
   // 立即关闭应用,观察请求是否被取消
   ```

### 验证定时器清除

1. **在控制台中手动创建定时器**
   ```javascript
   import { requestManager } from './services/requestManager';
   
   // 创建一个轮询定时器
   let count = 0;
   const intervalId = requestManager.createInterval(() => {
     count++;
     console.log(`轮询执行第 ${count} 次`);
   }, 1000);
   
   // 等待几秒后关闭应用
   // 确认定时器被清除 (count 不再增加)
   ```

2. **检查定时器状态**
   ```javascript
   // 在清理前检查
   console.log('是否正在清理:', requestManager.isCleaningUp());
   // 输出: false
   
   // 关闭应用后,这个值会变为 true
   ```

### 验证服务器资源释放

1. **监控后端请求日志**

在服务器端 (如果有访问权限),监控请求日志:

```bash
# 查看 PM2 日志
pm2 logs timao-backend --lines 50

# 或者查看 Nginx 日志
tail -f /var/log/nginx/access.log
```

2. **确认行为**:
   - 应用运行时: 每 30 秒有健康检查请求
   - 应用关闭后: 健康检查请求立即停止
   - 没有 "hanging" 的请求 (长时间无响应的请求)

## 性能指标

### 预期指标

- **清理执行时间**: < 100ms
- **请求取消延迟**: < 10ms  
- **定时器清除数量**: 通常 3-10 个 (取决于打开的页面)
- **请求取消数量**: 通常 0-5 个 (取决于请求时机)

### 测量方法

在 `requestManager.ts` 的 `cleanup()` 方法开始和结束时添加性能标记:

```typescript
public cleanup(): void {
  const startTime = performance.now();
  
  // ... 清理逻辑 ...
  
  const endTime = performance.now();
  console.log(`✅ 资源清理完成 (耗时: ${(endTime - startTime).toFixed(2)}ms)`);
}
```

## 故障排查

### 问题: 看不到清理日志

**可能原因**:
1. 开发者工具在应用关闭后关闭,日志丢失
2. 日志级别被过滤

**解决方案**:
- 在应用关闭前截图保存日志
- 使用主进程日志 (在终端中查看)
- 添加持久化日志记录

### 问题: 请求没有被取消

**诊断步骤**:
1. 检查请求是否使用了 `fetchJsonWithAuth`
2. 在 Network 面板中查看请求状态
3. 添加断点到 `requestManager.cleanup()`

**修复建议**:
```typescript
// 确保所有请求都使用正确的方法
import { fetchJsonWithAuth } from './services/http';
const response = await fetchJsonWithAuth('main', '/api/data');
```

### 问题: 定时器没有被清除

**诊断步骤**:
1. 检查是否使用了 `requestManager.createInterval`
2. 在 `cleanup()` 方法中添加日志

**修复建议**:
```typescript
// 使用追踪的定时器
import { requestManager } from './services/requestManager';
const intervalId = requestManager.createInterval(() => {...}, 5000);
```

## 自动化测试 (未来工作)

### 单元测试示例

```typescript
import { requestManager } from './requestManager';

describe('RequestManager', () => {
  beforeEach(() => {
    requestManager.reset();
  });
  
  test('should track and cancel abort controllers', () => {
    const controller = requestManager.createAbortController();
    expect(controller.signal.aborted).toBe(false);
    
    requestManager.cleanup();
    expect(controller.signal.aborted).toBe(true);
  });
  
  test('should clear timers', () => {
    let executed = false;
    requestManager.createTimeout(() => {
      executed = true;
    }, 1000);
    
    requestManager.cleanup();
    
    // 等待超过定时器时间
    await new Promise(resolve => setTimeout(resolve, 1500));
    expect(executed).toBe(false);
  });
});
```
