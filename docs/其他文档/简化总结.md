## 🔄 请求清理机制（保留并优化）

当用户关闭应用时：

```
1. Electron 主进程触发 'before-quit'
   ↓
2. 发送清理信号到渲染进程
   ↓
3. 渲染进程执行清理:
   - 取消所有 HTTP 请求
   - 清除所有定时器
   ↓
4. 等待 500ms 确保清理完成
   ↓
5. 应用正常退出
```

**效果**: 避免服务器继续处理已关闭客户端的请求

## 💡 关键决策

### 为什么不在本地启动后端？

1. **不必要的复杂性**
   - 增加打包体积
   - 增加启动时间
   - 增加维护成本

2. **用户体验差**
   - 需要安装 Python
   - 需要配置数据库
   - 占用更多资源

3. **已有更好的方案**
   - 后端已部署到公网
   - 用户可以直接连接
   - 统一管理、统一升级

### 为什么保留请求清理机制？

1. **资源管理很重要**
   - 避免服务器资源浪费
   - 防止后端继续处理无效请求
   - 提供更好的用户体验

2. **实现成本低**
   - 仅增加 ~300 行代码
   - 不影响应用体积
   - 不影响启动速度

3. **适用范围广**
   - 适用于所有 HTTP 请求
   - 适用于所有定时器
   - 自动追踪、自动清理

## 🚀 使用指南

### 开发模式

```bash
# 1. 启动前端开发服务器
cd electron/renderer
npm run dev

# 2. 启动 Electron（新终端）
cd electron
npm run dev
```

### 生产打包

```bash
# 使用简化的打包脚本
cd /www/wwwroot/wwwroot/timao-douyin-live-manager
./scripts/build-electron-production.sh
```

### 配置公网 IP

如果需要更改后端地址，修改 `electron/renderer/.env.production`:

```bash
VITE_FASTAPI_URL=http://你的公网IP
VITE_STREAMCAP_URL=http://你的公网IP
VITE_DOUYIN_URL=http://你的公网IP
```
