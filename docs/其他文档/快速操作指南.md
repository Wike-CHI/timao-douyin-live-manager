## 📋 操作步骤

### 第1步：重启后端服务 ⚡

**方法1：使用脚本（推荐）**
```bash
cd /www/wwwroot/wwwroot/timao-douyin-live-manager
./scripts/restart-backend.sh
```

**方法2：手动命令**
```bash
cd /www/wwwroot
pm2 restart timao-backend
pm2 logs timao-backend --lines 20
```

**预期输出**：
```
✅ 服务已重启
📋 显示最近20行日志：
...
INFO:     Application startup complete.
```

### 第2步：测试JWT验证 🧪

**1. 登录测试**
```bash
curl -X POST http://127.0.0.1:11111/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username_or_email":"dev_admin","password":"DevAdmin@2024!"}'
```

**预期响应**：
```json
{
  "success": true,
  "token": "eyJ...",
  "user": {...}
}
```

**2. API测试（使用token）**
```bash
# 替换<token>为上一步获取的token
curl http://127.0.0.1:11111/api/live/sessions \
  -H "Authorization: Bearer <token>"
```

3. 清空LocalStorage和SessionStorage
4. 或者使用无痕模式访问

## 🚨 故障排查

### 问题1：服务启动失败

**症状**：pm2 restart后服务立即停止

**排查**：
```bash
pm2 logs timao-backend --lines 50
```

**常见原因**：
- 语法错误：检查auth.py是否有错误
- 导入错误：检查logger是否正确导入

### 问题2：登录失败（401）

**症状**：curl登录返回401错误

**排查**：
```bash
# 查看后端日志
pm2 logs timao-backend

# 检查JWT key是否正确
grep SECRET_KEY /www/wwwroot/wwwroot/timao-douyin-live-manager/server/.env
```

**解决**：
- 确保SECRET_KEY配置正确
- 确保用户名密码正确

### 问题3：前端无法连接

**症状**：前端显示网络错误

**排查**：
```bash
# 检查后端是否运行
pm2 status timao-backend

# 检查端口是否监听
netstat -tulnp | grep 11111
```

**解决**：
- 确保后端正在运行
- 确保防火墙允许11111端口

### 问题4：数据流断开

**症状**：AI分析数据不完整，转写记录为0

**排查**：
```bash
# 查看后端日志中的JWT验证
pm2 logs timao-backend | grep "JWT验证"

# 检查WebSocket连接
pm2 logs timao-backend | grep "WebSocket"
```

**解决**：
- 确保token正确传递
- 检查SSE和WebSocket的query参数
