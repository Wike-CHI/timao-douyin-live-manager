# 请求清理机制 - 快速参考

## 🚀 快速开始

### 使用追踪的 HTTP 请求

```typescript
import { fetchJsonWithAuth } from './services/http';

// ✅ 推荐 - 自动追踪,应用关闭时自动取消
const response = await fetchJsonWithAuth('main', '/api/users');
```

### 使用追踪的定时器

```typescript
import { requestManager } from './services/requestManager';

// ✅ 推荐 - 自动追踪,应用关闭时自动清除
const timerId = requestManager.createTimeout(() => {
  console.log('延时任务');
}, 5000);

const intervalId = requestManager.createInterval(() => {
  console.log('轮询任务');
}, 10000);

// 手动清除 (可选)
requestManager.clearTimeout(timerId);
requestManager.clearInterval(intervalId);
```

### 注册自定义清理逻辑

```typescript
import { requestManager } from './services/requestManager';

// 应用关闭时会自动执行
requestManager.registerCleanup(() => {
  console.log('清理我的资源');
  // 关闭 WebSocket、清理缓存等
});
```

## ❌ 避免的做法

```typescript
// ❌ 不推荐 - 不会被自动清理
const timerId = setTimeout(() => {...}, 5000);
const intervalId = setInterval(() => {...}, 10000);

// ❌ 不推荐 - 请求不会被自动取消
fetch('http://api.example.com/data')
  .then(response => response.json());
```

## 📦 核心 API

### RequestManager

```typescript
// 创建资源
createAbortController(): AbortController
createTimeout(callback: () => void, delay: number): NodeJS.Timeout | number
createInterval(callback: () => void, delay: number): NodeJS.Timeout | number

// 清除资源
clearTimeout(timer: NodeJS.Timeout | number): void
clearInterval(interval: NodeJS.Timeout | number): void

// 注册清理
registerCleanup(callback: () => void): void
unregisterCleanup(callback: () => void): void

// 执行清理
cleanup(): void

// 检查状态
isCleaningUp(): boolean
```

## 🎯 React 组件示例

```tsx
import { useEffect, useRef } from 'react';
import { requestManager } from '../services/requestManager';

function MyComponent() {
  const intervalRef = useRef<number | null>(null);
  
  useEffect(() => {
    // 创建轮询
    intervalRef.current = requestManager.createInterval(() => {
      fetchData();
    }, 5000);
    
    // 组件卸载时清除
    return () => {
      if (intervalRef.current !== null) {
        requestManager.clearInterval(intervalRef.current);
      }
    };
  }, []);
  
  return <div>My Component</div>;
}
```

## 🐛 故障排查

### 问题: 请求没有被取消

**检查**:
- 是否使用了 `fetchJsonWithAuth`?
- 是否手动创建了 `AbortController` 但没有注册?

**解决**:
```typescript
// 使用推荐的方法
import { fetchJsonWithAuth } from './services/http';
const response = await fetchJsonWithAuth('main', '/api/data');
```

### 问题: 定时器没有被清除

**检查**:
- 是否使用了 `requestManager.createInterval`?
- 是否直接使用了原生 `setInterval`?

**解决**:
```typescript
// 使用追踪的定时器
import { requestManager } from './services/requestManager';
const intervalId = requestManager.createInterval(() => {...}, 5000);
```

### 问题: 看不到清理日志

**检查**:
- 开发者工具是否打开?
- 是否在应用关闭后立即关闭了终端?

**解决**:
- 在关闭前截图保存日志
- 查看主进程日志 (在启动 Electron 的终端中)
