## 📊 方案对比：本地部署 vs 云端API

### 方案1: 本地部署SenseVoice（不推荐当前配置）

**优点**：
- ✅ 数据不出服务器（隐私更好）
- ✅ 无API调用成本（按转写次数）
- ✅ 延迟稍低（本地处理）

**缺点**：
- ❌ **需要16GB内存**（当前8GB不足）
- ❌ CPU推理慢（无GPU）
- ❌ 服务器成本增加 ￥200-300/月
- ❌ 稳定性风险（内存不足可能OOM）

**资源使用模拟**：
```
正常运行:
├─ 系统+服务: 4.4GB
├─ SenseVoice: 2.5GB (平均)
└─ 总计: ~7GB / 8GB (87%使用率) ← 危险区域

峰值情况:
├─ 系统+服务: 4.4GB
├─ SenseVoice: 4GB (峰值)
└─ 总计: ~8.4GB / 8GB (105%) ← OOM崩溃！
```

## 🔧 快速集成指南

### 1. 讯飞API集成（Python）
```python
# 安装SDK
pip install websocket-client

# server/app/services/xfyun_transcription.py
import websocket
import json
import base64

class XfyunTranscription:
    def __init__(self, app_id, api_key, api_secret):
        self.app_id = app_id
        self.api_key = api_key
        self.api_secret = api_secret
        
    async def transcribe(self, audio_data: bytes) -> str:
        # 1. 音频编码
        audio_base64 = base64.b64encode(audio_data).decode()
        
        # 2. 构建请求
        ws_url = self._get_ws_url()
        
        # 3. WebSocket连接发送音频
        result = await self._send_audio(ws_url, audio_base64)
        
        return result
```
