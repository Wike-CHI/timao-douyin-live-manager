**存储位置**：**始终在服务器**

## 🔍 代码验证

### 复盘服务读取数据的代码

**文件**：`server/app/services/live_review_service.py`

```python
def _load_session_data(self, session: LiveSession) -> Dict[str, Any]:
    """加载直播数据文件
    
    从 records/live_logs/ 目录加载弹幕、转写、事件数据
    """
    data = {
        "comments": [],
        "transcript": "",
        "events": []
    }
    
    # ✅ 读取弹幕文件（文本）
    if session.comment_file and Path(session.comment_file).exists():
        with open(session.comment_file, 'r', encoding='utf-8') as f:
            data["comments"] = [json.loads(line) for line in f]
    
    # ✅ 读取转写文件（文本）
    if session.transcript_file and Path(session.transcript_file).exists():
        with open(session.transcript_file, 'r', encoding='utf-8') as f:
            data["transcript"] = f.read()
    
    # ❌ 不读取音频文件（live_audio.wav）
    # 音频文件不在复盘流程中使用
    
    return data
```

**关键发现**：
- ✅ 读取 `session.transcript_file`（转写文本）
- ✅ 读取 `session.comment_file`（弹幕文本）
- ❌ **不读取** `live_audio.wav`（音频文件）

## ✅ 为什么不需要改路径？

### 1. 转写文本不受影响

```
实时监听 → 服务器转写 → 保存文本（服务器）
                              ↑
                         始终在这里
                         
音频文件（方案A或B）→ 归档（客户端或服务器）
                    ↑
               不影响复盘
```

### 2. 音频文件只是副产品

**音频文件的用途**：
- 📦 **归档**：留作记录
- 🎵 **手动播放**：用户想听回放
- 🔄 **重新转写**（可选功能，需额外开发）

**复盘不需要音频文件**：
- 已有实时转写文本
- 转写质量由讯飞ASR保证
- 无需重新转写

## 🔮 未来场景：从音频重新转写

### 场景描述

如果实时转写出错或质量不佳，用户想**从本地音频重新转写**

### 需要新增功能

#### 选项1：上传到服务器转写

```javascript
// 1. 上传音频文件
const uploadResult = await window.electron.uploadAudioFile(localPath);

// 2. 服务器端转写
const transcriptResult = await fetch('/api/audio/transcribe', {
    method: 'POST',
    body: JSON.stringify({
        audio_path: uploadResult.serverPath,
        session_id: sessionId
    })
});

// 3. 更新转写文本
await fetch(`/api/sessions/${sessionId}/transcript`, {
    method: 'PUT',
    body: JSON.stringify({
        transcript: transcriptResult.text
    })
});
```

**优点**：
- ✅ 服务器有ASR引擎
- ✅ 转写质量有保证

**缺点**：
- ⚠️ 需要上传（带宽占用）
- ⚠️ 音频文件可能很大

---

#### 选项2：客户端本地转写

```javascript
// 客户端集成ASR（复杂）
const transcriptResult = await window.electron.transcribeLocal(localPath);
```

**优点**：
- ✅ 不需要上传

**缺点**：
- ❌ Electron打包ASR引擎（复杂）
- ❌ 或者调用云端API（成本）

## ✅ 最终结论

### 当前复盘功能：✅ 不需要改路径

**原因**：
- 复盘读取的是转写文本（`transcript_file`）
- 转写文本由服务器实时生成并保存
- 音频文件（无论在服务器还是客户端）不影响复盘

**数据依赖**：
```
复盘依赖
    ├─ transcript_file（转写文本）✅ 在服务器
    ├─ comment_file（弹幕文本）✅ 在服务器
    └─ live_audio.wav（音频文件）❌ 不需要
```

---

## 📌 记住

**音频文件 ≠ 转写文本**

- **音频文件**：原始录音，用于归档和播放
- **转写文本**：ASR实时生成的文本，用于复盘分析

**方案B只改变音频文件位置，不改变转写流程**

- ✅ 复盘读取转写文本，不读音频
