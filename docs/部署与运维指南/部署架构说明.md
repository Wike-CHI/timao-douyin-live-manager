# 提猫直播助手 - 部署架构说明

## 📁 当前部署方式：就地部署（In-Place Deployment）

### 核心原理

**不复制文件**，直接在Git项目目录运行服务。

```
项目目录（Git仓库）
/www/wwwroot/wwwroot/timao-douyin-live-manager/
    ↓
直接运行 Python 代码
    ↓
systemd 管理进程
```

## 🔄 数据流

### 1. 客户端请求流程
```
Electron App (Windows)
    ↓ HTTP Request
    ↓ http://129.211.218.135/api/xxx
Nginx (Port 80)
    ↓ proxy_pass
    ↓ http://127.0.0.1:8181/api/xxx
Uvicorn/FastAPI (Port 8181)
    ↓ import server.app.main
Python Code (项目目录/server/)
    ↓ 执行业务逻辑
    ↓ 返回 JSON
```

### 2. 代码更新流程
```
开发者修改代码
    ↓
git push 到远程仓库
    ↓
服务器: git pull
    ↓
systemctl restart talkingcat-backend
    ↓
新代码立即生效（所有客户端自动连接到新后端）
```

## 🎯 为什么选择就地部署？

### ✅ 优点

1. **部署简单**
   - 不需要复制文件
   - 不需要打包构建
   - 直接运行源代码

2. **更新方便**
   ```bash
   git pull                          # 拉取最新代码
   systemctl restart talkingcat-backend  # 重启服务
   # 完成！
   ```

3. **开发友好**
   - 可以直接在服务器上修改调试
   - 不需要额外的构建步骤
   - Git 历史完整保留

4. **Python 特性**
   - Python 是解释型语言，不需要编译
   - 虚拟环境隔离依赖
   - 代码可以直接执行

### ❌ 缺点

1. **Git 仓库在生产环境**
   - `.git` 文件夹占用空间
   - 可能包含开发文件

2. **权限管理**
   - 开发文件和运行文件在同一目录
   - 需要注意文件权限

3. **版本管理**
   - 没有明确的"发布版本"
   - 依赖 Git commit

## 🆚 对比其他部署方式

### 方式1: 就地部署（当前）

```
/www/wwwroot/wwwroot/timao-douyin-live-manager/
    ↓ 直接运行
systemd → uvicorn → server/app/main.py
```

**适用场景**：
- 快速迭代开发
- 单服务器部署
- Python Web 应用

---

### 方式2: 复制部署（传统）

```
项目目录（开发）          复制发布目录（生产）
/home/dev/project/  →   /opt/app/releases/v1.0.0/
                    →   /opt/app/current/ → symlink to releases/v1.0.0/
```

**优点**：
- 明确的版本管理
- 可以快速回滚（切换 symlink）
- 生产环境不包含 Git 仓库

**缺点**：
- 需要额外的部署脚本
- 占用更多磁盘空间
- 更新流程复杂

**实现示例**：
```bash
# 部署脚本
VERSION=v1.0.0
rsync -av --exclude='.git' /home/dev/project/ /opt/app/releases/$VERSION/
ln -sfn /opt/app/releases/$VERSION /opt/app/current
systemctl restart app
```

---

### 方式3: Docker 容器化

```
Dockerfile
    ↓ docker build
Docker Image
    ↓ docker run
Container (隔离环境)
```

**优点**：
- 环境完全隔离
- 跨平台一致性
- 易于扩展（多容器）

**缺点**：
- 需要 Docker 知识
- 资源开销稍大
- 构建时间较长

**实现示例**：
```dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY server/ ./server/
CMD ["uvicorn", "server.app.main:app", "--host", "0.0.0.0", "--port", "8181"]
```

---

### 方式4: 编译打包（适用于编译型语言）

```
源代码
    ↓ 编译/打包
二进制文件 / JAR / 可执行文件
    ↓ 部署
生产服务器
```

**适用语言**：Go, Java, C++, Rust

**Python 不适用**：Python 是解释型语言，不需要编译

## 🔧 当前部署的关键组件

### 1. systemd 服务配置
**文件**: `/etc/systemd/system/talkingcat-backend.service`

```ini
[Service]
Type=simple
User=root
WorkingDirectory=/www/wwwroot/wwwroot/timao-douyin-live-manager  ← 项目目录
ExecStart=/www/wwwroot/wwwroot/timao-douyin-live-manager/.venv/bin/python -m uvicorn server.app.main:app --host 0.0.0.0 --port 8181
                                                          ↑ 使用项目内的虚拟环境
Restart=always         ← 自动重启
MemoryMax=2G          ← 内存限制
```

### 2. Nginx 反向代理
**文件**: `/www/server/panel/vhost/nginx/129.211.218.135.conf`

```nginx
location /api/ {
    proxy_pass http://127.0.0.1:8181/api/;  ← 转发到本地 Python 进程
}
```

### 3. Python 虚拟环境
**位置**: `/www/wwwroot/wwwroot/timao-douyin-live-manager/.venv/`

- 隔离 Python 依赖
- 不影响系统 Python
- 包含所有项目依赖包

### 4. 环境变量
**文件**: `server/.env`

```env
AI_SERVICE=qwen
AI_API_KEY=sk-xxx
MYSQL_HOST=xxx.mysql.rds.aliyuncs.com
REDIS_HOST=localhost
```

## 🚀 如何切换到其他部署方式？

### 切换到复制部署

```bash
# 1. 创建发布目录
mkdir -p /opt/talkingcat/releases

# 2. 复制项目（排除 .git）
rsync -av --exclude='.git' --exclude='node_modules' \
    /www/wwwroot/wwwroot/timao-douyin-live-manager/ \
    /opt/talkingcat/releases/v1.0.0/

# 3. 创建虚拟环境
cd /opt/talkingcat/releases/v1.0.0
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.all.txt

# 4. 创建软链接
ln -sfn /opt/talkingcat/releases/v1.0.0 /opt/talkingcat/current

# 5. 修改 systemd 配置
vim /etc/systemd/system/talkingcat-backend.service
# WorkingDirectory=/opt/talkingcat/current
# ExecStart=/opt/talkingcat/current/.venv/bin/python -m uvicorn ...

# 6. 重启服务
systemctl daemon-reload
systemctl restart talkingcat-backend
```

### 切换到 Docker 部署

```bash
# 1. 创建 Dockerfile
cat > Dockerfile << 'EOF'
FROM python:3.11-slim
WORKDIR /app
COPY requirements.all.txt .
RUN pip install -r requirements.all.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
COPY server/ ./server/
COPY config/ ./config/
ENV PYTHONPATH=/app
CMD ["uvicorn", "server.app.main:app", "--host", "0.0.0.0", "--port", "8181"]
EOF

# 2. 构建镜像
docker build -t talkingcat-backend:v1.0.0 .

# 3. 运行容器
docker run -d \
    --name talkingcat-backend \
    -p 8181:8181 \
    --env-file server/.env \
    --restart always \
    talkingcat-backend:v1.0.0

# 4. Nginx 配置保持不变（仍然代理到 8181）
```

## 💡 推荐场景

### 当前方式适合：
✅ 单服务器部署  
✅ 快速迭代开发  
✅ Python Web 应用  
✅ 团队规模小（1-3人）  
✅ 不需要复杂版本管理  

### 建议切换到其他方式：
❌ 多服务器集群部署 → Docker + Kubernetes  
❌ 需要严格版本控制 → 复制部署  
❌ 多环境管理（开发/测试/生产） → Docker  
❌ 需要快速回滚 → 复制部署 + Symlink
