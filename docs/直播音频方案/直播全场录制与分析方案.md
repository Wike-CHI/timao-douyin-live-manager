# 直播全场录制与分析解决方案

## 📋 问题分析

### 核心问题：抖音直播流 URL 过期限制

**真实原因**：
- ⚠️ **抖音平台限制**：通过 `streamget` 库获取的直播流 URL 存在时效性
- ⚠️ **URL 有效期**：抖音的 FLV/M3U8 流地址通常在 **30-60 分钟** 后失效
- ⚠️ **防盗链机制**：URL 中包含时间戳和签名参数，过期后无法继续播放
- ⚠️ **非系统内部限制**：这是抖音 API 层面的限制，不是您代码的问题

**系统现状**：
```python
# live_report_service.py 中的录制流程
handler = get_platform_handler(live_url)  # streamget
info = await handler.get_stream_info(live_url)
record_url = info.record_url or info.flv_url  # 获取流地址

# 使用 ffmpeg 录制
await self._start_ffmpeg(record_url, seg_secs, pattern)
```

**问题表现**：
1. ❌ 录制 30 分钟后，流 URL 失效，ffmpeg 报错
2. ❌ 长直播（>30分钟）会中断录制
3. ❌ 无法获得完整录制数据
4. ❌ 报告不完整，数据缺失

---

## ✅ 解决方案

### 方案一：定时刷新流 URL（推荐 - 最稳定）

#### 原理
- 每隔 **20-25 分钟**自动重新获取新的流 URL
- 在旧 URL 失效前完成切换
- 保证录制的连续性
- 对用户完全透明

#### 架构设计

```
┌─────────────────────────────────────────────────────────┐
│  智能流 URL 刷新机制                                     │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  [开始录制]                                              │
│       ↓                                                  │
│  获取初始流 URL → 启动 ffmpeg                             │
│       ↓                                                  │
│  后台定时器 (每 20 分钟触发)                              │
│       ├─ 重新获取流 URL                                  │
│       ├─ 无缝切换 ffmpeg 进程                            │
│       └─ 保存当前分段                                    │
│       ↓                                                  │
│  持续监控 + 自动刷新                                      │
│       ├─ 检测直播是否结束                                │
│       ├─ 检测 ffmpeg 是否异常                            │
│       └─ 异常时自动重启                                  │
│       ↓                                                  │
│  [用户点击停止] 或 [直播结束]                             │
│       ↓                                                  │
│  汇总所有分段 → 生成完整报告                              │
└─────────────────────────────────────────────────────────┘
```

```python
# server/app/services/live_report_service.py

class LiveReportService:
    def __init__(self, records_root: Optional[str] = None) -> None:
        # ... 现有代码 ...
        self._url_refresh_task: Optional[asyncio.Task] = None  # 🆕 URL 刷新任务
        self._url_refresh_interval = 20 * 60  # 🆕 20分钟刷新一次
        self._current_record_url: Optional[str] = None  # 🆕 当前使用的流 URL
    
    async def start(self, live_url: str, segment_minutes: int = 30) -> LiveReportStatus:
        """
        开始录制直播
        
        Args:
            live_url: 直播间 URL (固定不变)
            segment_minutes: 分段时间(默认30分钟，用于文件切分)
        """
        if self._session is not None and self._ffmpeg_proc is not None:
            raise RuntimeError("Live report session already started")
        
        try:
            # 1. 获取初始流信息
            live_id = _parse_douyin_live_id(live_url)
            handler = get_platform_handler(live_url=live_url)
            if handler is None:
                raise RuntimeError("Unsupported live URL")
            
            # 获取第一次流 URL
            info = await handler.get_stream_info(live_url)
            record_url = self._extract_record_url(info)
            
            if not record_url:
                raise RuntimeError("Failed to resolve record URL (streams unavailable)")
            
            # 保存当前流 URL
            self._current_record_url = record_url
            
            # ... 创建 session ...
            anchor = _safe_name(info.get("anchor_name", ""))
            day = time.strftime("%Y-%m-%d", time.localtime())
            session_id = f"live_{platform}_{anchor}_{int(time.time())}"
            # ...
            
            self._session = LiveReportStatus(
                session_id=session_id,
                live_url=live_url,  # 🆕 保存原始直播间 URL，不是流 URL
                room_id=live_id,
                anchor_name=anchor,
                # ...
            )
            
            # 2. 启动 ffmpeg 录制
            seg_secs = max(300, int(segment_minutes) * 60)
            pattern = str(out_dir / f"{anchor}_%Y%m%d_%H%M%S_%03d.mp4")
            self._ffmpeg_proc = await self._start_ffmpeg(record_url, seg_secs, pattern)
            
            # 3. 🆕 启动 URL 自动刷新任务
            self._url_refresh_task = asyncio.create_task(self._auto_refresh_stream_url())
            
            # 4. 启动弹幕收集
            self._comment_task = asyncio.create_task(self._collect_comments())
            
            logger.info(f"✅ 录制已启动，将每 {self._url_refresh_interval // 60} 分钟自动刷新流地址")
            
            return self._session
            
        except Exception as e:
            logger.error(f"启动录制失败: {e}")
            raise
    
    async def _auto_refresh_stream_url(self):
        """
        🆕 自动刷新流 URL 的后台任务
        
        工作流程:
        1. 每 20 分钟触发一次
        2. 重新获取流 URL
        3. 无缝切换 ffmpeg 进程
        4. 保存当前分段
        """
        try:
            while self._session and self._ffmpeg_proc:
                # 等待 20 分钟
                await asyncio.sleep(self._url_refresh_interval)
                
                logger.info("🔄 开始刷新直播流 URL...")
                
                try:
                    # 1. 重新获取流地址
                    handler = get_platform_handler(live_url=self._session.live_url)
                    info = await handler.get_stream_info(self._session.live_url)
                    new_record_url = self._extract_record_url(info)
                    
                    if not new_record_url:
                        logger.warning("⚠️ 无法获取新的流地址，保持当前流")
                        continue
                    
                    # 2. 检查直播是否仍在进行
                    is_live = info.get("is_live") if isinstance(info, dict) else getattr(info, "is_live", True)
                    if not is_live:
                        logger.info("📴 检测到直播已结束，停止录制")
                        await self.stop()
                        break
                    
                    # 3. 切换到新的流 URL
                    logger.info(f"🔄 切换流地址: {new_record_url[:50]}...")
                    
                    # 3.1 优雅停止旧的 ffmpeg 进程
                    if self._ffmpeg_proc:
                        try:
                            await self._ffmpeg_proc.send_signal(signal.SIGTERM)
                            await asyncio.wait_for(self._ffmpeg_proc.wait(), timeout=5.0)
                        except asyncio.TimeoutError:
                            await self._ffmpeg_proc.kill()
                    
                    # 3.2 启动新的 ffmpeg 进程
                    seg_secs = self._session.segment_seconds
                    out_dir = Path(self._session.recording_dir)
                    anchor = self._session.anchor_name
                    pattern = str(out_dir / f"{anchor}_%Y%m%d_%H%M%S_%03d.mp4")
                    
                    self._ffmpeg_proc = await self._start_ffmpeg(new_record_url, seg_secs, pattern)
                    self._current_record_url = new_record_url
                    
                    logger.info("✅ 流地址刷新成功，录制继续")
                    
                except Exception as e:
                    logger.error(f"❌ 刷新流地址失败: {e}")
                    # 继续使用旧的流，等待下次刷新
                    continue
        
        except asyncio.CancelledError:
            logger.info("URL 刷新任务已取消")
        except Exception as e:
            logger.error(f"URL 刷新任务异常: {e}")
    
    def _extract_record_url(self, info) -> Optional[str]:
        """提取录制 URL（兼容 dict 和对象）"""
        if isinstance(info, dict):
            return info.get("record_url") or info.get("flv_url") or info.get("m3u8_url")
        else:
            return (
                getattr(info, "record_url", None)
                or getattr(info, "flv_url", None)
                or getattr(info, "m3u8_url", None)
            )
    
    async def stop(self) -> None:
        """停止录制"""
        if self._session is None:
            raise RuntimeError("No active session")
        
        logger.info("🛑 停止录制...")
        
        # 1. 🆕 取消 URL 刷新任务
        if self._url_refresh_task:
            self._url_refresh_task.cancel()
            try:
                await self._url_refresh_task
            except asyncio.CancelledError:
                pass
            self._url_refresh_task = None
        
        # 2. 停止 ffmpeg
        if self._ffmpeg_proc:
            try:
                await self._ffmpeg_proc.send_signal(signal.SIGTERM)
                await asyncio.wait_for(self._ffmpeg_proc.wait(), timeout=10.0)
            except asyncio.TimeoutError:
                await self._ffmpeg_proc.kill()
            self._ffmpeg_proc = None
        
        # 3. 停止弹幕收集
        if self._comment_task:
            self._comment_task.cancel()
            try:
                await self._comment_task
            except asyncio.CancelledError:
                pass
            self._comment_task = None
        
        # 4. 更新状态
        self._session.stopped_at = _now_ts()
        
        logger.info("✅ 录制已停止")
```

```python
# server/app/services/live_report_service.py

async def _start_ffmpeg(
    self, 
    stream_url: str, 
    segment_seconds: int, 
    output_pattern: str
) -> AsyncProcess:
    """
    启动 ffmpeg 进程录制直播流
    
    Args:
        stream_url: 直播流 URL
        segment_seconds: 分段时长(秒)
        output_pattern: 输出文件名模式
    
    Returns:
        AsyncProcess: ffmpeg 进程对象
    """
    cmd = [
        "ffmpeg",
        "-loglevel", "warning",
        "-y",  # 覆盖已存在的文件
        
        # 🆕 输入选项优化
        "-reconnect", "1",  # 自动重连
        "-reconnect_streamed", "1",  # 流式重连
        "-reconnect_delay_max", "5",  # 最大重连延迟 5 秒
        "-rw_timeout", "10000000",  # 读写超时 10 秒
        "-i", stream_url,
        
        # 编码选项（直接复制流，不重新编码）
        "-c", "copy",
        "-movflags", "faststart",
        
        # 🆕 分段选项
        "-f", "segment",
        "-segment_time", str(segment_seconds),
        "-segment_format", "mp4",
        "-reset_timestamps", "1",  # 重置时间戳
        "-strftime", "1",  # 启用时间格式化
        
        output_pattern
    ]
    
    logger.info(f"🎬 启动 ffmpeg: {' '.join(cmd[:10])}...")
    
    proc = await create_subprocess_exec(
        *cmd,
        stdout=asyncio.subprocess.PIPE,
        stderr=asyncio.subprocess.PIPE
    )
    
    return proc
```

```python
# server/app/services/live_report_service.py

async def _monitor_ffmpeg_health(self):
    """
    🆕 监控 ffmpeg 进程健康状态
    
    检测:
    1. 进程是否异常退出
    2. 输出文件是否在增长
    3. 网络是否断连
    """
    try:
        last_file_size = 0
        no_growth_count = 0
        
        while self._session and self._ffmpeg_proc:
            await asyncio.sleep(30)  # 每 30 秒检查一次
            
            # 1. 检查进程是否存活
            if self._ffmpeg_proc.returncode is not None:
                logger.error(f"❌ ffmpeg 进程异常退出，返回码: {self._ffmpeg_proc.returncode}")
                # 尝试重启
                await self._restart_ffmpeg()
                continue
            
            # 2. 检查文件是否在增长
            try:
                out_dir = Path(self._session.recording_dir)
                files = list(out_dir.glob("*.mp4"))
                if files:
                    latest_file = max(files, key=lambda f: f.stat().st_mtime)
                    current_size = latest_file.stat().st_size
                    
                    if current_size == last_file_size:
                        no_growth_count += 1
                        logger.warning(f"⚠️ 文件大小未增长 ({no_growth_count}/3)")
                        
                        if no_growth_count >= 3:
                            logger.error("❌ 文件长时间无增长，可能网络断连，尝试刷新流")
                            await self._restart_ffmpeg()
                            no_growth_count = 0
                    else:
                        no_growth_count = 0
                        last_file_size = current_size
            except Exception as e:
                logger.warning(f"健康检查失败: {e}")
    
    except asyncio.CancelledError:
        logger.info("健康监控任务已取消")
    except Exception as e:
        logger.error(f"健康监控任务异常: {e}")

async def _restart_ffmpeg(self):
    """🆕 重启 ffmpeg 进程"""
    logger.info("🔄 重启 ffmpeg 进程...")
    
    try:
        # 1. 停止旧进程
        if self._ffmpeg_proc:
            try:
                await self._ffmpeg_proc.kill()
            except:
                pass
        
        # 2. 重新获取流 URL
        handler = get_platform_handler(live_url=self._session.live_url)
        info = await handler.get_stream_info(self._session.live_url)
        new_record_url = self._extract_record_url(info)
        
        if not new_record_url:
            raise RuntimeError("无法获取新的流地址")
        
        # 3. 启动新进程
        seg_secs = self._session.segment_seconds
        out_dir = Path(self._session.recording_dir)
        anchor = self._session.anchor_name
        pattern = str(out_dir / f"{anchor}_%Y%m%d_%H%M%S_%03d.mp4")
        
        self._ffmpeg_proc = await self._start_ffmpeg(new_record_url, seg_secs, pattern)
        self._current_record_url = new_record_url
        
        logger.info("✅ ffmpeg 重启成功")
    
    except Exception as e:
        logger.error(f"❌ ffmpeg 重启失败: {e}")
        raise
```

## ⚠️ 重要说明

### 为什么不能简单延长录制时间？

**错误理解**：
> "将 30 分钟改为 240 分钟就能录制 4 小时"

**真实情况**：
```python
# ❌ 这样做无效！
seg_secs = 240 * 60  # 设置为 4 小时

# 原因：抖音流 URL 在 30-60 分钟后就会失效
# 即使 ffmpeg 设置了 4 小时，到 30 分钟时流就断了
# ffmpeg 会报错：HTTP error 403 Forbidden 或 404 Not Found
```

**URL 失效示例**：

```
初始 URL (有效 30 分钟):
https://pull-f5-hs.douyincdn.com/live-hs/12345_xxxx.flv?sign=abc&timestamp=1699012345

30 分钟后:
❌ URL 过期 → HTTP 403/404
✅ 需要重新调用 streamget API 获取新 URL

新 URL (有效 30 分钟):
https://pull-f5-hs.douyincdn.com/live-hs/12345_yyyy.flv?sign=def&timestamp=1699014145
```

### 技术根源

1. **抖音防盗链机制**
   - URL 中包含时间戳签名（`timestamp=xxx&sign=xxx`）
   - 签名有效期 30-60 分钟
   - 过期后必须重新获取

2. **streamget 库的限制**
   - `streamget` 只负责**获取流 URL**
   - 不负责**维持流的有效性**
   - 需要应用层自己处理刷新

3. **其他平台类似问题**
   - 快手、B站等平台也有 URL 时效性
   - 普遍有效期 20-90 分钟不等
   - 都需要定时刷新机制

## ⚠️ 注意事项

### 性能考虑

1. **URL 刷新频率**
   - 推荐 20 分钟刷新一次
   - 抖音 URL 有效期通常 30-60 分钟
   - 提前刷新避免中断

2. **切换时的数据丢失**
   - 切换过程可能丢失 1-2 秒
   - 对整体影响可忽略
   - 可通过优化 ffmpeg 参数减少

3. **内存占用**
   - 弹幕数据 ≈ 10-50MB（10万条弹幕）
   - 转写文本 ≈ 1-5MB
   - 建议服务器内存 ≥ 8GB

4. **磁盘空间**
   - 每小时 ≈ 2-4GB 视频文件
   - 4小时直播 ≈ 8-16GB
   - 建议预留 100GB 以上

### 异常情况处理

1. **网络中断**
   - ffmpeg 自动重连（`-reconnect 1`）
   - 健康监控检测并重启
   - 最多丢失 30 秒数据

2. **直播突然结束**
   - 健康监控检测文件不增长
   - 自动调用 `stop()`
   - 保存已录制的所有分段

3. **URL 刷新失败**
   - 记录警告日志
   - 继续使用旧 URL（直到失效）
   - 下次定时器再次尝试
