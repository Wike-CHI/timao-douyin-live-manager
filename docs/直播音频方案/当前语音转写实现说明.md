## 🎯 语音转写完整流程

```
┌─────────────────────────────────────────────────────┐
│ 1. 用户启动直播监听（前端）                          │
│    └─ 提供：抖音直播URL或房间ID                     │
└───────────────────┬─────────────────────────────────┘
                    │
                    ↓ API请求: POST /api/live_audio/start
┌─────────────────────────────────────────────────────┐
│ 2. 服务器端启动流程（LiveAudioStreamService）        │
│                                                      │
│  🔗 StreamCap：解析直播URL                          │
│     └─ 输入：https://live.douyin.com/123456789     │
│     └─ 输出：真实流地址（FLV/HLS/M3U8）             │
│                                                      │
│  🎬 ffmpeg：拉流 + 音频提取                         │
│     └─ 命令：ffmpeg -i <流地址> \                   │
│                -map 0:a -ac 1 -ar 16000 -f s16le pipe:1 \ ← 实时转写
│                -map 0:a -ac 1 -ar 16000 -f wav audio.wav   ← 保存文件
│                                                      │
│  🎙️ 音频数据流（两路输出）                           │
│     ├─ pipe:1 → 实时转写                            │
│     └─ audio.wav → 保存到服务器（可选）             │
└───────────────────┬─────────────────────────────────┘
                    │
                    ↓ PCM 16k/16bit/mono 音频流
┌─────────────────────────────────────────────────────┐
│ 3. 实时转写引擎（ASR）                               │
│                                                      │
│  📌 方案A：SenseVoice（默认）                        │
│     └─ 本地模型：models/iic/SenseVoiceSmall        │
│     └─ 优势：免费、隐私、低延迟                     │
│     └─ 劣势：需要GPU（推荐），CPU慢                │
│                                                      │
│  📌 方案B：科大讯飞（临时替代）✅ 当前启用            │
│     └─ 触发：环境变量 USE_IFLYTEK_ASR=1            │
│     └─ WebSocket实时流式转写                        │
│     └─ 优势：云端、无需GPU、准确度高               │
│     └─ 劣势：需要API密钥、有使用限制                │
│                                                      │
│  🎤 转写输出：原始文本（可能有噪音）                 │
└───────────────────┬─────────────────────────────────┘
                    │
                    ↓ 原始转写文本
┌─────────────────────────────────────────────────────┐
│ 4. 文本后处理（服务器端）                            │
│                                                      │
│  🧹 ChineseCleaner：中文清洗                         │
│     └─ 去除空格、标点统一、繁简转换                 │
│                                                      │
│  🛡️ HallucinationGuard：防幻觉                       │
│     └─ 过滤低置信度/低能量文本                      │
│     └─ 防止"嗯嗯啊啊"等无效转写                     │
│                                                      │
│  📝 SentenceAssembler：分句器                        │
│     └─ 将短文本组装成完整句子                      │
│     └─ 智能断句（标点、静音）                       │
│                                                      │
│  🔄 HotwordReplacer：热词替换（可选）                │
│     └─ 专有名词修正                                 │
│                                                      │
│  ✅ 输出：干净的转写文本                             │
└───────────────────┬─────────────────────────────────┘
                    │
                    ↓ 清洗后的文本
┌─────────────────────────────────────────────────────┐
│ 5. 文本存储 + 实时推送                               │
│                                                      │
│  💾 持久化存储（JSONL文件）                          │
│     └─ 路径：records/live_logs/{live_id}/{date}/   │
│              transcripts_{session_id}.jsonl         │
│     └─ 格式：每行一个JSON对象                       │
│        {                                            │
│          "timestamp": 1699512345678,                │
│          "text": "大家好，欢迎来到直播间",          │
│          "is_final": true                           │
│        }                                            │
│                                                      │
│  📡 实时推送（WebSocket）                            │
│     └─ 事件：transcript                             │
│     └─ 发送给所有连接的客户端                       │
│     └─ 前端实时显示转写文本                         │
└───────────────────┬─────────────────────────────────┘
                    │
                    ↓ 用于后续功能
┌─────────────────────────────────────────────────────┐
│ 6. 后续功能（基于转写文本）                          │
│                                                      │
│  📊 AI分析                                           │
│     ├─ 风格画像（通义千问）                         │
│     ├─ 话术生成（通义千问）                         │
│     └─ 情绪分析                                      │
│                                                      │
│  📝 报告生成（LiveReportService）                    │
│     └─ 读取转写文本 + 弹幕数据                      │
│     └─ 生成直播复盘报告                             │
│                                                      │
│  🔄 复盘分析（LiveReviewService）                    │
│     └─ 读取历史转写文本                             │
│     └─ 生成分析报告                                 │
└─────────────────────────────────────────────────────┘
```

## 🔑 关键组件说明

### 1. LiveAudioStreamService

**文件**：`server/app/services/live_audio_stream_service.py`

**职责**：
- ✅ 管理直播音频流的生命周期（启动、停止）
- ✅ 调用StreamCap解析直播URL
- ✅ 启动ffmpeg拉流
- ✅ 选择ASR引擎（SenseVoice/讯飞）
- ✅ 管理后处理流程
- ✅ 持久化转写文本

**核心方法**：
```python
async def start(live_url_or_id: str, session_id: Optional[str] = None) -> LiveAudioStatus
async def stop() -> LiveAudioStatus
def status() -> LiveAudioStatus
```

---

### 2. ASR引擎选择

#### 方案A：SenseVoice（本地）

**文件**：`server/modules/ast/sensevoice_service.py`

**配置**：
- 模型路径：`models/iic/SenseVoiceSmall`
- 需要：FunASR框架

**优势**：
- ✅ 完全免费
- ✅ 隐私保护（本地）
- ✅ 低延迟（无网络请求）

**劣势**：
- ⚠️ 需要GPU（推荐）或强CPU
- ⚠️ 4C8G服务器可能不够用

---

#### 方案B：科大讯飞（云端）✅ **当前启用**

**文件**：`server/modules/ast/iflytek_asr_adapter.py`

**配置**：
- 环境变量：`USE_IFLYTEK_ASR=1`（已设置）
- API密钥：`server/.env` 中的 `XFYUN_*` 配置

**优势**：
- ✅ 云端处理，无需GPU
- ✅ 准确度高
- ✅ 稳定可靠

**劣势**：
- ⚠️ 需要API密钥
- ⚠️ 有使用限制（按时长计费）

**接入方式**：
```python
# 环境变量检查（live_audio_stream_service.py:837）
use_iflytek = os.getenv("USE_IFLYTEK_ASR", "0") == "1"

if use_iflytek and IFLYTEK_AVAILABLE:
    self._sv = IFlyTekASRService()
    await self._sv.initialize()
else:
    # 使用SenseVoice（默认）
    self._sv = SenseVoiceService(config=SenseVoiceConfig(...))
```

### 3. ffmpeg音频流处理

**多输出模式**（当前实现）：
```bash
ffmpeg -i <直播流> \
  # 输出1: 实时转写（pipe）
  -map 0:a -ac 1 -ar 16000 -f s16le pipe:1 \
  # 输出2: 保存音频文件（可选）
  -map 0:a -ac 1 -ar 16000 -f wav live_audio.wav
```

**音频格式**：
- 采样率：16kHz（ASR标准）
- 声道：单声道（Mono）
- 格式：PCM s16le（实时）/ WAV（保存）

---

### 4. 文本后处理流程

```python
# live_audio_stream_service.py:195-200

self._cleaner = ChineseCleaner()  # 中文清洗
self._guard = HallucinationGuard(...)  # 防幻觉
self._assembler = SentenceAssembler(...)  # 分句器
self._hotword = HotwordReplacer()  # 热词替换（可选）
self._corrector = PhoneticCorrector()  # 语音修正（可选）
```

**处理流程**：
```
原始转写 → 清洗 → 防幻觉过滤 → 分句 → 热词替换 → 最终文本
```

---

### 5. 转写文本存储

**JSONL格式**（每行一个JSON）：
```jsonl
{"timestamp": 1699512345678, "text": "大家好", "is_final": true}
{"timestamp": 1699512346789, "text": "欢迎来到直播间", "is_final": true}
```

**存储路径**：
```
records/
├── live_logs/               ← 转写文本（复盘读取）
│   └── {live_id}/
│       └── {date}/
│           └── transcripts_{session_id}.jsonl
│
└── sessions/                ← 音频文件（归档）
    └── {session_id}/
        └── live_audio.wav
```

**复盘读取**：
- ✅ 读取：`transcripts_{session_id}.jsonl`（转写文本）
- ❌ 不读取：`live_audio.wav`（音频归档）

## 🔄 数据流对比

### 实时转写模式（当前）

```
直播流 → ffmpeg → pipe → ASR引擎 → 后处理 → WebSocket推送 + 文本保存
                   ↓
              live_audio.wav（可选保存）

```

**注意**：复盘**不使用**音频文件，只读取已转写的文本！

## 📊 性能和资源

**当前配置**：4C8G腾讯云服务器

## 🔍 故障排查

### 转写没有输出？

1. **检查ASR引擎**：
   ```bash
   grep "USE_IFLYTEK_ASR" server/.env
   grep "USE_IFLYTEK_ASR" .env  # 注意：根目录.env可能覆盖
   ```

2. **查看日志**：
   ```bash
   tail -f logs/backend.log | grep -i "asr\|转写\|iflytek"
   ```

3. **检查音频流**：
   ```bash
   # 查看音频接收状态
   GET /api/live_audio/status
   # 应该看到 "is_receiving_audio": true
   ```

### 音频文件没有生成？

**原因**：客户端本地录制（方案B）尚未完全实施

**临时方案**：音频保存在服务器 `records/sessions/{session_id}/live_audio.wav`
