**审查人**: 叶维哲

## 📋 方案选择

**最终选择：方案B（客户端本地录制）**

**原因**：
- ✅ **符合KISS原则**：简单、直接、本地化
- ✅ **不占服务器空间**：音频直接保存到用户电脑
- ✅ **节省带宽**：不需要从服务器下载
- ✅ **用户可管理**：音频文件在用户文档目录

---

## 🎯 技术架构

```
┌───────────────────────────────────────────┐
│ Electron客户端（用户电脑）                 │
│                                           │
│  1. 获取流地址 ──API请求──> 云服务器       │
│     └─ GET /api/live_audio/stream-url    │
│                             ↓             │
│  2. 启动本地ffmpeg          └─ 返回流URL   │
│     └─ spawn(ffmpeg, args)                │
│         ↓                                 │
│  3. 拉流录制音频                           │
│     └─ 从抖音CDN拉流 ──直接连接──> 抖音CDN │
│         ↓                                 │
│  4. 保存到本地                            │
│     └─ Documents/TalkingCat/audio/       │
│                                           │
└───────────────────────────────────────────┘
```

**核心优势**：
- 🎙️ **完全本地化**：录制过程不经过服务器
- 💾 **用户控制**：音频文件在用户电脑，可随时管理
- 🚀 **带宽友好**：不占用服务器带宽
- 📦 **空间友好**：不占用服务器硬盘

## 🔧 技术实现

### 1. 服务器端API

**文件**: `server/app/api/live_audio.py`

**新增端点**:
```python
@router.get("/stream-url/{live_url_or_id}")
async def get_stream_url(live_url_or_id: str):
    """
    获取直播流地址（供客户端本地录制）
    
    Returns:
        {
            "live_id": "123456",
            "stream_url": "https://...",
            "anchor_name": "主播名",
            "room_title": "直播间标题"
        }
    """
```

**功能**：
- 解析抖音直播URL或房间ID
- 使用StreamCap获取真实流地址
- 返回流URL和主播信息

---

### 2. Electron本地录制服务

**文件**: `electron/services/localAudioRecorder.js`

**核心类**: `LocalAudioRecorder`

**方法**：
```javascript
// 开始录制
async startRecording(liveUrlOrId, apiUrl, options)

// 停止录制
stopRecording()

// 获取状态
getStatus()

// 获取已录制列表
getAudioFiles()

// 删除文件
deleteAudioFile(filePath)

// 打开目录
openAudioDirectory()
```

**录制流程**：
1. 从服务器API获取流地址
2. 启动本地ffmpeg进程
3. 拉流并保存为WAV格式
4. 保存到用户文档目录

**ffmpeg命令**：
```bash
ffmpeg -loglevel warning \
  -user_agent "Mozilla/5.0 ..." \
  -headers "referer:https://live.douyin.com" \
  -i <stream_url> \
  -vn -ac 1 -ar 16000 -f wav \
  <save_path>
```

---

### 3. IPC集成（Electron主进程）

**文件**: `electron/main.js`

**IPC处理器**：

## 🎬 使用流程

### 客户端调用（JavaScript）

```javascript
// 1. 开始本地录制
const result = await window.electron.localAudioStart(
    'https://live.douyin.com/123456',  // 直播URL
    'http://129.211.218.135',          // API地址
    { sessionId: 'live_123456' }       // 可选参数
);

if (result.success) {
    console.log('录制已开始:', result.data.savePath);
}

// 2. 停止录制
const stopResult = await window.electron.localAudioStop();
console.log('录制已停止:', stopResult.savePath);

// 3. 获取状态
const status = await window.electron.localAudioStatus();
console.log('录制状态:', status.data);

// 4. 获取文件列表
const files = await window.electron.localAudioList();
console.log('已录制文件:', files.data);

// 5. 打开音频目录
await window.electron.localAudioOpenDir();
```

## ⚠️ 注意事项

### 1. ffmpeg依赖

- ✅ 已集成到项目：`tools/ffmpeg/`
- ✅ 打包时自动包含
- ⚠️ 如果缺失，录制会失败

### 2. 用户权限

- 需要文档目录写权限（通常默认有）
- Mac可能需要用户授权

### 3. 性能考虑

- 录制时ffmpeg占用CPU：5-10%
- 内存占用：约50-100MB
- 磁盘IO：实时写入，无明显影响

### 4. 服务器端改动

- ⚠️ 保留之前的服务器录制代码（作为备用）
- ✅ 新增API不影响现有功能
- 🎚️ **未来可添加配置**：用户选择本地或云端录制
