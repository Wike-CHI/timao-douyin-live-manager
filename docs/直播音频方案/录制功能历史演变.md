## 🔍 为什么会录制视频？

### 原始设计（LiveReportService）

**文件**：`server/app/services/live_report_service.py`

**流程**：
```python
"""
Live Report Service
Integrates StreamCap's recording backend (streamget + ffmpeg), collects Douyin
danmu, runs offline SenseVoice transcription on 30-minute segments, and composes
final transcript and a recap report artifact.

录制流程：
1. 录制完整视频（MP4，每30分钟一个分段）
2. 直播结束后，离线处理：
   - 提取音频（PCM 16k）
   - SenseVoice批量转写
   - 生成转写文本
3. 结合弹幕数据，生成报告
"""
```

**核心代码**（live_report_service.py:770-790）：
```python
async def generate_report(self):
    # 转写每个视频分段
    for seg in self._session.segments:
        seg_path = Path(seg["path"])  # mp4文件
        pcm_path = seg_path.with_suffix(".pcm")
        
        # 提取音频
        await self._extract_pcm(seg_path, pcm_path)
        pcm_bytes = pcm_path.read_bytes()
        
        # SenseVoice批量转写（离线）
        res = await sv.transcribe_audio(pcm_bytes)
        text = res.get("text", "")
        
        transcripts.append({
            "seq": seg.get("seq"),
            "path": str(seg_path),
            "text": text,
        })
```

**为什么录制视频**：
- ❌ **不是为了观看回放**
- ❌ **不是为了保存视频内容**
- ✅ **是为了离线转写音频**（提取PCM → SenseVoice转写）

## 🆕 新方案：实时转写（不需要录制）

**文件**：`server/app/services/live_audio_stream_service.py`

**流程**：
```python
# 实时拉流
ffmpeg -i 直播流 -map 0:a -ac 1 -ar 16000 -f s16le pipe:1

# 实时转写（讯飞ASR）
pcm_chunk → IFlyTekASRService.transcribe() → 转写文本

# 实时保存
transcripts.jsonl  # 转写文本实时写入
```

**优势**：
- ✅ **不需要录制视频**（节省空间）
- ✅ **实时转写**（低延迟）
- ✅ **云端处理**（不需要GPU）
- ✅ **转写文本实时可用**（不需要等直播结束）

---

## 🔄 两套系统对比

| 对比项 | LiveReportService（旧） | LiveAudioStreamService（新）✅ |
|--------|-------------------------|-------------------------------|
| **用途** | 录制视频 + 离线转写 | 实时转写 |
| **转写引擎** | SenseVoice（本地） | 讯飞ASR（云端） |
| **转写时机** | 直播结束后（离线） | 直播中（实时） |
| **视频录制** | ✅ 录制MP4 | ❌ 不录制 |
| **资源需求** | 需要GPU（4C8G不够）| 不需要GPU ✅ |
| **空间占用** | 大（视频~500MB/小时）| 小（文本<1MB/小时） |
| **延迟** | 高（直播结束后处理）| 低（实时） |
| **转写文本** | 生成到 transcripts.jsonl | 实时写入 transcripts.jsonl |

## 💡 现在的问题：冗余

### 1. 录制视频的作用已被替代

**旧方案**：
```
录制视频 → 提取音频 → SenseVoice转写 → 转写文本
```

**新方案**：
```
实时拉流 → 讯飞ASR转写 → 转写文本 ✅（已有）
```

**结论**：**视频录制已无必要** ❌

---

### 2. 复盘只需要转写文本

**复盘读取**：
```python
# server/app/services/live_review_service.py

# ✅ 读取转写文本
transcript_file = "records/live_logs/.../transcripts.jsonl"

# ❌ 不读取视频
# video_file = "records/sessions/.../segment_001.mp4"  # 不用
```

**结论**：**视频文件复盘不使用** ❌

---

### 3. 音频归档也不必要

**当前实现**（live_audio_stream_service.py:502-540）：
```python
# ffmpeg多输出：
# 输出1: pipe → ASR实时转写 ✅（需要）
# 输出2: live_audio.wav → 保存音频 ❓（不需要）

cmd = [
    ffmpeg,
    "-i", 直播流,
    "-map", "0:a", "-f", "s16le", "pipe:1",  # 实时转写 ✅
    "-map", "0:a", "-f", "wav", "audio.wav"  # 保存音频 ❓
]
```

**问题**：
- 转写文本已经实时生成 ✅
- 音频文件只是"归档"，复盘不用 ❌
- 如果需要"听回放"，才需要音频 ❓

**建议**：
- 如果不需要听回放 → 不保存音频 ✅
- 如果需要听回放 → 保存到客户端本地（方案B）

## ✅ 建议的优化方案

### 方案A：完全移除录制（推荐）✅

**简化架构**：
```
直播流 → 实时转写（讯飞ASR）→ 保存转写文本 → 复盘读取文本
```

**移除**：
- ❌ LiveReportService的视频录制
- ❌ LiveAudioStreamService的音频保存

**保留**：
- ✅ 实时转写（讯飞ASR）
- ✅ 转写文本保存（transcripts.jsonl）
- ✅ 弹幕保存（comments.jsonl）

**优势**：
- 简单：符合KISS原则 ✅
- 省空间：不保存视频/音频 ✅
- 满足需求：复盘有转写文本即可 ✅

---

### 方案B：保留音频归档（可选）

**如果用户需要"听回放"**：

```
直播流 → 实时转写（讯飞ASR）→ 转写文本
         ↓
      可选：保存音频到客户端本地
      └─ Documents/TalkingCat/audio/
```

**用途**：
- 听直播回放（查证转写准确性）
- 音频剪辑
- 存档备份

**实现**：客户端本地录制（方案B，已设计）

---

### 方案C：完全移除LiveReportService

**极致简化**：

**移除**：
- ❌ `LiveReportService`（整个服务）
- ❌ 视频录制功能
- ❌ SenseVoice离线转写

**保留**：
- ✅ `LiveAudioStreamService`（实时转写）
- ✅ `DouyinWebRelay`（弹幕采集）
- ✅ 报告生成逻辑（迁移到其他服务）

**架构**：
```
┌─────────────────────────────────────┐
│ LiveAudioStreamService              │
│ - 实时转写（讯飞ASR）               │
│ - 保存转写文本                      │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│ DouyinWebRelay                      │
│ - 采集弹幕                          │
│ - 保存弹幕数据                      │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│ LiveReviewService（新）             │
│ - 读取转写文本 + 弹幕               │
│ - 生成复盘报告                      │
└─────────────────────────────────────┘
```

## 🎯 结论

### 为什么会录制视频？

**原因**：**历史遗留**

最初设计时：
- 没有实时转写 → 需要录制视频 → 离线提取音频 → SenseVoice转写
- 录制视频的**唯一目的**就是为了**离线转写**

现在：
- 有实时转写（讯飞ASR）✅
- 转写文本实时生成 ✅
- **不再需要录制视频** ❌

### 应该怎么做？

**推荐：方案A（完全移除录制）**

- ❌ 不录制视频
- ❌ 不保存音频（除非需要听回放）
- ✅ 只保存转写文本（已实时生成）
- ✅ 只保存弹幕数据

**架构**：
```
实时拉流 → 讯飞ASR转写 → 转写文本（transcripts.jsonl）
         → 采集弹幕    → 弹幕数据（comments.jsonl）
                         ↓
                      复盘读取 → 生成报告
```

**符合KISS原则**：简单、直接、满足需求 ✅
