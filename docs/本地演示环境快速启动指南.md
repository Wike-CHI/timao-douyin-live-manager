## 📋 环境说明

### 🎯 配置特点

- ✅ **数据库**：使用云端阿里云RDS（无需本地MySQL）
- ✅ **后端端口**：15000（避免与生产环境11111冲突）

## 🚀 快速启动

### 方式1：一键启动（推荐）

```bash
npm run dev
```

**说明**：自动启动所有服务
- 🔵 后端服务（端口15000）
- 🟢 前端开发服务器（端口10050）
- 🟡 Electron应用

### 方式2：分步启动

适合调试或单独启动某个服务。

```bash
# 终端1：启动后端服务
npm run services

# 终端2：启动前端开发服务器
npm run dev:renderer

# 终端3：启动Electron应用
npm run dev:electron
```

### 方式3：使用批处理脚本

```bash
# Windows
scripts\构建与启动\start-dev.bat

# 或
npm run start:dev
```

## ✅ 启动前检查

### 1. 确认分支

```bash
git branch
# 应该显示: * local-view
```

### 2. 检查端口占用

```bash
npm run port:check
```

如果端口被占用，清理端口：

```bash
npm run kill:ports
```

### 3. 验证环境配置

```bash
# 检查后端端口配置
cat server/.env | grep BACKEND_PORT
# 应该显示: BACKEND_PORT=15000

# 检查数据库配置
cat server/.env | grep MYSQL_HOST
# 应该显示: MYSQL_HOST=rm-bp1sqxf05yom2hwdhko.mysql.rds.aliyuncs.com
```

## 🐛 故障排查

### 问题1：端口被占用

**症状**：启动时提示端口已被占用

**解决方案**：
```bash
# 方法1：清理所有项目端口
npm run kill:ports

# 方法2：清理特定端口
npm run kill:backend  # 清理15000端口
npm run kill:frontend # 清理10050端口
```

### 问题2：数据库连接失败

**症状**：后端启动失败，提示数据库连接错误

**检查步骤**：
1. 确认网络连接正常
2. 确认云数据库配置正确
3. 检查 `server/.env` 中的数据库配置

```bash
cat server/.env | grep MYSQL
```

### 问题3：前端无法连接后端

**症状**：前端页面显示网络错误

**检查步骤**：
1. 确认后端服务已启动
```bash
curl http://127.0.0.1:15000/health
```

2. 检查前端代理配置（应该已配置好）
```bash
cat electron/renderer/vite.config.ts | grep port
```

### 问题4：Electron应用无法启动

**症状**：Electron窗口不显示

**检查步骤**：
1. 确认前端开发服务器已启动
```bash
curl http://127.0.0.1:10050
```

2. 等待前端服务器完全启动后再启动Electron
```bash
wait-on http://127.0.0.1:10050/health && npm run dev:electron
```

## 📌 注意事项

### ⚠️ 重要提醒

1. **分支隔离**
   - local-view 分支永不合并到 main
   - 只用于本地演示和视频录制
   - 配置文件与生产环境不同

2. **数据库说明**
   - 使用云端RDS，与生产环境共享数据
   - 测试时注意不要影响生产数据
   - 建议使用测试账号和测试直播间

3. **端口配置**
   - 后端：15000（本地专用）
   - 前端：10050（本地专用）
   - 生产环境使用不同端口（11111）

4. **性能优化**
   - 开发模式启用了热重载
   - 生产打包需切换到 main 分支
