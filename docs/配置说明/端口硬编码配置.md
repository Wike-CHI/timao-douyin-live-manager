## 配置概述

为了快速启动演示测试，已将前后端端口硬编码：

- **后端端口**: `11111`
- **前端端口**: `10065`

## 修改的文件清单

### 1. 后端服务启动配置

**文件**: `scripts/构建与启动/service_launcher.py`

```python

```

**修改位置**:
- `start_all_services()` 方法 - 启动服务时
- `health_check()` 方法 - 健康检查时

## 启动命令

使用硬编码端口配置后，启动命令保持不变：

```bash
# 开发环境启动（自动杀端口、启动所有服务）
npm run dev

# 或分步启动
npm run dev:step1  # 启动后端服务（端口 11111）
npm run dev:step2  # 启动前端服务（端口 10065）
npm run dev:step3  # 启动 Electron 主进程
```

## 注意事项

### 1. 防火墙配置

如果遇到连接问题，请检查防火墙是否阻止了端口：

**Windows 防火墙**:
```powershell
# 添加入站规则允许端口 11111
netsh advfirewall firewall add rule name="TalkingCat Backend" dir=in action=allow protocol=TCP localport=11111

# 添加入站规则允许端口 10065
netsh advfirewall firewall add rule name="TalkingCat Frontend" dir=in action=allow protocol=TCP localport=10065
```

### 2. 端口冲突

如果端口被其他程序占用：

1. 使用 `npm run kill:ports` 清理端口
2. 或手动杀死占用进程
3. 或修改硬编码的端口号（需要同步修改所有相关文件）

### 3. 环境变量无效

硬编码配置后，以下环境变量将**失效**：

- `BACKEND_PORT` - 后端端口配置
- `ELECTRON_RENDERER_URL` - 前端地址配置
- `VITE_FASTAPI_URL` - 前端 API 地址配置

如需修改端口，必须直接修改源码中的硬编码值。

## 恢复环境变量配置

如需恢复使用环境变量配置，需要：

1. 修改 `scripts/构建与启动/service_launcher.py`，恢复 `os.getenv("BACKEND_PORT", "11111")`
2. 修改 `electron/renderer/src/services/apiConfig.ts`，恢复环境变量读取
3. 修改 `electron/main.js`，恢复环境变量读取
4. 修改 `electron/renderer/package.json`，使用环境变量或参数
5. 修改 `package.json`，恢复环境变量引用
