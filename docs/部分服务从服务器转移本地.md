# 云端本地混合部署架构设计

## 📋 架构概述

### 背景
- **问题**：服务器内存不足，无法支撑所有服务
- **解决方案**：将服务拆分为云端服务和本地服务
  - **云端**：用户系统、支付系统（轻量级，低内存占用）
  - **本地**：弹幕拉取、语音转写、AI处理等（重度服务，打包到客户端）

### 架构优势
✅ **降低服务器成本**：云端只运行轻量服务，内存需求 < 1GB  
✅ **提升用户体验**：本地服务响应更快，无网络延迟  
✅ **数据隐私保护**：敏感的直播数据不上传云端  
✅ **灵活扩展**：本地服务可使用用户的硬件资源

## 🔄 服务交互流程

### 1. 应用启动流程

```
用户启动应用
    ↓
Electron 主进程启动
    ↓
启动本地 FastAPI 服务 (16000端口)
    ├── 加载 SenseVoice 模型
    ├── 初始化 VAD 检测
    └── 启动 WebSocket 服务
    ↓
渲染进程加载
    ↓
检查登录状态
    ├── 未登录 → 调用云端登录 API (15000)
    └── 已登录 → 验证 Token (15000)
    ↓
进入主界面
    ├── 本地服务：弹幕拉取、语音转写
    └── 云端服务：用户信息、订阅状态
```

### 2. 用户登录流程

```
用户输入账号密码
    ↓
Electron → 云端 API (15000)
POST /api/auth/login
    ↓
云端验证账号密码
    ↓
返回 Token + 用户信息
    ↓
Electron 存储 Token
    ├── localStorage (渲染进程)
    └── electron-store (主进程)
    ↓
后续请求携带 Token
```

### 3. 语音转写流程

```
用户点击"开始转写"
    ↓
Electron → 本地 API (16000)
POST /api/transcribe/start
    ↓
本地服务加载模型
    ├── SenseVoice (已预加载)
    └── VAD 检测器
    ↓
接收音频流 (WebSocket)
    ↓
实时转写 → 返回文本
    ↓
Electron 显示转写结果
```

### 4. 订阅验证流程

```
用户使用高级功能
    ↓
Electron → 云端 API (15000)
GET /api/user/subscription
    ↓
云端验证订阅状态
    ├── 已订阅 → 返回权限
    └── 未订阅 → 返回受限
    ↓
Electron 根据权限控制功能
```

### Electron 集成本地服务

**main.js 启动本地服务**:
```javascript
// electron/main/index.js
const { spawn } = require('child_process');
const path = require('path');

let localAPIProcess = null;
const LOCAL_API_PORT = 16000;

// 启动本地 FastAPI 服务
async function startLocalAPI() {
  const pythonPath = process.env.PYTHON_PATH || 'python';
  const apiScript = path.join(__dirname, '../../server/local/main.py');
  
  localAPIProcess = spawn(pythonPath, [
    '-m', 'uvicorn',
    'server.local.main:app',
    '--host', '127.0.0.1',
    '--port', LOCAL_API_PORT.toString(),
    '--workers', '1'
  ]);

  localAPIProcess.stdout.on('data', (data) => {
    console.log(`[Local API] ${data}`);
  });

  localAPIProcess.stderr.on('data', (data) => {
    console.error(`[Local API Error] ${data}`);
  });

  // 等待服务启动
  await waitForService(`http://127.0.0.1:${LOCAL_API_PORT}/health`);
  console.log('✅ 本地 API 服务启动成功');
}

// 等待服务就绪
function waitForService(url, maxRetries = 30) {
  return new Promise((resolve, reject) => {
    let retries = 0;
    const interval = setInterval(async () => {
      try {
        const response = await fetch(url);
        if (response.ok) {
          clearInterval(interval);
          resolve();
        }
      } catch (error) {
        retries++;
        if (retries >= maxRetries) {
          clearInterval(interval);
          reject(new Error('本地服务启动超时'));
        }
      }
    }, 1000);
  });
}

// 应用启动
app.whenReady().then(async () => {
  // 1. 启动本地服务
  await startLocalAPI();
  
  // 2. 创建窗口
  createWindow();
});

// 应用退出时关闭本地服务
app.on('quit', () => {
  if (localAPIProcess) {
    localAPIProcess.kill();
  }
});
```

### 前端 API 调用

**API 配置**:
```typescript
// electron/renderer/src/config/api.ts
export const API_CONFIG = {
  // 云端服务（用户、支付）
  CLOUD_API: import.meta.env.VITE_CLOUD_API || 'https://your-server.com:15000',
  
  // 本地服务（直播、转写）
  LOCAL_API: 'http://127.0.0.1:16000',
};

// API 客户端
export class APIClient {
  // 云端请求
  static async cloudRequest(endpoint: string, options = {}) {
    const token = localStorage.getItem('token');
    return fetch(`${API_CONFIG.CLOUD_API}${endpoint}`, {
      ...options,
      headers: {
        'Authorization': `Bearer ${token}`,
        'Content-Type': 'application/json',
        ...options.headers,
      },
    });
  }

  // 本地请求
  static async localRequest(endpoint: string, options = {}) {
    return fetch(`${API_CONFIG.LOCAL_API}${endpoint}`, {
      ...options,
      headers: {
        'Content-Type': 'application/json',
        ...options.headers,
      },
    });
  }
}
```

**使用示例**:
```typescript
// 登录（云端）
async function login(username: string, password: string) {
  const response = await APIClient.cloudRequest('/api/auth/login', {
    method: 'POST',
    body: JSON.stringify({ username, password }),
  });
  const data = await response.json();
  localStorage.setItem('token', data.token);
  return data;
}

// 开始转写（本地）
async function startTranscribe() {
  const response = await APIClient.localRequest('/api/transcribe/start', {
    method: 'POST',
  });
  return response.json();
}
```

## 🔐 安全考虑

### 1. Token 管理
- **云端 Token**：用于云端 API 鉴权，存储在 localStorage
- **刷新机制**：Token 过期自动刷新
- **安全存储**：敏感信息使用 electron-store 加密存储

### 2. 本地服务安全
- **端口绑定**：只绑定 127.0.0.1，不允许外部访问
- **进程隔离**：本地服务只响应本地请求
- **数据加密**：敏感配置加密存储

### 3. 通信安全
- **HTTPS**：云端 API 使用 HTTPS
- **CORS**：严格限制跨域访问
- **请求签名**：关键请求添加签名验证

## 🎯 模型下载与初始化方案

### 背景与目标

**问题**：
- SenseVoice 模型文件约 **2.3GB**，打包到安装包会导致：
  - 安装包体积过大（> 3GB），用户下载困难
  - 占用大量磁盘空间，即使用户不使用语音转写功能
  - 打包、分发成本高

**解决方案**：
- ✅ **按需下载**：首次启动或首次使用转写功能时下载模型
- ✅ **断点续传**：网络中断后可继续下载，避免重新开始
- ✅ **CDN 加速**：使用 CDN 分发，提升下载速度
- ✅ **完整性校验**：SHA256 校验确保文件完整性
- ✅ **进度展示**：实时显示下载进度，提升用户体验
- ✅ **容错机制**：网络异常、校验失败自动重试

#### 完整下载流程图

```
应用启动
    ↓
检查本地模型
    ├── 已存在且校验通过 → 直接使用
    └── 不存在或损坏 → 进入下载流程
    ↓
获取 CDN manifest.json
    ├── 成功 → 解析模型清单
    └── 失败 → 使用内置 manifest（降级）
    ↓

    ├── 用户确认 → 开始下载
    ├── 用户取消 → 禁用转写功能
    └── 后台下载选项 → 异步下载
    ↓
分片下载（支持断点续传）
    ├── 每 10MB 一个分片
    ├── 并发下载（最多 3 个分片）
    └── 实时更新进度条
    ↓
SHA256 完整性校验
    ├── 校验通过 → 移动到正式目录
    └── 校验失败 → 删除并重试（最多 3 次）
    ↓
更新模型注册表
    ↓
初始化模型
    ↓
可用状态
```

```javascript
// electron/main/model-downloader.js
const fs = require('fs-extra');
const path = require('path');
const fetch = require('node-fetch');
const crypto = require('crypto');

class ModelDownloader {
  constructor(modelInfo, targetPath) {
    this.modelInfo = modelInfo;      // manifest 中的模型信息
    this.targetPath = targetPath;    // 最终保存路径
    this.tempPath = `${targetPath}.part`;  // 临时文件
    this.chunkSize = 10 * 1024 * 1024;     // 10MB 分片
    this.maxConcurrent = 3;                 // 最大并发数
    this.maxRetries = 3;                    // 最大重试次数
  }

  async download(onProgress) {
    const { url, size, sha256 } = this.modelInfo;
    
    // 1. 检查是否有未完成的下载
    let downloaded = 0;
    if (await fs.pathExists(this.tempPath)) {
      const stat = await fs.stat(this.tempPath);
      downloaded = stat.size;
      console.log(`📦 发现未完成的下载，已下载: ${downloaded}/${size}`);
    }

    // 2. 分片下载
    const chunks = this.calculateChunks(downloaded, size);
    console.log(`🔗 分片数量: ${chunks.length}`);

    for (const chunk of chunks) {
      await this.downloadChunk(url, chunk, onProgress);
    }

    // 3. 校验文件
    console.log('🔍 校验文件完整性...');
    const isValid = await this.verifySHA256(this.tempPath, sha256);
    if (!isValid) {
      throw new Error('文件校验失败，SHA256 不匹配');
    }

    // 4. 移动到正式目录
    await fs.move(this.tempPath, this.targetPath, { overwrite: true });
    console.log('✅ 模型下载完成');
  }

  calculateChunks(downloaded, total) {
    const chunks = [];
    let start = downloaded;
    
    while (start < total) {
      const end = Math.min(start + this.chunkSize - 1, total - 1);
      chunks.push({ start, end });
      start = end + 1;
    }
    
    return chunks;
  }

  async downloadChunk(url, { start, end }, onProgress) {
    let retries = 0;
    
    while (retries < this.maxRetries) {
      try {
        const response = await fetch(url, {
          headers: {
            'Range': `bytes=${start}-${end}`,
          },
        });

        if (!response.ok && response.status !== 206) {
          throw new Error(`HTTP ${response.status}`);
        }

        const buffer = await response.buffer();
        
        // 追加写入临时文件
        await fs.appendFile(this.tempPath, buffer);
        
        // 更新进度
        if (onProgress) {
          const stat = await fs.stat(this.tempPath);
          const progress = (stat.size / this.modelInfo.size) * 100;
          onProgress({ downloaded: stat.size, progress });
        }

        return; // 成功
      } catch (error) {
        retries++;
        console.error(`❌ 下载分片失败 (${retries}/${this.maxRetries}):`, error.message);
        
        if (retries >= this.maxRetries) {
          throw error;
        }
        
        // 指数退避重试
        await this.sleep(Math.pow(2, retries) * 1000);
      }
    }
  }

  async verifySHA256(filePath, expectedHash) {
    return new Promise((resolve, reject) => {
      const hash = crypto.createHash('sha256');
      const stream = fs.createReadStream(filePath);
      
      stream.on('data', (chunk) => hash.update(chunk));
      stream.on('end', () => {
        const actualHash = hash.digest('hex');
        resolve(actualHash === expectedHash);
      });
      stream.on('error', reject);
    });
  }

  sleep(ms) {
    return new Promise(resolve => setTimeout(resolve, ms));
  }
}

module.exports = ModelDownloader;
```

```javascript
const { ipcMain } = require('electron');
const ModelDownloader = require('./model-downloader');

class ModelManager {
  constructor() {

    this.downloadStates = new Map(); // modelId -> state
    this.setupIPC();
  }

  setupIPC() {
    ipcMain.on('start-model-download', async (event, { modelId }) => {

        event.sender.send('model-download-progress', progress);
      });
    });

    ipcMain.on('pause-model-download', (event, { modelId }) => {
      this.pauseDownload(modelId);
    });

    ipcMain.on('resume-model-download', (event, { modelId }) => {

    });

    ipcMain.on('cancel-model-download', (event, { modelId }) => {
      this.cancelDownload(modelId);
    });
  }

    try {
      // 1. 获取 manifest
      const manifest = await this.fetchManifest();
      const modelInfo = manifest.models.find(m => m.id === modelId);
      
      if (!modelInfo) {
        throw new Error(`模型 ${modelId} 不存在`);
      }

      // 2. 准备下载路径
      const targetPath = this.getModelPath(modelId);
      await fs.ensureDir(path.dirname(targetPath));

      // 3. 创建下载器

        modelInfo.files[0],  // 主文件
        targetPath
      );
      this.downloaders.set(modelId, downloader);

      // 4. 开始下载
      onProgress({ state: 'downloading', progress: 0 });
      
      await downloader.download((progress) => {
        onProgress({
          state: 'downloading',
          ...progress,
          total: modelInfo.files[0].size,
        });
      });

      // 5. 下载完成
      onProgress({ state: 'available', progress: 100 });
      
      // 6. 更新注册表
      await this.updateRegistry(modelId, {
        version: modelInfo.version,
        downloadedAt: Date.now(),
      });

    } catch (error) {
      console.error('下载失败:', error);
      onProgress({
        state: 'error',
        error: error.message,
      });
    }
  }

  async fetchManifest() {
    const manifestUrl = 'https://cdn.timao.com/models/manifest.json';
    
    try {
      const response = await fetch(manifestUrl);
      if (!response.ok) throw new Error('获取 manifest 失败');

      return require('./manifest-fallback.json');
```

```javascript
// electron/main/model-error-handler.js

class ModelErrorHandler {
  static handleDownloadError(error, context) {
    const { modelId, retries } = context;
    
    // 1. 网络错误
    if (error.code === 'ENOTFOUND' || error.code === 'ETIMEDOUT') {
      if (retries < 3) {
        console.log(`🔄 网络错误，重试 (${retries + 1}/3)`);
        return { action: 'retry', delay: Math.pow(2, retries) * 1000 };
      }
      return {
        action: 'fail',
        message: '网络连接失败，请检查网络后重试',
        fallback: 'use_mirror',
      };
    }

    // 2. SHA256 校验失败
    if (error.message.includes('SHA256')) {
      if (retries < 3) {
        console.log('🔄 校验失败，重新下载');
        return { action: 'retry', clearCache: true };
      }
      return {
        action: 'fail',
        message: '文件损坏，请联系客服',
        fallback: 'report_error',
      };
    }

    // 3. 磁盘空间不足
    if (error.code === 'ENOSPC') {
      return {
        action: 'fail',
        message: '磁盘空间不足，请清理后重试',
        required: '3GB',
      };
    }

    // 4. 权限错误
    if (error.code === 'EACCES' || error.code === 'EPERM') {
      return {
        action: 'fail',
        message: '没有写入权限，请以管理员身份运行',
      };
    }

    // 5. 用户取消
    if (error.code === 'CANCELLED') {
      return {
        action: 'cancel',
        message: '用户取消下载',
      };
    }

    // 6. 未知错误
    return {
      action: 'fail',
      message: `下载失败: ${error.message}`,
      fallback: 'report_error',
    };
  }

  static async switchToMirror(modelInfo) {
    // 切换到备用 CDN
    const mirrors = modelInfo.mirrors || [];
    for (const mirrorUrl of mirrors) {
      try {
        console.log(`🔄 尝试备用地址: ${mirrorUrl}`);
        const response = await fetch(mirrorUrl, { method: 'HEAD' });
        if (response.ok) {
          return mirrorUrl;
        }
      } catch (error) {
        console.warn(`备用地址失败: ${mirrorUrl}`);
      }
    }
    throw new Error('所有下载地址均不可用');
  }

  static async reportError(error, context) {
    // 上报错误到服务器（用于监控）
    try {
      await fetch('https://api.timao.com/v1/error-report', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          type: 'model_download_error',
          error: error.message,
          stack: error.stack,
          context,
          userAgent: navigator.userAgent,
          timestamp: Date.now(),
        }),
      });
    } catch (e) {
      console.error('错误上报失败:', e);
    }
  }
}

module.exports = ModelErrorHandler;
```

```python
    downloader = create_downloader(manifest.model_info)

    if exists(temp_file):
        downloaded_size = get_file_size(temp_file)
        log(f"发现未完成下载，已下载 {downloaded_size} 字节")
        downloader.resume_from(downloaded_size)

    total_size = manifest.model_info.size
    chunk_size = 10 * 1024 * 1024  # 10MB
    chunks = calculate_chunks(downloaded_size, total_size, chunk_size)
    
    for chunk in chunks:
        retry_count = 0
        max_retries = 3
        
        while retry_count < max_retries:
            try:
                # HTTP Range 请求
                data = download_chunk_with_range(
                    url=manifest.model_info.url,
                    start=chunk.start,
                    end=chunk.end
                )
                
                # 追加写入临时文件
                append_to_file(temp_file, data)
                
                # 更新进度
                current_size = get_file_size(temp_file)
                progress = (current_size / total_size) * 100
                emit_progress_event({
                    "state": "downloading",
                    "progress": progress,
                    "downloaded": current_size,
                    "total": total_size,
                    "speed": calculate_speed(),
                    "eta": calculate_eta()
                })
                
                break  # 成功，跳出重试循环
                
            except NetworkError as e:
                retry_count += 1
                log(f"下载失败，重试 {retry_count}/{max_retries}: {e}")
                
                if retry_count >= max_retries:
                    # 尝试切换到备用 CDN
                    mirror_url = switch_to_mirror(manifest.model_info.mirrors)
                    if mirror_url:
                        log(f"切换到备用地址: {mirror_url}")
                        manifest.model_info.url = mirror_url
                        retry_count = 0  # 重置重试次数
                    else:
                        raise DownloadError("所有下载地址均失败")
                
                # 指数退避
                sleep(2 ** retry_count)
    
    # 7. SHA256 完整性校验
    emit_progress_event({"state": "verifying"})
    
    expected_hash = manifest.model_info.sha256
    actual_hash = calculate_sha256(temp_file)
    
    if actual_hash != expected_hash:
        log(f"SHA256 校验失败: {actual_hash} != {expected_hash}")
        delete_file(temp_file)
        
        # 重试下载
        if retry_count < max_retries:
            log("校验失败，重新下载")
            return download_model_on_first_use()
        else:
            raise VerifyError("文件校验失败，请联系客服")
    
    # 8. 移动到正式目录
    move_file(temp_file, final_path)
    log("✅ 模型下载完成")
    
    # 9. 更新注册表
    update_model_registry({
        "model_id": manifest.model_info.id,
        "version": manifest.model_info.version,
        "downloaded_at": current_timestamp(),
        "file_path": final_path,
        "sha256": actual_hash
    })
    
    # 10. 加载模型
    emit_progress_event({"state": "available", "progress": 100})
    return load_model_from_local()


def check_model_exists_and_valid():
    """
    检查本地模型是否存在且有效
    """
    if not exists(final_path):
        return False
    
    # 读取注册表
    registry = load_model_registry()
    if model_id not in registry:
        return False
    
    # 快速校验（可选：跳过 SHA256，只检查文件大小）
    expected_size = manifest.model_info.size
    actual_size = get_file_size(final_path)
    
    if actual_size != expected_size:
        log("文件大小不匹配，需要重新下载")
        return False
    
    # 完整校验（首次启动时）
    if not registry[model_id].get("verified"):
        expected_hash = manifest.model_info.sha256
        actual_hash = calculate_sha256(final_path)
        
        if actual_hash != expected_hash:
            log("SHA256 校验失败，需要重新下载")
            delete_file(final_path)
            return False
        
        # 标记为已校验
        registry[model_id]["verified"] = True
        save_model_registry(registry)
    
    return True
```

```javascript
class ModelVersionChecker {
  constructor() {
    this.checkInterval = 24 * 60 * 60 * 1000; // 24小时检查一次
  }

  async checkForUpdates() {
    try {
      // 1. 获取最新 manifest
      const latestManifest = await this.fetchManifest();
      
      // 2. 读取本地注册表
      const registry = await this.loadRegistry();
      
      // 3. 比较版本
      for (const model of latestManifest.models) {
        const localVersion = registry[model.id]?.version;
        const remoteVersion = model.version;
        
        if (!localVersion || this.isNewer(remoteVersion, localVersion)) {
          console.log(`🆕 发现新版本: ${model.id} ${localVersion} → ${remoteVersion}`);
          
          // 4. 提示用户更新
          this.notifyUpdate(model);
        }
      }
    } catch (error) {
      console.error('检查更新失败:', error);
    }
  }

  isNewer(remoteVersion, localVersion) {
    // 简单版本比较（实际应使用 semver）
    return remoteVersion > localVersion;
  }

  notifyUpdate(model) {
    // 发送通知到渲染进程
    const { BrowserWindow } = require('electron');
    const win = BrowserWindow.getFocusedWindow();
    
    if (win) {
      win.webContents.send('model-update-available', {
        modelId: model.id,
        name: model.name,
        currentVersion: '旧版本',
        latestVersion: model.version,
        changelog: '性能优化、准确率提升',
      });
    }
  }

  startAutoCheck() {
    // 首次检查
    this.checkForUpdates();
    
    // 定时检查
    setInterval(() => {
      this.checkForUpdates();
    }, this.checkInterval);
  }
}
```

## X.X 下载进度展示方案（新增）

### 一、目标

- 在模型按需下载场景中，提供清晰、实时的下载与校验进度展示；
- 支持断点续传后进度恢复；
- 在“下载”与“校验”两个阶段分别显示独立进度信息；
- 用户可随时暂停、继续或取消下载，校验失败时给出明确提示与重试选项。

---

### 二、UI 形式（显示项）

- **百分比**：显示总下载/校验进度的百分比（例如 0.0% ~ 100.0%），精确到 0.1%。
- **进度条**：视觉化进度条显示已完成比例（下载与校验分别用两条或分段式进度条）。
- **下载速度**：当前瞬时速度（例如 2.3 MB/s），可用滑动平均平滑抖动。
- **已下载 / 总大小**：例如 1.2 GB / 2.1 GB。
- **剩余时间（ETA）**：基于当前速度估算剩余秒数并格式化显示（秒/分钟/小时）。
- **阶段标签**：明确标注当前阶段：Downloading / Checking / Paused / Failed / Completed。
- **操作按钮**：暂停、继续、取消、重试（校验失败场景）。

---

### 三、进度更新策略

- **更新周期**：UI 每 100ms 至 500ms 更新一次（默认 200ms）。
  - 过于频繁会导致 UI 卡顿；过于稀疏会降低交互感。可在低性能设备上适当延长到 500ms。
- **事件驱动 + 定时器混合**：以下载器事件（每个分片完成）驱动精确更新，并由定时器（200ms）触发界面刷新以平滑速度/ETA计算。
- **速度计算**：使用 N 秒滑动窗口（默认 3 秒）计算平均速度，避免瞬时抖动。

---

### 四、断点续传的进度恢复方式

流程说明：
1. 启动或恢复下载时，检查临时文件（例如 `model.bin.part`）的当前字节数；
2. 将已下载字节数作为起点，重新计算剩余分片并继续发起 Range 请求；
3. UI 将基于本地已下载字节数立即恢复进度显示（progress = downloaded / total * 100）；
4. 若 manifest 或模型大小变化，提示用户并提供重新开始或继续（兼容策略）。

要点：
- 临时文件大小即为“已下载”字节数，必须在下载器启动前读取并通知渲染进程；
- 在恢复时，先发送一次 progress 事件到 UI（以避免短暂空白）。

---

### 五、下载与校验两个阶段的独立进度显示

- **下载阶段（Downloading）**：显示下载进度条、百分比、速度、已下载/总大小、ETA；当分片并发下载时，显示总体已完成比例即可。
- **校验阶段（Checking）**：下载完成后进入校验阶段，切换显示到“校验进度条”（可显示校验已处理字节 / 总字节或简单旋转动画 + 文案“正在校验文件完整性”）。
- **UI 切换规则**：当下载器报告 `state='verifying'`，将下载进度条置为 100% 并显示校验进度视图；校验过程中继续显示取消按钮（但暂停没有意义），校验失败则进入“失败处理”视图。

---

### 六、校验失败重试时的 UI 提示逻辑

流程与交互：
1. 校验失败（SHA256 校验不匹配）触发 `VERIFY_FAILED` 状态；
2. UI 显示明确的错误信息："校验失败：文件损坏或不完整"，并显示三选项："重试下载"、"切换备用源"、"取消并上报"；
3. 若用户选择自动重试，展示退避重试次数（例如重试 3 次）；每次重试前显示倒计时与当前尝试次数；
4. 若重试全部失败，提示用户联系支持并提供错误日志上传选项。

交互要点：
- 在重试期间，按钮变为“停止重试”；显示当前重试次数和下一次重试倒计时；
- 切换备用源时，提示来源变化且记录到日志以便排查。

---

### 七、用户操作（取消 / 暂停 / 继续）

- **取消（Cancel）**：立即停止所有下载请求，删除临时文件或保留（取决于用户选择：保留可恢复 / 删除释放空间）；UI 返回“未下载”或“已取消”状态。
- **暂停（Pause）**：暂停所有活跃请求，保留临时文件并更新状态为 `Paused`，UI 显示已暂停并保留当前进度；
- **继续（Resume）**：从临时文件大小处恢复分片队列并继续下载；UI 切回 `Downloading`。

注意：
- 对用户友好：在磁盘空间不足或网络条件差时，给出明确提示并建议在 Wi‑Fi 下重试；
- 在“取消但保留临时文件”模式下，下次恢复可节省已下载流量。

---

### 八、状态机（简化版）

```
States: Downloading / Paused / Checking / Failed / Completed

Transitions:
  Idle -> Downloading
  Downloading -> Paused
  Paused -> Downloading
  Downloading -> Checking
  Checking -> Completed
  Checking -> Failed
  Failed -> Downloading (retry)
  Any -> Cancelled
```

UI 映射：
- Downloading -> 显示下载进度条、速度、ETA、暂停/取消按钮；
- Paused -> 显示已暂停视图、继续/取消按钮；
- Checking -> 显示校验动画与校验进度；
- Failed -> 显示错误信息与重试/切换源/取消按钮；
- Completed -> 显示完成并启用“开始使用”按钮。

### 十、可配置项与建议

- `ui_refresh_interval_ms`：默认 `200` ms，可配置（范围 100 ~ 500 ms）；
- `speed_window_seconds`：速度计算滑动窗口，默认 3 秒；
- `chunk_size`：默认 10MB，移动网络时可降低到 1-4MB；
- `max_concurrent`：默认 3，设备性能差时设为 1；
- `max_retries`：校验/下载失败的最大重试次数，推荐 3 次。

### 十一、实施要点与验收标准

1. **首次恢复进度**：发现临时 `.part` 文件时，UI 应在 1s 内展示恢复后的进度值；
2. **进度准确性**：下载完成后进度应达到 100%，校验通过并切换到 Completed；
3. **重试反馈**：校验失败时展示用户友好的重试计数与切换备用源入口；
4. **用户控制**：暂停/继续/取消在 UI 上可直接操作并生效；
5. **性能**：在常见消费级网络（50Mbps）下，UI 更新无卡顿且速度/ETA计算稳定。
