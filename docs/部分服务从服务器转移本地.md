## 一、设计目的

### 1.1 核心目标
- **安装包轻量化**：禁止将模型文件（约 2.3GB）打包到安装程序中，确保安装包大小 < 200MB
- **按需下载**：用户首次启动软件时，自动触发模型下载流程
- **用户体验优化**：提供可视化下载进度、支持断点续传、确保下载失败不影响软件基础功能

### 1.2 业务价值
- **降低用户门槛**：安装包小（< 200MB），下载快，降低用户首次使用门槛
- **节省成本**：仅需要 AI 功能的用户才下载模型，节省 CDN 流量成本
- **提升可用性**：下载失败可降级运行基础功能，不影响软件整体可用性

### 2.2 断点续传实现方案

**技术方案**：基于 HTTP Range 请求

**实现步骤**：

1. **记录下载进度**：
   - 在本地存储已下载的字节数（`download_progress.json`）
   - 格式：
     ```json
     {
       "downloaded_bytes": 1024000,
       "total_bytes": 2400000000,
       "cdn_index": 0,
       "retry_count": 1,
       "last_update": "2025-11-15T10:30:00Z"
     }
     ```

2. **Range 请求**：
   ```http
   GET /models/sensevoice_small.onnx HTTP/1.1
   Host: cdn.example.com
   Range: bytes=1024000-
   ```

3. **分片下载**：
   - 每次下载 10MB 分片
   - 每个分片下载完成后更新进度文件
   - 支持随时暂停和恢复

**伪代码**：
```python
def download_with_resume(url, local_path, progress_file):
    # 读取已下载的字节数
    downloaded_bytes = load_progress(progress_file)
    
    # 设置 Range 请求头
    headers = {"Range": f"bytes={downloaded_bytes}-"}
    
    # 发起请求
    response = requests.get(url, headers=headers, stream=True)
    total_bytes = int(response.headers.get("Content-Length", 0)) + downloaded_bytes
    
    # 追加写入文件
    with open(local_path, "ab") as f:
        for chunk in response.iter_content(chunk_size=10*1024*1024):  # 10MB
            f.write(chunk)
            downloaded_bytes += len(chunk)
            save_progress(progress_file, downloaded_bytes, total_bytes)
            notify_progress(downloaded_bytes, total_bytes)
```

### 2.4 SHA256 校验流程

**校验时机**：
1. 下载完成后立即校验
2. 每次软件启动时校验现有模型（可选）

**校验流程**：
```
下载完成
    ↓
计算文件 SHA256 哈希值
    ↓
与预期哈希值对比
    ├── 匹配 → 标记下载完成，删除进度文件
    └── 不匹配 → 触发重试流程
```

**预期哈希值存储**（model_manifest.json）：
```json
{
  "model_name": "sensevoice_small.onnx",
  "version": "1.0.0",
  "sha256": "a3d5f8e9b2c1d4e6f7a8b9c0d1e2f3a4b5c6d7e8f9a0b1c2d3e4f5a6b7c8d9e0",
  "size_bytes": 2400000000,
  "download_urls": ["..."]
}
```

**伪代码**：
```python
import hashlib

def verify_model(file_path, expected_hash):
    sha256 = hashlib.sha256()
    with open(file_path, "rb") as f:
        for chunk in iter(lambda: f.read(4096), b""):
            sha256.update(chunk)
    
    actual_hash = sha256.hexdigest()
    
    if actual_hash == expected_hash:
        return True, "校验成功"
    else:
        return False, f"校验失败: 期望 {expected_hash}, 实际 {actual_hash}"
```

### 2.5 自动重试策略

**重试场景**：
1. 下载过程中网络中断
2. SHA256 校验失败
3. CDN 源返回错误

**重试策略**：
```
重试配置：
- 最大重试次数：3 次
- 重试延迟：指数退避（1s, 2s, 4s）
- CDN 切换：每次重试切换到下一个 CDN 源
```

**状态机**：
```
Idle (空闲)
  ↓ [触发下载]
Downloading (下载中)
  ├── [网络中断] → Paused (暂停)
  ├── [用户取消] → Cancelled (已取消)
  ├── [下载完成] → Verifying (校验中)
  └── [下载失败] → Failed (失败)
  
Paused (暂停)
  ├── [恢复下载] → Downloading
  └── [重新启动软件] → 检查进度 → Downloading
  
Verifying (校验中)
  ├── [校验成功] → Completed (完成)
  └── [校验失败] → Retry (重试)
  
Failed (失败)
  ├── [重试次数 < 3] → Retry
  └── [重试次数 ≥ 3] → Error (错误)
  
Retry (重试)
  ↓ [延迟后]
  ├── [切换 CDN] → Downloading
  └── [无可用 CDN] → Error
  
Completed (完成)
  ↓
  软件正常启动
  
Error (错误)
  ↓
  降级模式启动（不加载 AI 功能）
```

**伪代码**：
```python
class ModelDownloader:
    def __init__(self):
        self.state = "Idle"
        self.retry_count = 0
        self.max_retries = 3
        self.current_cdn_index = 0
    
    async def download(self):
        self.state = "Downloading"
        
        while self.retry_count < self.max_retries:
            try:
                # 选择 CDN 源
                cdn = self.cdn_sources[self.current_cdn_index]
                
                # 下载文件
                await self.download_from_cdn(cdn)
                
                # 校验文件
                self.state = "Verifying"
                if self.verify_sha256():
                    self.state = "Completed"
                    return True
                else:
                    raise VerificationError("SHA256 校验失败")
                    
            except (NetworkError, VerificationError) as e:
                self.retry_count += 1
                self.state = "Retry"
                
                if self.retry_count < self.max_retries:
                    # 切换 CDN
                    self.current_cdn_index = (self.current_cdn_index + 1) % len(self.cdn_sources)
                    # 指数退避
                    await asyncio.sleep(2 ** self.retry_count)
                else:
                    self.state = "Error"
                    return False
```

### 2.7 失败处理与恢复流程

**场景 1：网络异常**
```
下载中断（网络断开）
    ↓
保存当前进度到本地文件
    ↓
状态切换到 "Paused"
    ↓
显示提示：网络连接中断，已保存进度
    ↓
用户选择：
    ├── 立即重试 → 恢复下载
    ├── 稍后重试 → 关闭提示，下次启动继续
    └── 取消下载 → 删除临时文件
```

**场景 2：SHA256 校验失败**
```
下载完成，开始校验
    ↓
计算 SHA256 哈希值
    ↓
哈希值不匹配
    ↓
删除已下载的文件
    ↓
重试计数 +1
    ↓
切换到下一个 CDN 源
    ↓
延迟后重新下载（指数退避）
    ↓
如重试次数 ≥ 3：
    └── 提示用户：模型下载失败，请检查网络或联系客服
```

**场景 3：用户取消下载**
```
用户点击"取消"按钮
    ↓
弹出确认对话框：
    "取消后将删除下载进度，是否确认？"
    ├── 确认 → 中止下载，删除临时文件和进度记录
    └── 取消 → 继续下载
```

**场景 4：软件二次启动检查**
```
软件启动
    ↓
检查模型文件状态
    ├── 文件完整且校验通过 → 直接加载模型
    ├── 文件不存在 → 触发下载流程
    ├── 文件存在但校验失败 → 删除文件，重新下载
    └── 发现未完成的下载进度 →
        ↓
        弹出提示：
        "检测到未完成的模型下载（已完成 45%），是否继续？"
        ├── 继续下载 → 从断点恢复
        ├── 重新下载 → 删除进度，从头开始
        └── 暂不下载 → 降级模式启动
```

### 3.2 分片下载 / Range 请求机制

**分片策略**：
- **分片大小**：10 MB（可根据网络速度动态调整）
- **并发下载**：单线程顺序下载（简化实现，避免并发复杂度）
- **Range 请求格式**：
  ```http
  Range: bytes=0-10485759        # 第 1 片（0-10MB）
  Range: bytes=10485760-20971519 # 第 2 片（10-20MB）
  Range: bytes=20971520-         # 最后一片（剩余部分）
  ```

**进度持久化**：
```json
{
  "model_name": "sensevoice_small.onnx",
  "download_url": "https://cdn1.timao.live/models/sensevoice_small.onnx",
  "total_bytes": 2400000000,
  "downloaded_bytes": 1048576000,
  "chunk_size": 10485760,
  "last_update": "2025-11-15T10:30:00Z",
  "cdn_index": 0,
  "retry_count": 1
}
```

**实现伪代码**：
```python
async def download_with_chunks(url, local_path, chunk_size=10*1024*1024):
    # 获取文件总大小
    response = await session.head(url)
    total_size = int(response.headers["Content-Length"])
    
    # 加载已下载的进度
    downloaded = load_progress() or 0
    
    # 计算起始位置
    start_byte = downloaded
    
    while start_byte < total_size:
        end_byte = min(start_byte + chunk_size - 1, total_size - 1)
        
        # Range 请求
        headers = {"Range": f"bytes={start_byte}-{end_byte}"}
        chunk_response = await session.get(url, headers=headers)
        
        # 追加写入文件
        with open(local_path, "ab") as f:
            async for data in chunk_response.content.iter_chunked(1024):
                f.write(data)
                start_byte += len(data)
                
                # 每下载 1MB 保存一次进度
                if start_byte % (1024*1024) == 0:
                    save_progress(start_byte, total_size)
                    notify_progress(start_byte, total_size)
        
        # 更新起始位置
        start_byte = end_byte + 1
```

### 3.3 SHA256 校验流程

**校验实现**：
```python
import hashlib

def calculate_sha256(file_path, chunk_size=8192):
    """
    计算文件的 SHA256 哈希值
    
    Args:
        file_path: 文件路径
        chunk_size: 每次读取的块大小（8KB）
    
    Returns:
        str: 64位十六进制哈希值
    """
    sha256_hash = hashlib.sha256()
    
    with open(file_path, "rb") as f:
        # 分块读取，避免大文件占用过多内存
        for chunk in iter(lambda: f.read(chunk_size), b""):
            sha256_hash.update(chunk)
    
    return sha256_hash.hexdigest()

def verify_model_integrity(file_path, expected_hash):
    """
    校验模型文件完整性
    
    Returns:
        (bool, str): (是否通过, 消息)
    """
    if not os.path.exists(file_path):
        return False, "文件不存在"
    
    actual_hash = calculate_sha256(file_path)
    
    if actual_hash.lower() == expected_hash.lower():
        return True, "校验通过"
    else:
        return False, f"校验失败\n期望: {expected_hash}\n实际: {actual_hash}"
```

**校验时机**：
1. 下载完成后立即校验
2. 软件每次启动时快速校验（可选，建议启用）
3. 用户手动触发校验（设置页面提供按钮）

### 3.4 CDN 切换策略

**切换触发条件**：
1. 连接超时（30 秒无响应）
2. 下载速度过慢（< 100 KB/s 持续 10 秒）
3. HTTP 错误（4xx, 5xx）
4. SHA256 校验失败（怀疑 CDN 文件损坏）

**切换逻辑**：
```python
class CDNManager:
    def __init__(self, cdn_list):
        self.cdn_list = cdn_list  # 按优先级排序
        self.current_index = 0
        self.failed_cdns = set()
    
    def get_current_cdn(self):
        return self.cdn_list[self.current_index]
    
    def switch_to_next(self):
        """切换到下一个可用的 CDN"""
        # 标记当前 CDN 失败
        self.failed_cdns.add(self.current_index)
        
        # 寻找下一个未失败的 CDN
        for i in range(len(self.cdn_list)):
            next_index = (self.current_index + 1 + i) % len(self.cdn_list)
            if next_index not in self.failed_cdns:
                self.current_index = next_index
                return self.get_current_cdn()
        
        # 所有 CDN 都失败
        raise NoCDNAvailableError("所有 CDN 源均不可用")
    
    def reset_failures(self):
        """重置失败记录（用于重试）"""
        self.failed_cdns.clear()
        self.current_index = 0
```

**健康检查**（可选优化）：
```python
async def cdn_health_check(cdn_url):
    """
    检查 CDN 健康状态
    返回：(是否可用, 延迟ms)
    """
    try:
        start = time.time()
        response = await session.head(cdn_url, timeout=5)
        latency = (time.time() - start) * 1000
        
        if response.status == 200:
            return True, latency
        else:
            return False, float('inf')
    except:
        return False, float('inf')

async def select_best_cdn(cdn_list):
    """选择延迟最低的 CDN"""
    results = await asyncio.gather(*[
        cdn_health_check(cdn["url"]) for cdn in cdn_list
    ])
    
    # 按延迟排序
    sorted_cdns = sorted(
        zip(cdn_list, results),
        key=lambda x: x[1][1]  # 按延迟排序
    )
    
    # 返回第一个可用的 CDN
    for cdn, (available, latency) in sorted_cdns:
        if available:
            return cdn
    
    # 如果都不可用，返回优先级最高的
    return cdn_list[0]
```

### 3.6 降级逻辑（确保软件可启动）

**降级策略**：
```
模型下载失败
    ↓
检测失败原因：
    ├── 网络不可用 → 提示用户检查网络
    ├── 所有 CDN 均失败 → 提示联系客服
    └── 用户取消下载 → 记录用户选择
    ↓
降级模式启动：
    ├── 禁用语音转写功能
    ├── 禁用 AI 智能回复功能
    ├── 保留弹幕拉取、礼物监控等基础功能
    └── 在 UI 显示警告：部分功能不可用
```

**UI 提示设计**：

**降级逻辑实现**：
```python
class ASRService:
    def __init__(self):
        self.model_loaded = False
        self.degraded_mode = False
    
    async def initialize(self):
        """初始化服务"""
        try:
            # 检查模型是否存在
            if not self.check_model_exists():
                # 触发下载
                success = await self.download_model()
                if not success:
                    self.enter_degraded_mode()
                    return
            
            # 加载模型
            self.load_model()
            self.model_loaded = True
            
        except Exception as e:
            logger.error(f"模型初始化失败: {e}")
            self.enter_degraded_mode()
    
    def enter_degraded_mode(self):
        """进入降级模式"""
        self.degraded_mode = True
        self.model_loaded = False
        logger.warning("进入降级模式：语音转写功能不可用")
    
    async def transcribe(self, audio_data):
        """语音转写（降级模式下抛出异常）"""
        if not self.model_loaded:
            raise ServiceUnavailableError(
                "语音识别服务不可用，请先下载模型"
            )
        
        # 正常转写逻辑
        return await self._do_transcribe(audio_data)
```

## 四、关键时序图与流程

### 4.1 首次启动完整时序图（文本描述）

```
用户 → Electron 主进程 → 模型管理器 → CDN 服务器 → 渲染进程

1. 用户双击启动软件
   ↓
2. Electron 主进程启动
   ↓
3. 调用模型管理器.checkModel()
   ↓
4. 检查本地文件：models/sensevoice_small.onnx
   ├── 文件存在 → 校验 SHA256
   │   ├── 校验通过 → [跳到步骤 15]
   │   └── 校验失败 → 删除文件，继续下载流程
   └── 文件不存在 → 继续下载流程
   ↓
5. 检查下载进度文件：cache/download_progress.json
   ├── 存在未完成进度 → 提示用户是否继续
   │   ├── 用户选择继续 → 从断点恢复
   │   └── 用户选择重新下载 → 删除进度文件
   └── 不存在 → 开始新下载
   ↓
6. 选择 CDN 源（优先级最高的）
   ↓
7. 发起 HTTP HEAD 请求，获取文件大小
   ↓
8. 创建临时文件：sensevoice_small.onnx.part
   ↓
9. 发起 Range 请求下载分片
   ↓
10. 每下载 1MB：
    ├── 更新进度文件
    ├── 通过 IPC 通知渲染进程
    └── 渲染进程更新 UI 进度条
   ↓
11. 下载过程中可能发生：
    ├── 网络中断 → 保存进度，状态切换到 Paused
    ├── 用户暂停 → 保存进度，状态切换到 Paused
    ├── 用户取消 → 删除临时文件，状态切换到 Cancelled
    └── CDN 失败 → 切换到下一个 CDN，继续下载
   ↓
12. 下载完成，重命名：.part → .onnx
   ↓
13. 计算 SHA256 哈希值
   ↓
14. 校验哈希值
    ├── 校验通过 → 删除进度文件，状态切换到 Completed
    └── 校验失败 → 
        ├── 重试次数 < 3 → 切换 CDN，延迟后重试
        └── 重试次数 ≥ 3 → 进入降级模式
   ↓
15. 加载模型到内存
   ↓
16. 启动本地 FastAPI 服务（端口 16000）
   ↓
17. 渲染进程加载完成，显示主界面
   ↓
18. 用户可正常使用所有功能
```

## 五、核心实现代码

### 5.1 Python 模型下载管理器

完整的 `ModelDownloader` 类实现请参考本文档第二章节的伪代码示例。

### 5.2 Electron 集成示例

```javascript
// electron/main/modelManager.js
const { ipcMain } = require('electron');
const { spawn } = require('child_process');
const path = require('path');

class ModelManager {
  constructor(mainWindow) {
    this.mainWindow = mainWindow;
    this.downloader = null;
    this.setupIPC();
  }
  
  setupIPC() {
    // 开始下载
    ipcMain.on('model-download-start', async () => {
      await this.startDownload();
    });
    
    // 暂停下载
    ipcMain.on('model-download-pause', () => {
      this.pauseDownload();
    });
    
    // 取消下载
    ipcMain.on('model-download-cancel', () => {
      this.cancelDownload();
    });
  }
  
  async startDownload() {
    const pythonPath = process.env.PYTHON_PATH || 'python';
    const scriptPath = path.join(__dirname, '../../server/local/download_model.py');
    
    this.downloader = spawn(pythonPath, [scriptPath]);
    
    // 监听标准输出（进度信息）
    this.downloader.stdout.on('data', (data) => {
      try {
        const progress = JSON.parse(data.toString());
        this.mainWindow.webContents.send('model-download-progress', progress);
      } catch (e) {
        console.log(data.toString());
      }
    });
    
    // 监听错误输出
    this.downloader.stderr.on('data', (data) => {
      console.error(`下载错误: ${data}`);
      this.mainWindow.webContents.send('model-download-error', data.toString());
    });
    
    // 监听进程退出
    this.downloader.on('close', (code) => {
      if (code === 0) {
        this.mainWindow.webContents.send('model-download-complete');
      } else {
        this.mainWindow.webContents.send('model-download-failed', code);
      }
    });
  }
  
  pauseDownload() {
    if (this.downloader) {
      this.downloader.kill('SIGTERM');
    }
  }
  
  cancelDownload() {
    if (this.downloader) {
      this.downloader.kill('SIGKILL');
    }
  }
}

module.exports = ModelManager;
```
