# 🧹 数据清理机制说明

## 📋 概述

为了防止用户重新登录后看到旧用户的数据残留，系统实现了自动数据清理机制。每次用户登录成功后，系统会自动清理所有直播相关的旧数据。

## 🎯 设计目标

1. **防止数据混淆**：避免新用户看到旧用户的直播数据
2. **保护隐私**：确保用户数据不会泄露给其他用户
3. **提升体验**：用户登录后看到的是干净的、全新的界面
4. **保留必要数据**：不清理用户设置、账号密码等需要保留的数据

## 🔧 实现方式

### 1. 清理触发时机

**登录成功后立即清理**（`LoginPage.tsx`）

```typescript
if (response.success) {
  // 🧹 清理旧用户的直播控制台数据
  console.log('🧹 正在清理旧的直播控制台数据...');
  
  // 1. 清理 Zustand store 中的数据
  clearAllData();
  
  // 2. 清理 storage 中的临时数据
  cleanupLiveData();
  
  // 3. 继续登录流程...
}
```

### 2. 清理的数据范围

#### 2.1 Zustand Store 数据（`useLiveConsoleStore`）

清理的状态包括：

- ✅ `liveInput` - 直播输入框内容
- ✅ `status` - 直播状态
- ✅ `latest` - 最新转写记录
- ✅ `log` - 转写记录日志（完整历史）
- ✅ `mode` - 转写模式
- ✅ `error` - 错误信息
- ✅ `backendLevel` - 音频级别
- ✅ `confSum` / `confCount` - 置信度统计
- ✅ `reportPaths` - 报告路径
- ✅ `reportStatus` - 报告状态
- ✅ `saveInfo` - 保存信息
- ✅ `styleProfile` - 主播风格画像
- ✅ `vibe` - 直播间氛围分析
- ✅ `persistTr` / `persistTrRoot` - 转写持久化设置
- ✅ `persistDm` / `persistDmRoot` - 弹幕持久化设置
- ✅ `aiEvents` - AI事件列表
- ✅ `answerScripts` - 应答话术列表

#### 2.2 SessionStorage 数据（`dataCleanup.ts`）

清理的存储项包括：

- ✅ `timao-live-console` - 直播控制台持久化数据
- ✅ `timao-live-status` - 直播状态
- ✅ `timao-websocket-state` - WebSocket 状态

#### 2.3 LocalStorage 临时数据

清理的临时数据包括：

- ✅ `timao-temp-transcript` - 临时转写数据
- ✅ `timao-temp-comments` - 临时弹幕数据

### 3. 不清理的数据

为了保持用户体验，以下数据**不会被清理**：

- ❌ **不清理** 认证 token（由 `useAuthStore` 管理）
- ❌ **不清理** 用户设置和偏好
- ❌ **不清理** 保存的账号密码（由 `useAccountStore` 管理）
- ❌ **不清理** API 配置

## 📁 相关文件

### 核心文件

1. **`electron/renderer/src/store/useLiveConsoleStore.ts`**
   - 实现 `clearAllData()` 方法
   - 清理所有 Zustand store 状态
   - 断开 WebSocket 连接
   - 清除 sessionStorage

2. **`electron/renderer/src/utils/dataCleanup.ts`**
   - 通用数据清理工具
   - `cleanupLiveData()` - 清理直播相关数据
   - `cleanupAllUserData()` - 清理所有用户数据（登出时）
   - `reportStorageUsage()` - 调试工具，报告存储使用情况

3. **`electron/renderer/src/pages/auth/LoginPage.tsx`**
   - 登录成功后调用清理逻辑
   - 先清理 store，再清理 storage

## 🚨 常见问题

### Q1: 为什么登录后数据还在？

**可能原因：**
1. 浏览器缓存导致前端代码未更新
2. 清理逻辑执行失败（检查控制台错误）
3. 数据存储在其他地方（不在清理范围内）

**解决方法：**
1. 硬刷新浏览器（Ctrl+Shift+R 或 Cmd+Shift+R）
2. 检查浏览器控制台的清理日志
3. 手动清理：
   ```javascript
   // 在浏览器控制台执行
   sessionStorage.clear();
   ```

### Q2: 清理会影响登录状态吗？

**不会。** 清理逻辑只清理**直播相关的数据**，不会清理：
- 认证 token
- 用户信息
- 用户设置

### Q3: 如何恢复误清理的数据？

**无法恢复。** 清理是**不可逆操作**。但以下数据可以恢复：
- 如果开启了"持久化转写"，转写记录保存在服务器
- 如果开启了"持久化弹幕"，弹幕保存在服务器
- AI 分析结果可以重新生成

## 📝 开发注意事项

### 添加新的状态时

如果你在 `useLiveConsoleStore` 中添加了新的状态，**务必**同步更新 `clearAllData()` 方法：

```typescript
clearAllData: () => {
  // ...
  set({
    // ... 现有状态
    newState: initialValue,  // ← 添加新状态的初始值
  });
  // ...
}
```

### 添加新的存储项时

如果你在 sessionStorage 或 localStorage 中添加了新的直播相关数据，**务必**同步更新 `dataCleanup.ts`：

```typescript
const itemsToClean = [
  'timao-live-console',
  'timao-new-item',  // ← 添加新的存储项
  // ...
];
```

## 🎉 总结

数据清理机制确保了：
1. ✅ 用户隐私保护
2. ✅ 数据安全隔离
3. ✅ 良好的用户体验
4. ✅ 系统状态一致性

**重要**：清理机制只在登录时触发，日常使用中不会自动清理数据。这样既保证了安全性，又不影响正常使用。
