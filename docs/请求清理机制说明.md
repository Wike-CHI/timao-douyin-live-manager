# 请求清理机制说明

## 📝 概述

本文档说明了提猫直播助手应用的请求清理机制,确保当用户关闭应用时,所有正在进行的后端请求和定时器都能被正确终止,避免服务器资源浪费。

## 🎯 问题背景

在使用远程后端服务的 Electron 应用中,存在以下资源管理问题:

1. **HTTP 请求未取消**: 用户关闭应用后,正在进行的 HTTP 请求可能仍在执行
2. **定时器未清除**: 定时轮询任务(如健康检查、数据刷新)继续运行
3. **服务器资源浪费**: 后端继续处理已关闭客户端的请求,造成资源浪费
4. **连接泄漏**: 长连接或 WebSocket 连接未正确关闭

## 🏗️ 架构设计

### 核心组件

```
┌─────────────────────────────────────────────────┐
│               Electron 主进程                    │
│  ┌──────────────────────────────────────────┐   │
│  │  app.on('before-quit', ...)              │   │
│  │  - 发送清理信号到渲染进程                  │   │
│  │  - 等待渲染进程清理完成 (500ms)            │   │
│  │  - 停止本地后端服务 (如果有)                │   │
│  └──────────────────────────────────────────┘   │
└─────────────────────────────────────────────────┘
                      ↓ IPC 消息
                      ↓ 'app-cleanup-request'
┌─────────────────────────────────────────────────┐
│              Electron 渲染进程                   │
│  ┌──────────────────────────────────────────┐   │
│  │  appCleanupService                       │   │
│  │  - 监听清理信号                            │   │

│  └──────────────────────────────────────────┘   │
│                    ↓                            │
│  ┌──────────────────────────────────────────┐   │

│  │  - 清除所有 setTimeout                    │   │

└─────────────────────────────────────────────────┘
```

## 📦 实现细节

### 1. RequestManager (请求管理器)

**文件**: `electron/renderer/src/services/requestManager.ts`

**功能**:
- 追踪所有正在进行的 HTTP 请求 (`AbortController`)
- 追踪所有定时器 (`setTimeout`, `setInterval`)
- 提供全局清理方法
- 支持注册自定义清理回调

**API**:
```typescript
// 创建被追踪的 AbortController
const controller = requestManager.createAbortController();

// 创建被追踪的定时器
const timerId = requestManager.createTimeout(() => {...}, 1000);
const intervalId = requestManager.createInterval(() => {...}, 5000);

// 清除定时器
requestManager.clearTimeout(timerId);
requestManager.clearInterval(intervalId);

// 注册清理回调
requestManager.registerCleanup(() => {
  console.log('执行自定义清理逻辑');
});

// 执行全局清理
requestManager.cleanup();
```

### 2. HTTP 请求自动追踪

**文件**: `electron/renderer/src/services/http.ts`

所有通过 `fetchJsonWithAuth` 发起的请求都会自动使用被追踪的 `AbortController`:

```typescript
export const fetchJsonWithAuth = async (
  service: keyof ApiConfig['services'],
  path: string,
  options: RequestInit = {},
  overrideBaseUrl?: string
) => {
  // 自动创建被追踪的 AbortController
  const controller = requestManager.createAbortController();
  
  return fetch(buildServiceUrl(service, path, overrideBaseUrl), {
    ...options,
    headers,
    signal: controller.signal, // 附加 signal
  });
};
```

### 3. API 配置管理器集成

**文件**: `electron/renderer/src/services/apiConfig.ts`

`ApiConfigManager` 的所有请求和定时器都已集成 `requestManager`:

- ✅ 健康检查定时器使用 `createInterval`
- ✅ 健康检查请求使用 `createAbortController`
- ✅ 请求重试延迟使用 `createTimeout`
- ✅ 注册了清理回调,自动停止健康检查

### 4. 应用清理服务

**文件**: `electron/renderer/src/services/appCleanup.ts`

**功能**:
- 监听来自 Electron 主进程的清理信号
- 监听浏览器的 `beforeunload` 事件
- 调用 `requestManager.cleanup()` 执行清理

**自动初始化**:
```typescript
// 在 main.tsx 中自动导入
import './services/appCleanup';
```

### 5. Electron 主进程清理逻辑

**文件**: `electron/main.js`

```javascript
app.on('before-quit', async (event) => {
  console.log('[electron] App is about to quit. Cleaning up resources...');
  
  // 通知所有渲染进程清理资源
  const allWindows = BrowserWindow.getAllWindows();
  allWindows.forEach((win) => {
    if (!win.isDestroyed()) {
      win.webContents.send('app-cleanup-request');
    }
  });
  
  // 等待渲染进程清理完成
  await new Promise(resolve => setTimeout(resolve, 500));
  
  // 停止本地后端服务
  await stopBackendServices();
});
```

### 6. Preload 脚本 IPC 通信

**文件**: `electron/preload.js`

暴露安全的 IPC 通信接口:

```javascript
contextBridge.exposeInMainWorld('electron', {
  ipcRenderer: {
    on: (channel, callback) => {
      const validChannels = ['app-cleanup-request'];
      if (validChannels.includes(channel)) {
        ipcRenderer.on(channel, callback);
      }
    },
    removeListener: (channel, callback) => {
      // ...
    }
  }
});
```

## 🔄 清理流程

### 用户关闭应用时的完整流程

```
1. 用户点击关闭按钮
   ↓
2. Electron 触发 'before-quit' 事件
   ↓
3. 主进程发送 'app-cleanup-request' 到所有渲染进程
   ↓
4. 渲染进程接收信号 → appCleanupService.cleanup()
   ↓
5. requestManager.cleanup() 执行:
   - 取消 12 个正在进行的请求 (示例)
   - 清除 5 个定时器
   - 清除 3 个轮询定时器
   - 执行 2 个注册的清理回调
   ↓
6. 主进程等待 500ms 让清理完成
   ↓
7. 主进程停止本地后端服务 (如果有)
   ↓
8. 应用正常退出 (app.exit(0))
```

### 被清理的资源

#### HTTP 请求
- ✅ 健康检查请求
- ✅ API 数据请求
- ✅ 重试中的请求
- ✅ 所有通过 `fetchJsonWithAuth` 发起的请求

#### 定时器
- ✅ 健康检查定时器 (30秒间隔)
- ✅ 直播状态轮询 (LiveConsolePage)
- ✅ 报告数据轮询 (ReportsPage, 2秒间隔)
- ✅ 所有通过 `createTimeout/createInterval` 创建的定时器

#### 其他资源
- ✅ 自定义清理回调
- ✅ 本地后端服务进程

## 💡 使用指南

### 在现有代码中使用

#### 1. 发起 HTTP 请求

**推荐方式**: 使用 `fetchJsonWithAuth` (自动追踪)

```typescript
import { fetchJsonWithAuth } from './services/http';

// 请求会被自动追踪,应用关闭时自动取消
const response = await fetchJsonWithAuth('main', '/api/users');
```

**手动方式**: 使用 `createAbortController`

```typescript
import { requestManager } from './services/requestManager';

const controller = requestManager.createAbortController();
const response = await fetch('https://api.example.com/data', {
  signal: controller.signal
});
```

#### 2. 创建定时器

**推荐方式**: 使用 `createTimeout/createInterval`

```typescript
import { requestManager } from './services/requestManager';

// 定时器会被自动追踪,应用关闭时自动清除
const timerId = requestManager.createTimeout(() => {
  console.log('定时任务');
}, 5000);

const intervalId = requestManager.createInterval(() => {
  console.log('轮询任务');
}, 10000);

// 需要时可以手动清除
requestManager.clearTimeout(timerId);
requestManager.clearInterval(intervalId);
```

**避免使用**: 直接使用原生 API

```typescript
// ❌ 不推荐 - 应用关闭时不会自动清除
const timerId = setTimeout(() => {...}, 5000);
const intervalId = setInterval(() => {...}, 10000);
```

#### 3. 注册自定义清理逻辑

如果你有自定义的资源需要清理:

```typescript
import { requestManager } from './services/requestManager';

// 在组件初始化时注册清理回调
const cleanup = () => {
  console.log('清理我的自定义资源');
  // 关闭 WebSocket、清理缓存等
};

requestManager.registerCleanup(cleanup);

// 组件卸载时移除清理回调 (可选)
requestManager.unregisterCleanup(cleanup);
```

### React 组件中的使用

```tsx
import { useEffect, useRef } from 'react';
import { requestManager } from '../services/requestManager';

function MyComponent() {
  const intervalRef = useRef<number | null>(null);
  
  useEffect(() => {
    // 创建被追踪的轮询定时器
    intervalRef.current = requestManager.createInterval(() => {
      console.log('轮询数据...');
      fetchData();
    }, 5000);
    
    // 组件卸载时清除定时器
    return () => {
      if (intervalRef.current !== null) {
        requestManager.clearInterval(intervalRef.current);
      }
    };
  }, []);
  
  return <div>My Component</div>;
}
```

## 🔧 故障排查

### 问题: 请求没有被取消

**可能原因**:
1. 请求没有使用 `fetchJsonWithAuth` 或手动创建的 `AbortController`
2. 请求已经完成,无需取消

**解决方案**:
```typescript
// 确保所有请求都使用被追踪的 AbortController
import { fetchJsonWithAuth } from './services/http';
const response = await fetchJsonWithAuth('main', '/api/data');
```

### 问题: 定时器没有被清除

**可能原因**:
1. 使用了原生 `setTimeout/setInterval` 而不是 `requestManager` 的方法
2. 定时器在清理前已经执行完毕

**解决方案**:
```typescript
// 使用追踪的定时器创建方法
import { requestManager } from './services/requestManager';
const intervalId = requestManager.createInterval(() => {...}, 5000);
```

### 问题: 清理回调没有执行

**可能原因**:
1. 清理回调中抛出了异常
2. 清理回调没有正确注册

**解决方案**:
```typescript
// 确保清理回调不抛出异常
requestManager.registerCleanup(() => {
  try {
    // 你的清理逻辑
  } catch (error) {
    console.error('清理失败:', error);
  }
});
```
