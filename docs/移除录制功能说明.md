# 移除录制功能说明

**审查人**: 叶维哲  
**创建时间**: 2025-11-09  
**修改时间**: 2025-11-09

---

## 📋 改动原因

**用户需求**：服务器硬盘空间不足，需要移除视频/音频录制功能

**分析结论**：
- ✅ **实时转写已生成文本**：讯飞ASR实时转写，文本保存到 `transcripts.jsonl`
- ✅ **复盘只读文本**：复盘功能只读转写文本，不读音频/视频
- ❌ **录制无意义**：录制视频/音频只是占用空间，没有实际用途

---

## 🔧 已完成的改动

### 1. ✅ 移除音频保存功能（LiveAudioStreamService）

**文件**：`server/app/services/live_audio_stream_service.py`

**改动前**：
```python
# ffmpeg多输出：同时输出到pipe（转写）+ 文件（保存）
ffmpeg -i 直播流 \
  -map 0:a -ac 1 -ar 16000 -f s16le pipe:1 \  # 实时转写
  -map 0:a -ac 1 -ar 16000 -f wav live_audio.wav  # 保存音频 ❌
```

**改动后**：
```python
# 只输出到pipe，不保存音频
ffmpeg -i 直播流 \
  -vn -ac 1 -ar 16000 -f s16le pipe:1  # 只给ASR实时转写
```

**效果**：
- ✅ 不再保存 `records/sessions/*/live_audio.wav`
- ✅ 节省磁盘空间（~60MB/小时）

---

### 2. ✅ 标记视频录制API为废弃（LiveReportService）

**文件**：`server/app/api/live_report.py`

**改动**：
```python
@router.post("/start", deprecated=True)  # 标记为废弃
async def start_live_report(req: StartLiveReportRequest):
    """⚠️ 已废弃：建议使用 /api/live_audio/start（实时转写，不录制视频）"""
    logger.warning("⚠️ 警告：此API已废弃，将录制视频占用大量空间！")
    # ...
```

**废弃的API端点**：
- ❌ `POST /api/report/live/start` - 启动视频录制（已废弃）
- ❌ `POST /api/report/live/stop` - 停止视频录制（已废弃）
- ⚠️ `POST /api/report/live/generate` - 生成报告（仍可用，但读取旧数据）

**推荐使用**：
- ✅ `POST /api/live_audio/start` - 实时转写（不录制）
- ✅ `POST /api/live_review/generate` - 复盘报告（读取转写文本）

## 📊 新旧方案对比

| 方案 | 磁盘占用 | 转写延迟 | 复盘功能 | 推荐 |
|------|---------|---------|---------|------|
| **旧方案**（LiveReportService）| **500MB/小时**（视频）| 离线（30分钟延迟）| ✅ | ❌ 已废弃 |
| **新方案**（LiveAudioStreamService）| **0MB**（不保存）| 实时（秒级）| ✅ | ✅ 推荐 |

**空间节省**：
- 1小时直播：节省 500MB
- 10小时直播：节省 5GB
- 100小时直播：节省 50GB

## 🔄 数据流程（新方案）

```
┌─────────────────────────────────────────────────────┐
│ 1. 用户启动监听（前端）                              │
│    └─ POST /api/live_audio/start                    │
└─────────────┬───────────────────────────────────────┘
              │
              ↓
┌─────────────────────────────────────────────────────┐
│ 2. 服务器拉流（实时处理）                            │
│    ├─ ffmpeg拉流 → 提取音频（PCM）                  │
│    └─ 讯飞ASR → 实时转写 → 保存文本                 │
│         └─ transcripts.jsonl ✅                     │
└─────────────┬───────────────────────────────────────┘
              │
              ↓
┌─────────────────────────────────────────────────────┐
│ 3. 复盘分析（读取文本）                              │
│    └─ POST /api/live_review/generate               │
│         └─ 读取 transcripts.jsonl ✅                │
│         └─ AI分析 → 生成报告                        │
└─────────────────────────────────────────────────────┘

❌ 不再保存：
   - ❌ 视频文件（*.mp4）
   - ❌ 音频文件（*.wav）
```

## ⚠️ 重要说明

### 旧数据处理

**已录制的视频/音频文件**：
- 位置：`records/sessions/*/`
- 文件：`*.mp4`, `*.wav`
- **建议**：可以手动删除，不影响复盘功能

**清理命令**（可选）：
```bash
# 查看占用空间
du -sh records/sessions/

# 删除所有视频文件（谨慎操作）
# find records/sessions/ -name "*.mp4" -delete

# 删除所有音频文件（谨慎操作）
# find records/sessions/ -name "*.wav" -delete
```

---

### 前端适配

**前端需要更新**：
- ❌ 停止调用 `/api/report/live/start`（视频录制）
- ✅ 改用 `/api/live_audio/start`（实时转写）

**Electron客户端**：
- 已移除的功能：`localAudioRecorder.js`（客户端本地录制）
- 原因：服务器实时转写已满足需求，不需要客户端录制
