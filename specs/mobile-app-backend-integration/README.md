# 移动端应用与后端服务集成 Spec

## 概述

本 spec 描述了如何将提猫直播助手的后端服务封装部署到服务器，并使用 React Native 构建跨平台移动端应用（iOS + Android）的完整方案。

## 文档结构

1. **[requirements.md](./requirements.md)** - 需求文档
   - 使用 EARS 简易需求语法描述功能需求
   - 包含 8 个核心需求点
   - 定义了验收标准和非功能需求
   - 确定了 UI 设计风格和配色方案

2. **[design.md](./design.md)** - 技术方案设计
   - 整体架构图（Mermaid）
   - 技术栈选型（后端 + 移动端）
   - Docker 容器化部署方案
   - Nginx 反向代理配置
   - API 接口设计
   - 移动端应用结构
   - 安全性设计
   - 部署流程

3. **[tasks.md](./tasks.md)** - 实施计划
   - 11 个阶段的详细任务拆分
   - 每个任务标注了关联的需求点
   - 定义了优先级（P0/P1/P2）

## 核心内容

### 后端服务部署
- Docker 容器化
- Nginx 反向代理和负载均衡
- SSL/TLS 证书配置
- 进程管理和自动重启
- 环境变量和配置管理

### 移动端应用
- React Native + TypeScript
- Redux Toolkit 状态管理
- React Navigation 路由
- WebSocket 实时通信
- 推送通知（FCM/APNs）
- 安全存储（Keychain/Keystore）
- **悬浮窗组件**（AI分析和话术生成）
- **隐藏转录字幕**（移动端特有）

### 核心功能模块
1. 用户认证与授权
2. 直播监控（实时弹幕、礼物）
   - **悬浮窗显示 AI 分析和话术建议**
   - **不显示语音转录字幕**
3. 数据统计与复盘
   - **手动触发复盘生成**（用户点击按钮）
   - **完整复盘能力**（前期版本所有用户都使用完整复盘，不按会员类型限制）
   - **备注**：按会员类型自动生成复盘功能暂时不实现，但已记录为未来功能
4. 订阅与支付管理
5. 设置与配置
6. 推送通知

## 下一步行动

1. **确认需求**：请仔细审阅 `requirements.md`，确认需求是否符合预期
2. **确认技术方案**：请审阅 `design.md`，确认技术选型和架构设计
3. **开始实施**：确认后，按照 `tasks.md` 中的任务列表开始实施

## 注意事项

- 每个阶段完成后需要确认才能进入下一阶段
- 如需要调整需求或方案，请及时沟通
- 实施过程中遇到问题请及时反馈
