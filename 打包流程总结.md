### 第一步：修改后端地址配置 ⚠️ **必须先做**

因为后端已部署到云服务器，需要修改配置让应用连接到远程服务器。

```bash

cd /www/wwwroot/wwwroot/timao-douyin-live-manager
./fix-backend-config.sh

# 根据提示手动修改 electron/main.js
vi electron/main.js

# 验证配置
./verify-config.sh
```

**关键修改**：
```javascript
// electron/main.js
const isDev = !app.isPackaged;

const serviceConfig = {
    main: {
        host: isDev ? '127.0.0.1' : '129.211.218.135',  // ✅ 添加 host
        port: '8181',
        healthPath: '/health'
    }
};
```

## ⚠️ 重要注意事项

### 1. 必须修改后端地址
打包前**必须**修改 `electron/main.js` 的配置，否则应用会尝试连接本地 `127.0.0.1`，导致无法使用。

### 2. 区分开发/生产环境
使用 `app.isPackaged` 区分：
- **开发环境**：连接本地 127.0.0.1
- **生产环境**：连接远程 129.211.218.135

### 3. 防火墙配置
确保服务器防火墙开放以下端口：
- **8181**：后端服务
- **10050**：前端服务（可选）

### 4. 版本检查接口
确保后端有版本检查接口：
```
GET /api/version/check
返回: { "version": "1.0.0", "hasUpdate": false }
```

## 🛠️ 常见问题

### Q1: 打包失败 - 内存不足
```bash
# 添加 swap
sudo fallocate -l 4G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```

### Q2: 应用无法连接后端
```bash
# 检查配置
grep -A 5 "serviceConfig" electron/main.js

# 检查后端服务
curl http://129.211.218.135:8181/health
```

### Q3: 前端构建失败
```bash
# 重新安装依赖
cd electron/renderer
rm -rf node_modules
npm install
npm run build
```

### Q4: 打包很慢
这是正常现象，electron-builder 打包需要：
- 下载依赖（第一次）
- 构建前端
- 打包应用
- 压缩文件

**预计时间**：5-15 分钟（取决于网络和配置）
