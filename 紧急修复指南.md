# 502 错误紧急修复指南

## 问题现象
前端无法连接后端，所有API请求返回 502 Bad Gateway 错误。

## 立即执行步骤

### 1. 运行诊断脚本
```bash
cd /www/wwwroot/wwwroot/timao-douyin-live-manager
chmod +x emergency-fix.sh
./emergency-fix.sh
```

查看输出，确认：
- ❌ 如果所有端口(8000/8001/8002)都未被占用 → **后端服务未启动**
- ❌ 如果Nginx状态异常 → **Nginx配置问题**

### 2. 重启所有服务（最快修复）
```bash
chmod +x restart-all-services.sh
./restart-all-services.sh
```

等待30秒后，尝试登录前端。

### 3. 如果仍然失败，检查日志

#### 后端服务日志
```bash
# FastAPI主服务日志
tail -f logs/main.log

# StreamCap服务日志
tail -f logs/streamcap.log

# Douyin服务日志
tail -f logs/douyin.log
```

#### Nginx日志
```bash
# Nginx错误日志
tail -f /var/log/nginx/error.log

# Nginx访问日志
tail -f /var/log/nginx/access.log
```

### 4. 常见问题排查

#### 问题A: 端口被占用
```bash
# 查看端口占用
netstat -tulnp | grep -E ":(8000|8001|8002)"

# 如果被其他进程占用，杀掉进程
kill -9 <PID>
```

#### 问题B: 虚拟环境依赖缺失
```bash
cd /www/wwwroot/wwwroot/timao-douyin-live-manager
source venv/bin/activate
pip install -r requirements.txt
```

#### 问题C: Nginx配置错误
```bash
# 测试Nginx配置
nginx -t

# 如果有错误，使用模板配置
cp nginx-config-template.conf /etc/nginx/conf.d/timao.conf
nginx -t
systemctl restart nginx
```

#### 问题D: 防火墙阻止
```bash
# 检查防火墙规则
iptables -L -n

# 开放必要端口
./open-ports.sh

# 验证端口
./verify-ports.sh

# 或检查云服务器安全组
# 必须开放端口: 10050（前端）, 8181（后端）
# 可选开放端口: 80（HTTP）, 443（HTTPS）
```

### 5. 手动启动服务（调试用）

如果自动脚本失败，手动启动：

```bash
cd /www/wwwroot/wwwroot/timao-douyin-live-manager
source venv/bin/activate

# 启动FastAPI主服务
python -m uvicorn server.main:app --host 0.0.0.0 --port 8000

# 在另一个终端启动StreamCap服务
python -m uvicorn server.streamcap_service:app --host 0.0.0.0 --port 8001

# 在另一个终端启动Douyin服务
python -m uvicorn server.douyin_service:app --host 0.0.0.0 --port 8002
```

观察是否有错误输出。
